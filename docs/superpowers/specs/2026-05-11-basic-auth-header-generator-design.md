# Basic Auth Header Generator Design

Date: 2026-05-11

## Summary

Add a browser-only ZeroTool utility named `Basic Auth Header Generator`. It generates HTTP Basic Authentication `Authorization` headers from a username and password, and decodes existing Basic auth headers or token-only values back into their username/password parts for verification.

The tool does not send network requests, does not test credentials against a URL, and does not persist entered credentials.

## Goal

Capture the `online basic auth header generator` search intent with a focused developer/security tool that complements the existing JWT, HMAC, PKCE, and cURL tools.

Primary user jobs:

- Generate `Authorization: Basic <token>` from username and password.
- Copy the full header, the token-only value, a cURL header example, and a Fetch headers snippet.
- Paste a full Basic header or token-only value and decode it to inspect the credential pair.
- Understand that Basic auth uses Base64 encoding, not encryption.

## Scope

In scope:

- New slug: `basic-auth-header-generator`.
- New category: `security`.
- Related tools: `jwt-decoder`, `jwt-generator`, `hmac-generator`, `curl-to-code`.
- One Astro tool component: `src/components/tools/BasicAuthHeaderGeneratorTool.astro`.
- Registry, metadata, icon, persistence policy, tool content, and a four-language guide blog.
- UTF-8 safe Basic auth encoding and decoding.
- Desktop/mobile and light/dark visual support through existing design tokens.

Out of scope:

- Sending HTTP requests.
- Storing, testing, or validating real credentials against a server.
- Full API client behavior.
- Any external runtime dependency.

## User Interface

Use a single `.tool-widget` panel with a compact two-column desktop layout that collapses to one column on mobile.

Generate panel:

- Username `.tool-input`.
- Password `.tool-input` with show/hide toggle.
- Primary action to generate the header.
- Output cards for full header, token-only value, cURL snippet, and Fetch snippet.
- Copy buttons using `.btn-copy`.

Decode panel:

- `.tool-textarea` accepting either `Authorization: Basic <token>`, `Basic <token>`, or `<token>`.
- Decode action.
- Result rows for username and password.
- Error status for invalid scheme, invalid Base64, malformed decoded text, or empty input.

Shared behavior:

- Use component prefix `bahg-` for custom CSS.
- Use `aria-live` status text for user feedback.
- Keep all controls within the `.tool-widget` width at 390px mobile viewport.
- Use tokens from `DESIGN.md` and shared classes from `src/styles/tool-common.css`.

## Encoding And Decoding Rules

Generation:

- Join credentials as `username + ':' + password`.
- Encode the joined string as UTF-8 bytes with `TextEncoder`.
- Convert bytes to Base64 without calling `btoa` directly on Unicode strings.
- Produce `Authorization: Basic <base64>`.

Decoding:

- Trim input.
- Accept optional `Authorization:` prefix case-insensitively.
- Accept optional `Basic` scheme case-insensitively.
- Reject non-Basic schemes when a scheme is present.
- Decode Base64 to UTF-8 with `TextDecoder`.
- Split on the first `:` only, so passwords may contain additional colons.
- Reject decoded strings without a colon.

## Privacy

Add `basic-auth-header-generator: 'disabled'` to `src/data/persistence.ts`.

Reason: usernames and passwords are credentials. The global persistence layer must wipe historical values for this slug on page load and the tool must not save inputs.

## Verification

Static verification:

- `node scripts/audit.mjs` passes with no FAIL.
- `npm run build` passes.

Manual browser verification:

- Open `/tools/basic-auth-header-generator/` and at least one localized route.
- Generate from `Aladdin` and `open sesame`, expecting token `QWxhZGRpbjpvcGVuIHNlc2FtZQ==`.
- Decode `Authorization: Basic QWxhZGRpbjpvcGVuIHNlc2FtZQ==`, expecting username `Aladdin` and password `open sesame`.
- Decode `Basic dXNlcjpwOmE6c3M=`, expecting username `user` and password `p:a:ss`.
- Invalid input shows an error and does not crash.
- Desktop light, mobile light, desktop dark, and mobile dark render without horizontal overflow.
- Console errors count is zero during generate, copy, decode, and clear flows.
