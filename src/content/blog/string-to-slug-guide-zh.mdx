Slug 是 URL 中用来标识页面内容的部分，以人类可读的方式呈现。在 `https://example.com/blog/how-to-deploy-to-production` 中，slug 就是 `how-to-deploy-to-production`。Slug 全部小写，用连字符代替空格，只包含 ASCII 安全字符。正确处理 slug 对 SEO、链接稳定性和用户体验都至关重要。

## 什么是合法的 Slug

URL slug 必须满足以下约束：

1. **全部小写** — 大多数 Web 服务器将 `My-Post` 和 `my-post` 视为不同路径；始终使用小写
2. **不含空格** — URL 中的空格会被编码为 `%20`；用连字符替代
3. **只含 ASCII 字符** — 带重音的字母（`é`、`ö`、`ñ`）在旧系统中会破坏 URL；需转写
4. **不含特殊字符** — `!`、`@`、`#`、`?`、`&` 是 URL 保留字符；需去除
5. **分隔符统一** — 推荐使用连字符（Google 官方 SEO 建议），或统一使用下划线
6. **首尾不含分隔符** — `/blog/-my-post-` 是格式错误的；需去除两端的分隔符
7. **不含连续分隔符** — `my--post` 暗示处理有问题；合并为单个分隔符

### 连字符还是下划线？

Google 官方文档明确建议在 URL 中使用连字符作为单词分隔符。下划线在某些社区有约定俗成的用法（Python 包名、部分 CMS 系统）。除非有特殊原因，否则使用连字符。

## Slug 转换的工作原理

### 第一步：转写重音字符

去除字符之前，先将带重音和特殊字符映射为 ASCII 等价形式：

```
café   → cafe
naïve  → naive
résumé → resume
Ñoño   → Nono
```

跳过这一步会丢失信息——`café` 变成 `caf`（`é` 被直接去掉）。

### 第二步：将符号替换为单词

某些符号有有意义的英文对应词：

```
AT&T  → at-and-t   (& → and)
C@t   → cat         (@ → at)
100%  → 100percent  (% → percent)
```

### 第三步：去除剩余非 ASCII 字符

转写后，去掉所有非字母、数字、空格、连字符、下划线的字符。

### 第四步：规范化空格和分隔符

将所有连续空格替换为所选分隔符，然后合并连续分隔符：

```
"Hello   World" → "hello-world"  (不是 "hello---world")
```

### 第五步：去除首尾分隔符

```
"-hello-world-" → "hello-world"
```

## 常见 Slug 示例

### 博客文章和文档

```
"如何将 Django 应用部署到 AWS" → ru-he-jiang-django-ying-yong-bu-shu-dao-aws
"Python 入门：10 个必备技巧"  → python-ru-men-10-ge-bi-bei-ji-qiao
"API 设计最佳实践"           → api-she-ji-zui-jia-shi-jian
```

注意：中文字符默认没有 ASCII 对应，会被过滤。建议中文内容的 URL 使用英文 slug（手动填写或用 pinyin 辅助）。

### 产品和包名

```
"React Query" → react-query
"@tanstack/react-query" → tanstack-react-query
"node-fetch v3" → node-fetch-v3
```

### 文件路径

```
"我的文档/项目笔记" → /my-documents/project-notes （需配合英文化处理）
"API 文档 (v2)"   → api-v2
```

## 代码实现

### JavaScript / TypeScript

```typescript
function toSlug(input: string, separator = '-'): string {
  const charMap: Record<string, string> = {
    à: 'a', á: 'a', â: 'a', ä: 'a', å: 'a', æ: 'ae',
    ç: 'c', è: 'e', é: 'e', ê: 'e', ë: 'e',
    ì: 'i', í: 'i', î: 'i', ï: 'i',
    ñ: 'n', ò: 'o', ó: 'o', ô: 'o', ö: 'o', ø: 'o',
    ù: 'u', ú: 'u', û: 'u', ü: 'u', ý: 'y', ß: 'ss',
    '&': 'and', '@': 'at', '%': 'percent',
  };

  return input
    .toLowerCase()
    .replace(/[^\u0000-\u007E]/g, c => charMap[c] ?? '')
    .replace(/[^a-z0-9\s\-_]/g, ' ')
    .replace(/[\s]+/g, separator)
    .replace(/[\-_]+/g, separator)
    .replace(new RegExp(`^${separator}+|${separator}+$`, 'g'), '');
}

// 使用示例
toSlug('Hello World!')  // → 'hello-world'
toSlug('café au lait')  // → 'cafe-au-lait'
toSlug('AT&T Corp.')    // → 'at-and-t-corp'
```

### Python

```python
import re
import unicodedata

def to_slug(text: str, separator: str = '-') -> str:
    # Unicode 规范化和 ASCII 转写
    text = unicodedata.normalize('NFKD', text)
    text = text.encode('ascii', 'ignore').decode('ascii')
    text = text.lower()
    # 去除特殊字符
    text = re.sub(r'[^a-z0-9\s\-_]', ' ', text)
    text = re.sub(r'[\s]+', separator, text)
    text = re.sub(r'[\-_]+', separator, text)
    text = text.strip(separator)
    return text

# 使用示例
to_slug('Hello World!')   # → 'hello-world'
to_slug('café résumé')    # → 'cafe-resume'
```

### Ruby on Rails

Rails 内置了 `parameterize` 方法：

```ruby
"Hello World!".parameterize           # => "hello-world"
"Hello World!".parameterize(separator: "_")  # => "hello_world"
"naïve café".parameterize             # => "naive-cafe"
```

## Slug 的 SEO 注意事项

**保持简短且描述性强。** Google 建议 URL 能反映页面内容。`how-to-build-a-rest-api` 优于 `how-to-build-a-restful-api-from-scratch-for-beginners-step-by-step`。

**包含主要关键词。** Slug 是排名信号之一。如果目标关键词是 "部署 Docker 容器"，slug `deploy-docker-container` 会有帮助。

**避免不必要的停用词。** "a"、"the"、"to" 等冠词增加长度但不带来 SEO 价值。`build-rest-api` 优于 `how-to-build-a-rest-api`（在不影响理解的前提下）。

**发布后不要修改 Slug。** 修改 slug 会导致所有现有反链失效并产生 404，除非设置 301 重定向。在发布前规划好 slug。

**重复内容使用 canonical URL。** 如果同一内容可通过多个路径访问，用 `<link rel="canonical">` 标签告知搜索引擎哪个 slug 是权威版本。

## 常见问题

### 结果为空

所有字符都被过滤了——通常因为输入只包含符号或没有转写映射的非 ASCII 字符：

```
"!!!"  → ""   （全部被去除）
"中文" → ""   （未配置 CJK 转写）
```

解决方案：为中日韩字符扩展字符映射表，或使用 `limax` 等支持多语言的专用库。

### 重复 Slug

两个不同的标题生成了相同的 slug：

```
"API 设计"  → api
"API 设计！" → api
```

在数据库驱动的系统中，在 slug 列上建立唯一索引，并为重复项追加数字后缀：`api-2`。

### 分隔符不一致

在 URL 方案中混用连字符和下划线会让爬虫和用户感到困惑。选定一种后在应用层面强制执行。
