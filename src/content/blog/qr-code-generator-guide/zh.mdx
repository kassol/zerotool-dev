## 二维码能存什么内容

二维码本质上是文本的视觉编码，最多能存几千个字符。手机识别到不同格式的内容时，会触发不同的行为：

| 内容类型 | 手机响应 | 格式示例 |
|---------|---------|---------|
| 网址 URL | 用浏览器打开 | `https://example.com` |
| Wi-Fi 凭据 | 提示连接网络 | `WIFI:T:WPA;S:网络名;P:密码;;` |
| 纯文本 | 显示文字 | 任意字符串 |
| 邮件 | 打开邮件客户端 | `mailto:user@example.com` |
| 电话 | 打开拨号界面 | `tel:+8613800138000` |
| 短信 | 打开短信界面 | `sms:+8613800138000?body=你好` |
| vCard 名片 | 添加到通讯录 | `BEGIN:VCARD...END:VCARD` |
| 地理位置 | 打开地图 | `geo:39.9042,116.4074` |

iOS 11+ 和 Android 9+ 的系统相机原生识别二维码，不需要单独安装扫码 App。

## 网址二维码

最常见的用法。粘贴 URL 直接生成。URL 越短，二维码图案越稀疏，远距离或低分辨率打印时扫描成功率越高。过长的 URL 建议先用短链服务处理。

实用建议：
- 用 HTTPS 而不是 HTTP
- 加 UTM 参数追踪扫码来源：`?utm_source=print&utm_medium=flyer&utm_campaign=2026q2`
- 打印前在 iOS 和 Android 上各测一遍

## Wi-Fi 连接二维码

Wi-Fi 二维码按照标准格式把网络名、密码、加密类型编码进去。Android 10+ 和 iOS 11+ 扫描后会弹出加入网络的提示，一键连接，不需要手动输密码。

格式：

```
WIFI:T:WPA;S:你的网络名称;P:你的密码;;
```

字段说明：
- `T`：加密类型，`WPA` 最常见，还有 `WEP`（已不安全）和 `nopass`（开放网络）
- `S`：Wi-Fi 名称（SSID）
- `P`：密码

生成器填好三个字段后自动产出正确格式。把二维码打印出来贴墙上，来访客人和新同事不用再问密码了。

## 联系人名片二维码

vCard 格式的二维码让对方扫一下就能把你的信息存进通讯录：

```
BEGIN:VCARD
VERSION:3.0
FN:张三
ORG:某某科技
TEL:+86-138-0013-8000
EMAIL:zhangsan@example.com
URL:https://example.com
END:VCARD
```

适合用在：纸质名片背面、会议门牌、电子邮件签名图片。

## 纠错级别

二维码包含冗余数据，即使部分被遮挡或破损也能正常识别。有四个级别：

| 级别 | 恢复能力 | 适用场景 |
|------|---------|---------|
| L（低） | ~7% | 干净的数字屏幕 |
| M（中） | ~15% | 大多数印刷品 |
| Q（高中） | ~25% | 预期有轻微磨损 |
| H（高） | ~30% | 中间放 Logo、恶劣环境 |

**M 级**适合大多数场景。如果要在二维码中心叠加 Logo，选 **H 级**——Logo 会遮挡部分数据，高纠错率来弥补。

注意：纠错级别越高，图案越密集，远距离或小尺寸时扫码成功率反而下降。别过度配置。

## 尺寸和格式

**尺寸**：印刷品最小实用尺寸约 2cm × 2cm，正常手持距离（15–20cm）可以顺利扫描。大幅海报或展板按比例放大即可。

**格式选择**：
- **PNG**：用于网页、邮件、文档等数字场景
- **SVG**：用于印刷，矢量格式任意放大不失真
- **JPEG**：避免使用，有损压缩会模糊边缘，影响扫码

要打印就下载 SVG。

## 打印前必须测试

打印前在真机上验证：

1. iOS 相机直接扫
2. Android 相机或微信扫一扫
3. 在实际扫描距离下测试
4. 确认行为正确——网址是否打开、Wi-Fi 是否连上、联系人是否添加成功

一旦印出来几百张，错误 URL 或错误密码就很麻烦了。

## 静态二维码 vs 动态二维码

**静态二维码**（免费工具生成的）把完整内容编码在图案里。打印后内容无法修改，URL 变了就要重新生成和打印。

**动态二维码**编码的是一个短跳转链接，后台可以随时改目标 URL，适合营销活动，但需要付费服务。

对开发者和个人来说，静态二维码完全够用。
