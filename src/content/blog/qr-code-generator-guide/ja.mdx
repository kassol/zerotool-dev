## QRコードがエンコードできるもの

QRコードは数千文字までの任意のテキストを格納できます。コンテンツの種類によって、スマートフォンがコードを解釈して実行するアクションが決まります：

| コンテンツの種類 | スマートフォンの動作 | フォーマット |
|-------------|----------------|--------|
| URL | ブラウザで開く | `https://example.com` |
| Wi-Fi認証情報 | ネットワークへの参加を促す | `WIFI:T:WPA;S:NetworkName;P:password;;` |
| プレーンテキスト | テキストを表示 | 任意の文字列 |
| メール | メールアプリを開く | `mailto:user@example.com` |
| 電話 | ダイアラーを開く | `tel:+1234567890` |
| SMS | SMSアプリを開く | `sms:+1234567890?body=Hello` |
| vCard連絡先 | 連絡先に追加 | `BEGIN:VCARD...END:VCARD` |
| 位置情報 | マップアプリを開く | `geo:37.7749,-122.4194` |

iOS 11+とAndroid 9+では、ほとんどのスマートフォンカメラアプリがネイティブにQRコードをデコードできます — 別途アプリは不要です。

## URL用QRコード

最も一般的なユースケースです。URLを貼り付けて生成します。QRコードパターンを密にせず、スキャンしやすくするために、URLは短くしましょう — 短いドメインかURLショートナーを使います。

URLのQRコードのヒント：
- HTTPSのURLを使用（HTTPのコードはスキャンできますがプロフェッショナルでない印象）
- アナリティクスでスキャンをトラッキングするためにUTMパラメーターを追加：`?utm_source=print&utm_medium=flyer`
- 印刷前に複数のデバイスでテスト

## Wi-Fi用QRコード

Wi-Fi QRコードは、Android 10+とiOS 11+がネイティブに読み取ってネットワークに自動参加できる標準フォーマットでネットワーク名（SSID）、パスワード、セキュリティタイプをエンコードします。

フォーマット：

```
WIFI:T:WPA;S:YourNetworkName;P:YourPassword;;
```

フィールドの値：
- `T`：セキュリティタイプ — `WPA`（最も一般的）、`WEP`、または `nopass`（オープンネットワーク）
- `S`：SSID（ネットワーク名）
- `P`：パスワード

オフィスの壁にあるWi-Fi QRコードは、すべての新しい来客がパスワードを尋ねる手間を省きます。フィールドに入力すると、ジェネレーターが正しいフォーマットで作成します。

## 連絡先情報用QRコード

vCard QRコードを使えば、1回のスキャンで相手が連絡先に詳細情報を追加できます。ジェネレーターは標準vCardフィールドをサポートしています：

```
BEGIN:VCARD
VERSION:3.0
FN:山田 太郎
ORG:株式会社サンプル
TEL:+81 3 0000 0000
EMAIL:taro@sample.co.jp
URL:https://sample.co.jp
END:VCARD
```

名刺、カンファレンスバッジ、メール署名に便利です。

## エラー訂正レベル

QRコードには冗長データが含まれており、部分的に隠れたり損傷したりしてもデコードできます。4つのレベル：

| レベル | 回復能力 | 使用例 |
|-------|------------------|----------|
| L（低） | 約7% | クリーンなデジタルディスプレイ |
| M（中） | 約15% | ほとんどの印刷物 |
| Q（クォータイル） | 約25% | 軽度の摩耗が想定される |
| H（高） | 約30% | ロゴオーバーレイ、過酷な環境 |

**レベルM** がほとんどのユースケースに適しています。QRコードの中央にロゴをオーバーレイする場合は **H** を使用してください — ロゴがデータの一部を塞ぎ、エラー訂正がそれを補います。

高いエラー訂正 = より複雑なパターン = 距離からのスキャンが難しい。クリーンな印刷上のシンプルなURLに対してやりすぎないように。

## サイズとフォーマット

**サイズ**：最小実用印刷サイズは約2cm × 2cmです。そのサイズでは、ほとんどのスマートフォンカメラが15〜20cmの距離からスキャンできます。壁のポスターや大型ディスプレイの場合は比例して拡大してください。

**フォーマット**：
- **PNG**：デジタル使用に最適 — ウェブサイト、メール、ドキュメント
- **SVG**：印刷に最適 — 無限にスケーラブル、どんなサイズでもピクセル化しない
- **JPEG**：避けてください — 非可逆圧縮がQRスキャナーに必要なシャープなエッジを劣化させます

任意のサイズで印刷する場合はSVGをダウンロードしてください。

## 印刷前のテスト

常に印刷前にQRコードをテストしてください：
1. iOS（Safariカメラ）でテスト
2. Android（Google LensまたはネイティブカメラARカメラ）でテスト
3. 意図したスキャン距離でテスト
4. アクションを確認 — URLが開くか、Wi-Fiが接続されるか？

URLの入力ミスや間違ったWi-Fiパスワードは、500枚のチラシを印刷した後では永続的な問題になります。

## 動的QRコード vs 静的QRコード

**静的QRコード**（無料ジェネレーターが生成するもの）はコンテンツ全体をパターンにエンコードします。印刷後はコンテンツを変更できません。URLが変わった場合は新しいQRコードが必要です。

**動的QRコード**は短いリダイレクトURLをエンコードします。印刷後もリダイレクト先を変更できます。変化する目的地を持つキャンペーンには便利ですが、有料サービスが必要です。

ほとんどの開発者や個人用途では静的コードで十分です。
