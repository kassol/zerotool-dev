YAML 是基础设施配置的通用语言——Kubernetes 清单、Docker Compose、GitHub Actions、Helm Chart、Ansible Playbook 都用它。但当你需要调用 API、编写脚本、或使用只认 JSON 的工具时，格式转换就成了绕不过去的步骤。

[在线 YAML 转 JSON →](/tools/yaml-to-json)

## 什么时候需要把 YAML 转成 JSON？

| 场景 | 原因 |
|---|---|
| 直接调用 Kubernetes REST API | API 接口要求 JSON 请求体 |
| Terraform / Pulumi 数据源 | 部分 Provider 只接受 JSON 格式配置 |
| 用 `jq` 提取字段 | `jq` 只处理 JSON，需要先转换 |
| SDK 配置文件 | 很多 SDK 原生读取 JSON，不支持 YAML |
| 调试 YAML 解析结果 | 用 JSON 确认解析器实际看到的结构 |

## 如何使用在线工具

无需安装任何工具：

1. 打开 [YAML 转 JSON 工具](/tools/yaml-to-json)

3. JSON 结果实时出现，语法错误同步高亮提示
4. 复制 JSON 或下载为 `.json` 文件

## 实战示例

### Kubernetes ConfigMap

典型的 Kubernetes 配置清单：

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: app-config
  namespace: production
data:
  DATABASE_URL: "postgres://db.internal:5432/app"
  LOG_LEVEL: "info"
  FEATURE_FLAGS: "new-ui,beta-api"
```

转换为 JSON：

```json
{
  "apiVersion": "v1",
  "kind": "ConfigMap",
  "metadata": {
    "name": "app-config",
    "namespace": "production"
  },
  "data": {
    "DATABASE_URL": "postgres://db.internal:5432/app",
    "LOG_LEVEL": "info",
    "FEATURE_FLAGS": "new-ui,beta-api"
  }
}
```

转换完成后可以直接用 `curl` 调用 Kubernetes API：

```bash
curl -X POST https://k8s.api/v1/namespaces/production/configmaps \
  -H "Content-Type: application/json" \
  -d @configmap.json
```

### Docker Compose

```yaml
version: "3.9"
services:
  api:
    image: myapp:latest
    ports:
      - "8080:8080"
    environment:
      NODE_ENV: production
    depends_on:
      - db
  db:
    image: postgres:16-alpine
    volumes:
      - pgdata:/var/lib/postgresql/data
volumes:
  pgdata:
```

转换 Compose 为 JSON 后，可以用 Python/Node.js 做程序化分析，比如自动检测镜像版本、扫描环境变量配置、生成服务拓扑图等。

### GitHub Actions 工作流

```yaml
name: Deploy
on:

jobs:
```

转换为 JSON 后，可以用标准 JSON 库构建工作流分析或生成工具。

## 转换规则一览

| YAML 特性 | JSON 等价形式 |
|---|---|
| `true` / `false` | `true` / `false` |
| `null` 或 `~` | `null` |
| 整数 | 无小数点的数字 |
| 浮点数 | 带小数点的数字 |
| 多行字符串（`\|` 或 `>`） | 单行字符串，换行转为 `\n` |
| 锚点和引用（`&` / `*`） | 展开为完整值 |
| 注释（`#`） | 删除 |
| 缩进层级 | 转为 `{}` 和 `[]` |

## 国内开发者常踩的坑

### 布尔值陷阱

YAML 1.1（目前绝大多数工具使用的版本）中，`yes`、`no`、`on`、`off` 是布尔值，不是字符串：

```yaml
# 这段 YAML
feature_enabled: yes
notifications: on
debug_mode: off
```

转换为 JSON 后：

```json
{
  "feature_enabled": true,
  "notifications": true,
  "debug_mode": false
}
```

国内 CI 平台配置（如 Jenkins Pipeline、GitLab CI）里很容易写出这类配置。如果要表达字符串 `"yes"`，必须加引号：`feature_enabled: "yes"`。

### URL 必须加引号

YAML 中冒号加空格（`: `）是键值分隔符，URL 直接写会解析出错：

```yaml
# 错误写法——解析器把 http 当成键名
endpoint: http://api.example.com/v1

# 正确写法
endpoint: "http://api.example.com/v1"
```

这在 Ansible playbook 和 Kubernetes manifest 中是高频错误。

### 八进制数字陷阱

`0755` 在 YAML 1.1 中会被解析为八进制数 493，而不是字符串。文件权限、chmod 值必须加引号：

```yaml
# 错误——0755 会变成整数 493
permissions: 0755

# 正确
permissions: "0755"
```

### 锚点在 JSON 中消失

YAML 支持锚点（`&`）和引用（`*`）来避免重复，但转换为 JSON 时这些引用会被完全展开：

```yaml
defaults: &defaults
  timeout: 30
  retries: 3

service_a:
  <<: *defaults
  host: a.example.com
```

转换后：

```json
{
  "defaults": { "timeout": 30, "retries": 3 },
  "service_a": { "timeout": 30, "retries": 3, "host": "a.example.com" }
}
```

锚点不复存在，所有值都被内联展开。这是预期行为。

## 命令行与代码转换

### `yq`（推荐的 CLI 工具）

```bash
# macOS 安装
brew install yq

# YAML 转 JSON
yq -o=json input.yaml > output.json

# 配合 jq 提取字段
yq -o=json input.yaml | jq '.metadata.name'
```

### Python

```python
import yaml
import json

with open("config.yaml") as f:
    data = yaml.safe_load(f)  # 不要用 yaml.load()，存在安全风险

with open("config.json", "w") as f:
    json.dump(data, f, indent=2, ensure_ascii=False)
```

### Node.js

```bash
npm install js-yaml
```

```javascript
const yaml = require('js-yaml');
const fs = require('fs');

const data = yaml.load(fs.readFileSync('config.yaml', 'utf8'));
fs.writeFileSync('config.json', JSON.stringify(data, null, 2));
```

### Bash 一行命令（Python 内置）

```bash
python3 -c "import yaml,json,sys; print(json.dumps(yaml.safe_load(sys.stdin), indent=2, ensure_ascii=False))" < config.yaml
```

## 转换前先验证 YAML

格式错误是转换失败最常见的原因：

- **不能用 Tab 缩进**：YAML 严格要求空格，一个 Tab 就会报错。编辑器务必配置"Tab 转空格"。
- **缩进不一致**：全文统一用两个空格是惯例。
- **特殊字符未加引号**：值中包含冒号、方括号、花括号时需要加引号。

在线工具会在转换时实时提示错误，精确到出问题的行号。
