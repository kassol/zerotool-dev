スラグはURLの中でページを人間が読める形式で識別する部分です。`https://example.com/blog/how-to-deploy-to-production` では、スラグは `how-to-deploy-to-production` です。スラグは小文字のみで、スペースの代わりにハイフンを使い、ASCIIセーフな文字のみを含みます。スラグを正しく扱うことはSEO、リンクの安定性、ユーザー体験に直結します。

[文字列をURLスラグに変換する →](/tools/string-to-slug)

## 有効なスラグの条件

URLスラグはこれらの制約を満たす必要があります：

1. **小文字のみ** — ほとんどのWebサーバーは`My-Post`と`my-post`を異なるパスとして扱います
2. **スペースなし** — URLのスペースは`%20`にエンコードされます；ハイフンを使用
3. **ASCII文字のみ** — アクセント文字（`é`、`ö`、`ñ`）は古いシステムでURLを壊す；変換が必要
4. **特殊文字なし** — `!`、`@`、`#`、`?`、`&`はURL予約文字；除去が必要
5. **区切り文字の統一** — ハイフン（GoogleのSEO推奨）またはアンダースコアを一貫して使用
6. **先頭・末尾の区切り文字なし** — `/blog/-my-post-`は不正な形式；両端の区切り文字を除去
7. **連続した区切り文字なし** — `my--post`は処理上の問題を示唆；単一の区切り文字に統合

### ハイフンかアンダースコアか

Googleの公式ドキュメントでは、URLの単語区切りにハイフンを推奨しています。アンダースコアはPythonパッケージや一部のCMSシステムで慣習的に使われています。特別な理由がない限り、ハイフンを使用しましょう。

## スラグ変換の仕組み

### ステップ1：アクセント文字の変換

文字を削除する前に、アクセント文字と特殊文字をASCII等価形式にマッピングします：

```
café   → cafe
naïve  → naive
résumé → resume
```

このステップをスキップすると情報が失われます——`café`は`caf`になります（`é`が完全に除去される）。

### ステップ2：記号を単語に置換

一部の記号には意味のある英語の対応語があります：

```
AT&T  → at-and-t   (& → and)
100%  → 100percent  (% → percent)
```

### ステップ3：残りの非ASCII文字を除去

変換後、文字・数字・スペース・ハイフン・アンダースコア以外をすべて除去します。

### ステップ4：空白と区切り文字の正規化

すべての連続する空白を選択した区切り文字に置換し、連続する区切り文字を統合します：

```
"Hello   World" → "hello-world"  ("hello---world" ではない)
```

### ステップ5：両端の区切り文字を除去

```
"-hello-world-" → "hello-world"
```

## コードでの実装

### JavaScript / TypeScript

```typescript
function toSlug(input: string, separator = '-'): string {
  const charMap: Record<string, string> = {
    à: 'a', á: 'a', â: 'a', ä: 'a', å: 'a', æ: 'ae',
    ç: 'c', è: 'e', é: 'e', ê: 'e', ë: 'e',
    ì: 'i', í: 'i', î: 'i', ï: 'i',
    ñ: 'n', ò: 'o', ó: 'o', ô: 'o', ö: 'o', ø: 'o',
    ù: 'u', ú: 'u', û: 'u', ü: 'u', ý: 'y', ß: 'ss',
    '&': 'and', '@': 'at', '%': 'percent',
  };

  return input
    .toLowerCase()
    .replace(/[^\u0000-\u007E]/g, c => charMap[c] ?? '')
    .replace(/[^a-z0-9\s\-_]/g, ' ')
    .replace(/[\s]+/g, separator)
    .replace(/[\-_]+/g, separator)
    .replace(new RegExp(`^${separator}+|${separator}+$`, 'g'), '');
}

// 使用例
toSlug('Hello World!')  // → 'hello-world'
toSlug('café au lait')  // → 'cafe-au-lait'
```

### Python

```python
import re
import unicodedata

def to_slug(text: str, separator: str = '-') -> str:
    text = unicodedata.normalize('NFKD', text)
    text = text.encode('ascii', 'ignore').decode('ascii')
    text = text.lower()
    text = re.sub(r'[^a-z0-9\s\-_]', ' ', text)
    text = re.sub(r'[\s]+', separator, text)
    text = re.sub(r'[\-_]+', separator, text)
    text = text.strip(separator)
    return text
```

### Ruby on Rails

Railsには`parameterize`メソッドが組み込まれています：

```ruby
"Hello World!".parameterize           # => "hello-world"
"naïve café".parameterize             # => "naive-cafe"
```

## スラグのSEO考慮事項

**短くて説明的なスラグを使用する。** GoogleはURLがページの内容を反映することを推奨しています。`how-to-build-a-rest-api`は`how-to-build-a-restful-api-from-scratch-for-beginners-step-by-step`よりも優れています。

**主要キーワードを含める。** スラグはランキングシグナルです。ターゲットキーワードが"deploy docker container"であれば、スラグ`deploy-docker-container`が有効です。

**不要なストップワードを避ける。** "a"、"the"、"to"などの冠詞は長さを増やすだけでSEO価値はありません。

**公開後にスラグを変更しない。** スラグを変更すると既存のバックリンクが無効になり、301リダイレクトを設定しない限り404が発生します。公開前にスラグを計画しましょう。

## よくある問題

### 空の結果

すべての文字が除去された場合——通常は入力が記号のみ、または変換マッピングのない非ASCII文字だけの場合：

```
"!!!" → ""          (すべて除去)
"日本語" → ""       (変換設定なし)
```

解決策：CJK文字の文字マッピングを拡張するか、多言語対応の`limax`などのライブラリを使用します。

### 重複スラグ

2つの異なるタイトルが同じスラグを生成する：

```
"API Design"  → api-design
"API Design!" → api-design
```

データベース駆動のシステムでは、slugカラムにユニークインデックスを設定し、重複には数字サフィックスを追加します：`api-design-2`。
