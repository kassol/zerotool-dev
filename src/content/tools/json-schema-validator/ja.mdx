<h2>使い方</h2>
<ol>
  <li>左側に <strong>JSONスキーマ</strong>、右側に <strong>JSONデータ</strong>を貼り付けます。</li>
  <li>ドロップダウンからスキーマドラフト（Draft-07またはDraft 2020-12）を選択します。</li>
  <li><strong>Validate</strong> ボタンをクリックします（または <kbd>Ctrl</kbd>+<kbd>Enter</kbd>）。</li>
  <li>有効な場合、データがスキーマに適合していることを示す緑のメッセージが表示されます。</li>
  <li>無効な場合、各エラーの <strong>JSONパス</strong>・エラーメッセージ・追加パラメータが一覧表示されます。</li>
</ol>

<p><strong>Load Example</strong> をクリックすると、サンプルのUserスキーマとJSONが自動入力され、すぐに試せます。</p>

<h2>エラー出力の見方</h2>

<p>各バリデーションエラーは3つの情報を含みます：</p>
<ul>
  <li><strong>パス</strong> — 失敗した値へのJSONポインター（例：<code>/items/0/price</code>、トップレベルは<code>(root)</code>）。</li>
  <li><strong>メッセージ</strong> — AJVによる分かりやすいエラー説明（例：<em>must be integer</em>、<em>must have required property 'name'</em>）。</li>
  <li><strong>パラメータ</strong> — 許可される型・最小値・必要なパターンなどの追加情報。</li>
</ul>

<h2>よく使うJSONスキーマのキーワード</h2>
<ul>
  <li><code>type</code> — JSONの型を制限：<code>string</code>・<code>number</code>・<code>integer</code>・<code>boolean</code>・<code>array</code>・<code>object</code>・<code>null</code>。</li>
  <li><code>required</code> — オブジェクトに必須のプロパティ名の配列。</li>
  <li><code>properties</code> — オブジェクトの各プロパティの制約を定義。</li>
  <li><code>additionalProperties: false</code> — <code>properties</code>に定義されていないプロパティを拒否。</li>
  <li><code>minimum</code> / <code>maximum</code> — 数値の範囲制約。</li>
  <li><code>minLength</code> / <code>maxLength</code> — 文字列の長さ制約。</li>
  <li><code>pattern</code> — 正規表現で文字列を検証（例：<code>"^[a-z]+"</code>）。</li>
  <li><code>enum</code> — 固定の値セットに制限。</li>
  <li><code>items</code> — 配列要素のスキーマを定義。</li>
</ul>

<h2>スキーマの例</h2>

```json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "type": "object",
  "required": ["name", "age"],
  "properties": {
    "name":  { "type": "string", "minLength": 1 },
    "age":   { "type": "integer", "minimum": 0 },
    "email": { "type": "string" }
  },
  "additionalProperties": false
}
```

<p>有効なデータ：<code>{'{"name":"Alice","age":30}'}</code><br />
無効なデータ：<code>{'{"name":"","age":-1,"extra":true}'}</code> — 3つのエラーが発生します。</p>
