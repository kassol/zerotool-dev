<h2>使用方法</h2>
<ol>
  <li>在左侧粘贴 <strong>JSON Schema</strong>，在右侧粘贴 <strong>JSON 数据</strong>。</li>
  <li>通过下拉菜单选择 Schema 草案版本（Draft-07 或 Draft 2020-12）。</li>
  <li>点击 <strong>Validate</strong>（或按 <kbd>Ctrl</kbd>+<kbd>Enter</kbd>）。</li>
  <li>若数据合法，绿色提示表示数据符合 Schema。</li>
  <li>若数据不合法，每条错误显示对应的 <strong>JSON 路径</strong>、错误信息和附加参数。</li>
</ol>

<p>点击 <strong>Load Example</strong> 可自动填入示例 User Schema 和对应 JSON，快速体验工具功能。</p>

<h2>读懂错误输出</h2>

<p>每条验证错误包含以下三部分：</p>
<ul>
  <li><strong>路径</strong> — JSON Pointer 格式的失败值路径（如 <code>/items/0/price</code>，根节点为 <code>(root)</code>）。</li>
  <li><strong>信息</strong> — AJV 输出的通俗错误描述（如 <em>must be integer</em>、<em>must have required property 'name'</em>）。</li>
  <li><strong>参数</strong> — 附加上下文，如允许的类型、最小值或所需的正则模式。</li>
</ul>

<h2>常用 JSON Schema 关键字</h2>
<ul>
  <li><code>type</code> — 限制 JSON 类型：<code>string</code>、<code>number</code>、<code>integer</code>、<code>boolean</code>、<code>array</code>、<code>object</code>、<code>null</code>。</li>
  <li><code>required</code> — 对象中必须包含的属性名数组。</li>
  <li><code>properties</code> — 为对象的各命名属性定义约束。</li>
  <li><code>additionalProperties: false</code> — 拒绝 <code>properties</code> 以外的任何属性。</li>
  <li><code>minimum</code> / <code>maximum</code> — 数值范围约束。</li>
  <li><code>minLength</code> / <code>maxLength</code> — 字符串长度约束。</li>
  <li><code>pattern</code> — 用正则验证字符串（如 <code>"^[a-z]+"</code>）。</li>
  <li><code>enum</code> — 限制为固定的值集合。</li>
  <li><code>items</code> — 定义数组元素的 Schema。</li>
</ul>

<h2>示例 Schema</h2>

```json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "type": "object",
  "required": ["name", "age"],
  "properties": {
    "name":  { "type": "string", "minLength": 1 },
    "age":   { "type": "integer", "minimum": 0 },
    "email": { "type": "string" }
  },
  "additionalProperties": false
}
```

<p>合法数据：<code>{'{"name":"Alice","age":30}'}</code><br />
非法数据：<code>{'{"name":"","age":-1,"extra":true}'}</code> — 会产生三条错误。</p>
