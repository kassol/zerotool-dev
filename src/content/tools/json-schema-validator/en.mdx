<h2>How to Use</h2>
<ol>
  <li>Paste your <strong>JSON Schema</strong> on the left and your <strong>JSON Data</strong> on the right.</li>
  <li>Select the schema draft (Draft-07 or Draft 2020-12) from the dropdown.</li>
  <li>Click <strong>Validate</strong> (or press <kbd>Ctrl</kbd>+<kbd>Enter</kbd>).</li>
  <li>If valid, a green message confirms the data matches the schema.</li>
  <li>If invalid, each error is listed with its <strong>JSON path</strong>, error message, and additional parameters.</li>
</ol>

<p>Click <strong>Load Example</strong> to populate both panels with a sample User schema and matching JSON for a quick demo.</p>

<h2>Reading the Error Output</h2>

<p>Each validation error shows three things:</p>
<ul>
  <li><strong>Path</strong> — the JSON Pointer to the failing value (e.g. <code>/items/0/price</code> or <code>(root)</code> for the top level).</li>
  <li><strong>Message</strong> — a plain-English description from AJV (e.g. <em>must be integer</em>, <em>must have required property 'name'</em>).</li>
  <li><strong>Parameters</strong> — additional context such as the allowed types, minimum value, or pattern that was required.</li>
</ul>

<h2>Common JSON Schema Keywords</h2>
<ul>
  <li><code>type</code> — restrict to a JSON type: <code>string</code>, <code>number</code>, <code>integer</code>, <code>boolean</code>, <code>array</code>, <code>object</code>, <code>null</code>.</li>
  <li><code>required</code> — array of property names that must be present in an object.</li>
  <li><code>properties</code> — define constraints for each named property of an object.</li>
  <li><code>additionalProperties: false</code> — reject any property not listed in <code>properties</code>.</li>
  <li><code>minimum</code> / <code>maximum</code> — numeric range constraints.</li>
  <li><code>minLength</code> / <code>maxLength</code> — string length constraints.</li>
  <li><code>pattern</code> — validate a string against a regex (e.g. <code>"^[a-z]+"</code>).</li>
  <li><code>enum</code> — restrict to a fixed set of values.</li>
  <li><code>items</code> — define the schema for array elements.</li>
</ul>

<h2>Example Schema</h2>

```json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "type": "object",
  "required": ["name", "age"],
  "properties": {
    "name":  { "type": "string", "minLength": 1 },
    "age":   { "type": "integer", "minimum": 0 },
    "email": { "type": "string" }
  },
  "additionalProperties": false
}
```

<p>Valid data: <code>{'{"name":"Alice","age":30}'}</code><br />
Invalid data: <code>{'{"name":"","age":-1,"extra":true}'}</code> — would produce three errors.</p>
