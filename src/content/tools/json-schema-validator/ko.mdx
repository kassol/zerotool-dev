<h2>사용 방법</h2>
<ol>
  <li>왼쪽에 <strong>JSON Schema</strong>를, 오른쪽에 <strong>JSON 데이터</strong>를 붙여넣습니다.</li>
  <li>드롭다운에서 스키마 초안(Draft-07 또는 Draft 2020-12)을 선택합니다.</li>
  <li><strong>Validate</strong> 버튼을 클릭합니다 (<kbd>Ctrl</kbd>+<kbd>Enter</kbd> 단축키 사용 가능).</li>
  <li>유효한 경우 데이터가 스키마에 일치한다는 녹색 메시지가 표시됩니다.</li>
  <li>유효하지 않은 경우 각 오류의 <strong>JSON 경로</strong>, 오류 메시지, 추가 파라미터가 나열됩니다.</li>
</ol>

<p><strong>Load Example</strong>을 클릭하면 샘플 User 스키마와 JSON이 자동으로 채워져 바로 사용해볼 수 있습니다.</p>

<h2>오류 출력 이해하기</h2>

<p>각 유효성 검사 오류에는 세 가지 정보가 포함됩니다:</p>
<ul>
  <li><strong>경로</strong> — 실패한 값의 JSON Pointer (예: <code>/items/0/price</code>, 최상위 레벨은 <code>(root)</code>).</li>
  <li><strong>메시지</strong> — AJV의 쉬운 설명 (예: <em>must be integer</em>, <em>must have required property 'name'</em>).</li>
  <li><strong>파라미터</strong> — 허용된 타입, 최솟값, 필요한 패턴 등 추가 컨텍스트.</li>
</ul>

<h2>주요 JSON Schema 키워드</h2>
<ul>
  <li><code>type</code> — JSON 타입 제한: <code>string</code>, <code>number</code>, <code>integer</code>, <code>boolean</code>, <code>array</code>, <code>object</code>, <code>null</code>.</li>
  <li><code>required</code> — 객체에 반드시 있어야 하는 속성 이름 배열.</li>
  <li><code>properties</code> — 객체의 각 명명된 속성에 대한 제약 정의.</li>
  <li><code>additionalProperties: false</code> — <code>properties</code>에 없는 속성 거부.</li>
  <li><code>minimum</code> / <code>maximum</code> — 숫자 범위 제약.</li>
  <li><code>minLength</code> / <code>maxLength</code> — 문자열 길이 제약.</li>
  <li><code>pattern</code> — 정규식으로 문자열 검증 (예: <code>"^[a-z]+"</code>).</li>
  <li><code>enum</code> — 고정된 값 집합으로 제한.</li>
  <li><code>items</code> — 배열 요소의 스키마 정의.</li>
</ul>

<h2>스키마 예시</h2>

```json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "type": "object",
  "required": ["name", "age"],
  "properties": {
    "name":  { "type": "string", "minLength": 1 },
    "age":   { "type": "integer", "minimum": 0 },
    "email": { "type": "string" }
  },
  "additionalProperties": false
}
```

<p>유효한 데이터: <code>{'{"name":"Alice","age":30}'}</code><br />
유효하지 않은 데이터: <code>{'{"name":"","age":-1,"extra":true}'}</code> — 세 개의 오류가 발생합니다.</p>
