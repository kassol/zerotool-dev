<h2>使い方</h2>
<ol>
  <li>ブラウザ DevTools の Network パネルからセッションを <strong>.har</strong> として書き出します。</li>
  <li>ファイルをドロップゾーンにドラッグ（またはクリックして選択）。解析はブラウザ内で完了します。</li>
  <li><strong>概要</strong>（統計）／<strong>ウォーターフォール</strong>（タイミング）／<strong>詳細</strong>（ヘッダー・Cookie・ボディ）／<strong>フィルタとエクスポート</strong>（切り出しダウンロード）を切り替えます。</li>
  <li>ウォーターフォール任意行のクリックで該当リクエストの詳細にジャンプします。</li>
  <li><strong>リセット</strong>でメモリ上の HAR を消去できます。</li>
</ol>

<h2>ウォーターフォールのフェーズ</h2>
<p>ウォーターフォールは HAR 1.2 <code>timings</code> を 6 セグメントで描画します：</p>
<ul>
  <li><strong>キュー / Stalled</strong> — <code>blocked</code></li>
  <li><strong>DNS 解決</strong> — <code>dns</code></li>
  <li><strong>初期接続</strong> — <code>connect</code>（HAR 1.2 では <code>ssl</code> も含まれる）</li>
  <li><strong>送信</strong> — <code>send</code></li>
  <li><strong>待機 (TTFB)</strong> — <code>wait</code></li>
  <li><strong>受信</strong> — <code>receive</code></li>
</ul>

<h2>プライバシー</h2>
<p>HAR には Authorization / Cookie / レスポンスボディなど機微な情報が含まれます。クラウドアナライザへの送信はリスクが高いです。ZeroTool は <code>FileReader</code> + <code>JSON.parse</code> でブラウザ内処理し、アップロードもリロード越しの永続化も行いません。</p>
