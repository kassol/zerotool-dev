The encoding uses Crockford Base32: 0-9 and A-Z excluding I, L, O, U to avoid visual ambiguity."

<h2>Why Sort Order Matters in Databases</h2>
<p>When using random UUIDs as primary keys in B-tree indexes (PostgreSQL, MySQL), insertions become non-sequential, leading to index fragmentation and slower range queries. Using ULIDs as primary keys keeps inserts nearly sequential, improving INSERT and SELECT performance significantly on large tables.</p>
