faqItems:
  - question: "ULID란 무엇인가요?"
    answer: "ULID(Universally Unique Lexicographically Sortable Identifier)는 Crockford Base32의 26자로 표현되는 128비트 식별자입니다. UUID v4와 달리 ULID는 사전순으로 정렬 가능합니다. 앞 10자는 48비트 밀리초 타임스탬프, 나머지 16자는 80비트 난수를 인코딩합니다."
  - question: "ULID의 구조는 어떻게 되나요?"
    answer: "ULID는 26자: 앞 10자는 타임스탬프(48비트, 밀리초 정밀도, 서기 10889년까지 유효), 나머지 16자는 암호학적으로 안전한 난수(80비트). I, L, O, U를 제외한 Crockford Base32(0-9 및 A-Z)로 인코딩됩니다."
  - question: "ULID와 UUID의 차이점은 무엇인가요?"
    answer: "UUID v4는 완전 랜덤으로 시간순 정렬이 불가합니다. ULID는 생성 순서로 사전순 정렬되어 데이터베이스 기본키로 최적입니다. 순차 삽입으로 B-트리 단편화가 줄어 대규모 테이블의 INSERT/SELECT 성능이 향상됩니다. ULID는 URL 안전하고 대소문자를 구분하지 않습니다."
  - question: "ULID 디코더는 무엇에 사용하나요?"
    answer: "기존 ULID를 붙여넣으면 생성 타임스탬프를 추출할 수 있습니다. 별도의 created_at 컬럼 없이 ID에서 직접 레코드 생성 시각을 확인할 수 있어 디버깅에 유용합니다."
  - question: "도구가 클라이언트 사이드에서만 동작하나요?"
    answer: "네. ULID 생성은 브라우저의 Web Crypto API(crypto.getRandomValues)로 난수를 생성하고 Date.now()로 타임스탬프를 얻습니다. 서버로 데이터를 전송하지 않습니다."

<h2>사용 방법</h2>
<ol>
  <li><strong>일괄 수</strong>(1–100)를 설정하여 한 번에 생성할 ULID 수를 결정합니다.</li>
  <li><strong>생성</strong> 버튼을 클릭하여 새 ULID를 만듭니다.</li>
  <li>각 행에 전체 ULID 문자열, 디코딩된 타임스탬프(UTC), 랜덤 컴포넌트가 표시됩니다.</li>
  <li>각 행의 <strong>복사</strong>로 단일 ULID를 복사하거나, <strong>모두 복사</strong>로 전체 ULID를 줄바꿈 구분 목록으로 복사합니다.</li>
  <li>하단의 <strong>ULID 디코더</strong>에 기존 ULID를 붙여넣어 타임스탬프를 추출합니다.</li>
</ol>

<h2>ULID 구조</h2>

```
01ARZ3NDEKTSV4RRFFQ69G5FAV
├─ 타임스탬프 (10자) ──┤├── 랜덤 컴포넌트 (16자) ──┤
 48비트 밀리초 타임스탬프    80비트 암호학적 난수
```

<h2>ULID vs UUID 비교</h2>

| 특성 | ULID | UUID v4 |
|---|---|---|
| 길이 | 26자 | 36자 (대시 포함) |
| 정렬 가능 | ✓ 예 | ✗ 아니오 |
| 타임스탬프 내장 | ✓ 예 | ✗ 아니오 |
| URL 안전 | ✓ 예 | ✓ 예 |
| 대소문자 구분 없음 | ✓ 예 | ✓ 예 |
| 랜덤 비트 | 80비트 | 122비트 |
