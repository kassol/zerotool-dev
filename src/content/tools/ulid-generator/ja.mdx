  - question: "ULIDとは何ですか？"
    answer: "ULID（Universally Unique Lexicographically Sortable Identifier）は、Crockford Base32の26文字で表現される128ビットの識別子です。UUID v4と異なり、ULIDは辞書順でソート可能です。先頭10文字が48ビットのミリ秒タイムスタンプ、残り16文字が80ビットの乱数です。"
  - question: "ULIDの構造を教えてください。"
    answer: "ULIDは26文字：先頭10文字がタイムスタンプ（48ビット、ミリ秒精度、西暦10889年まで有効）、残り16文字が暗号学的に安全な乱数（80ビット）。Crockford Base32（0-9とA-Z、I・L・O・Uを除く）でエンコードされます。"
  - question: "ULIDとUUIDの違いは何ですか？"
    answer: "UUID v4は完全にランダムで時系列ソートができません。ULIDは生成順に辞書順ソートされるため、データベースの主キーとして最適です。順次挿入によりB木の断片化が減少し、大規模テーブルのINSERT・SELECT性能が向上します。ULIDはURL安全で大文字小文字を区別しません。"

<h2>使い方</h2>
<ol>
  <li><strong>バッチ数</strong>（1–100）を設定して、一度に生成するULIDの数を決めます。</li>
  <li><strong>生成</strong>ボタンをクリックして新しいULIDを作成します。</li>
  <li>各行にULIDの全文字列、デコードされたタイムスタンプ（UTC）、ランダム部分が表示されます。</li>
  <li>各行の<strong>コピー</strong>で単一ULIDをコピー、<strong>すべてコピー</strong>で全ULIDを改行区切りでコピーします。</li>
  <li>下部の<strong>ULIDデコーダー</strong>に既存のULIDを貼り付けてタイムスタンプを抽出できます。</li>
</ol>

<h2>ULID 構造</h2>

```
01ARZ3NDEKTSV4RRFFQ69G5FAV
├─ タイムスタンプ（10文字）─┤├── ランダム部分（16文字）──┤
 48ビット ミリ秒タイムスタンプ   80ビット 暗号学的乱数
```

<h2>ULID vs UUID 比較</h2>

| 特徴 | ULID | UUID v4 |
|---|---|---|
| 長さ | 26文字 | 36文字（ハイフン含む）|
| ソート可能 | ✓ はい | ✗ いいえ |
| タイムスタンプ内蔵 | ✓ はい | ✗ いいえ |
| URL安全 | ✓ はい | ✓ はい |
| 大文字小文字不区別 | ✓ はい | ✓ はい |
| ランダムビット数 | 80ビット | 122ビット |
