<h2>なぜ今もマルチファイルのファビコンセットが必要か</h2>

<p>ファビコンはタブのアイコンだけではありません。検索結果のドメイン横、ブラウザのブックマーク、iOS ユーザーが「ホーム画面に追加」したときのアイコン、Android の PWA インストールカードまで、登場場面ごとに必要なサイズが異なり、フォーマットも微妙に違います。<code>favicon.ico</code> 一枚だけ置いて済ませると、Retina ディスプレイやインストール可能 PWA で目に見える粗が残ります。</p>

<p>このツールはモダンなセットを一括生成します — レガシー互換のための ICO、現代の <code>&lt;link rel="icon"&gt;</code> スタックに合わせた複数サイズ PNG、iOS 向けの apple-touch-icon、そして maskable な Android アイコンを宣言する webmanifest です。</p>

<h2>使い方</h2>

<ol>
  <li>素材を選びます。4 つのタブはそれぞれ、絵文字（プリセットまたは貼り付け）、1〜4 文字のテキスト、アップロードした PNG/JPG/SVG（10 MB まで）、貼り付けた SVG マークアップを受け付けます。</li>
  <li>背景と形状を設定します。透過はマスク前提のデザイン向け、塗りつぶし色はブランドマーク向け。形状は正方形・角丸（12.5% 半径）・円形の 3 種。Padding スライダーが内容物をセーフエリアの内側に収めます。</li>
  <li>ライブプレビューを確認します。重要な 4 サイズ（16・32・64・180 px）が同期描画されます。16 と 32 px はタブバーでの可読性、180 px は iOS のホーム画面アイコンを示します。</li>
  <li><strong>Generate package</strong> をクリックします。ブラウザが約 1 秒で 9 ファイルを生成。<strong>Download ZIP</strong> で保存、<strong>Copy HTML snippet</strong> で <code>&lt;head&gt;</code> に貼る <code>&lt;link&gt;</code> タグをコピーできます。</li>
</ol>

<h2>HTML スニペットの中身</h2>

<p>生成されるスニペットは約 7 行で、現代のあらゆる場面をカバーします：</p>

<ul>
  <li><code>&lt;link rel="icon" type="image/x-icon" href="/favicon.ico"&gt;</code> — 古いブラウザや favicon.ico を優先するブックマーク向けのフォールバック。</li>
  <li><code>&lt;link rel="icon" type="image/png" sizes="..."&gt;</code> 16/32/48 — モダンブラウザは sizes が一致する PNG を優先選択し、ICO をスキップします。</li>
  <li><code>&lt;link rel="apple-touch-icon"&gt;</code> 180×180 — iOS Safari が「ホーム画面に追加」時に使用。</li>
  <li><code>&lt;link rel="manifest"&gt;</code> — Android Chrome と PWA インストーラーが site.webmanifest を読み、その中で <code>purpose: "any maskable"</code> として 192/512 PNG を宣言します。</li>
  <li><code>&lt;meta name="theme-color"&gt;</code> — モバイルのアドレスバーと、インストール済み PWA のタイトルバーの色を制御します。</li>
</ul>

<h2>パッケージの作り方</h2>

<p>レンダラーはターゲットサイズごとに新しい canvas を作り、選択した背景と形状マスクを適用し、padding 分だけ内側に素材を中央配置で描画します。続いて <code>canvas.toBlob('image/png')</code> でコンパクトな PNG データを取得します。16/32/48 の 3 つの PNG は PNG 圧縮エントリ形式で 1 つの favicon.ico に統合します — ICONDIR ヘッダと 16 バイトの ICONDIRENTRY 3 件の後ろに 3 つの PNG ペイロードを連結する形です。</p>

<p>ZIP ライターは STORED 方式（DEFLATE しない）です：PNG はすでに圧縮済みで再 DEFLATE のメリットは 1% 未満、コードは 5 KB 増えます。ファイル名は General Purpose Bit Field の bit 11 で UTF-8 を宣言するため、将来名前を変更しても非 ASCII 名が壊れません。</p>

<h2>プライバシーとオフライン</h2>

<ul>
  <li>ファイルはアップロードされません。ブラウザのネットワークパネルで Generate をクリックしてみてください。リクエストは一切発生しません。</li>
  <li>ページをキャッシュ後はオフラインでも動作します。Canvas API・ZIP・ICO ライターはすべてページバンドル内です。</li>
  <li>トラッキングピクセルなし。利用統計のため generate と download のイベントを 1 件ずつ記録しますが、ペイロードはツール名のみです。</li>
</ul>

<h2>つまずきやすい点</h2>

<ul>
  <li><strong>外部画像参照のある SVG。</strong> SVG が <code>&lt;image href="https://..."&gt;</code> を含むと、ブラウザは canvas を blob として読み戻すのを拒否します（クロスオリジン汚染）。貼り付け前に外部画像を Data URI にインライン化してください。</li>
  <li><strong>企業管理 Linux で絵文字が空枠になる。</strong> 一部の IT 管理 Linux ディストロにはカラー絵文字フォントが入っていません。プレビューが空枠になる場合は Text または Image タブに切り替えてください。</li>
  <li><strong>16 px で細部が消える。</strong> 1.5 px 未満の線は 16×16 で消失します。32 で鮮明だが 16 で消えるなら、デザインを単純化するか、背景色でコントラストを補ってください。</li>
</ul>
