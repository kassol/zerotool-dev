<h2>Why a multi-file favicon set still matters</h2>

<p>A site's favicon is more than a tab icon. It shows up in search results next to your domain, in browser bookmarks, on the home screen when an iOS user adds your site to the dock, and inside Android's PWA install card. Each of those surfaces wants a different size, and a few of them want a slightly different format. Shipping a single <code>favicon.ico</code> and hoping for the best leaves visible rough edges on retina screens and on installable PWAs.</p>

<p>This tool produces the full modern set in one click — ICO for legacy compatibility, multiple PNG sizes for the contemporary <code>&lt;link rel="icon"&gt;</code> stack, an apple-touch-icon for iOS, and a webmanifest with maskable Android icons.</p>

<h2>How to use</h2>

<ol>
  <li>Pick a source. The four tabs accept an emoji (preset or pasted), 1–4 typeset characters, an uploaded PNG/JPG/SVG image up to 10 MB, or pasted SVG markup.</li>
  <li>Choose a background and shape. Transparent for masked designs, a solid color for branded marks. Square, rounded (12.5 % corner radius), or circle. The padding slider keeps your content out of the safe-area edges.</li>
  <li>Watch the live preview render at the four crucial sizes — 16, 32, 64, and 180 pixels. The 16 and 32 px tiles tell you whether your mark survives the tab bar; the 180 px tile is the iOS home-screen icon.</li>
  <li>Click <strong>Generate package</strong>. The browser produces eleven files in roughly one second. <strong>Download ZIP</strong> saves them; <strong>Copy HTML snippet</strong> gives you the <code>&lt;link&gt;</code> tags to paste into your <code>&lt;head&gt;</code>.</li>
</ol>

<h2>The HTML snippet, explained</h2>

<p>The generated snippet covers every modern surface in roughly seven lines:</p>

<ul>
  <li><code>&lt;link rel="icon" type="image/x-icon" href="/favicon.ico"&gt;</code> — fallback for older browsers and bookmarks that still parse <code>favicon.ico</code> first.</li>
  <li><code>&lt;link rel="icon" type="image/png" sizes="..."&gt;</code> at 16, 32, and 48 — modern browsers prefer the PNG with a matching <code>sizes</code> attribute and skip the ICO.</li>
  <li><code>&lt;link rel="apple-touch-icon"&gt;</code> at 180×180 — Safari on iOS uses this when a user adds the page to the home screen.</li>
  <li><code>&lt;link rel="manifest"&gt;</code> — points Android Chrome and PWA installers at <code>site.webmanifest</code>, which advertises the 192 and 512 PNGs with <code>purpose: "any maskable"</code>.</li>
  <li><code>&lt;meta name="theme-color"&gt;</code> — colors the address bar on mobile and the title bar on installed PWAs.</li>
</ul>

<h2>How the package is built</h2>

<p>The renderer creates a fresh canvas at each target size, applies the chosen background and shape mask, draws the source content centered with the padding offset, and asks <code>canvas.toBlob('image/png')</code> for a compact PNG. The 16, 32, and 48 PNGs are then packed into a single <code>favicon.ico</code> using the modern PNG-compressed entry format — three PNG payloads concatenated behind an <code>ICONDIR</code> header and three 16-byte <code>ICONDIRENTRY</code> records.</p>

<p>The ZIP packer writes everything with the STORED method (no DEFLATE) because PNG is already compressed and re-deflating the bytes saves less than 1 % at the cost of an extra 5 KB of code. Filenames are flagged as UTF-8 in the general purpose bit field so non-ASCII names — should you ever rename the package — survive cleanly.</p>

<h2>Common pitfalls</h2>

<ul>
  <li><strong>SVG with external images.</strong> When an SVG references <code>&lt;image href="https://..."&gt;</code> the browser refuses to read the canvas back as a blob (cross-origin taint). Inline the image as a Data URI before pasting.</li>
  <li><strong>Empty emoji on a corporate Linux box.</strong> Some IT-managed Linux distros ship without a color emoji font. The preview will show empty boxes; switch to the Text or Image tab to bypass the issue.</li>
  <li><strong>Tiny details on the 16 px tile.</strong> Lines thinner than 1.5 px disappear at 16×16. If your mark looks crisp at 32 but vanishes at 16, simplify the design or lean on the background color for separation.</li>
</ul>
