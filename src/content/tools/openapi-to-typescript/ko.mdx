<h2>사용 방법</h2>
<ol>
  <li>OpenAPI 3.0.x 또는 3.1.x 문서(YAML 또는 JSON)를 왼쪽 패널에 붙여 넣습니다.</li>
  <li>입력하면 300 ms 디바운스로 실시간 생성됩니다.</li>
  <li>선택적으로 <strong>루트 네임스페이스</strong> 변경, 모든 속성을 optional 처리, path 오퍼레이션 타입 포함, Zod 스키마 동시 출력을 선택할 수 있습니다.</li>
  <li><strong>복사</strong> 버튼으로 전체 TypeScript 출력을 가져갑니다.</li>
</ol>

<h2>생성되는 내용</h2>
<ul>
  <li><strong>components/schemas</strong>: 각 스키마가 `export interface` 또는 `export type`이 됩니다. 객체는 interface, 기본형 / 유니언 / enum / 교차 타입은 type alias로 출력됩니다.</li>
  <li><strong>$ref</strong>: `#/components/schemas/...` 참조는 타입 이름으로 해석되며, 스펙의 선언 순서를 그대로 따릅니다.</li>
  <li><strong>oneOf / anyOf</strong>: 유니언 타입 `A | B | C`. <strong>allOf</strong>는 교차 타입 `A & B`.</li>
  <li><strong>enum</strong>: 문자열 또는 숫자 리터럴 유니언.</li>
  <li><strong>nullable</strong>: 3.0의 `nullable: true`와 3.1의 `type: ['X', 'null']` 모두 `X | null`로 변환.</li>
  <li><strong>format 힌트</strong>: `format: date-time`에는 ISO 8601 JSDoc 주석을 추가하고, `format: binary`는 `Blob`이 됩니다.</li>
  <li><strong>additionalProperties</strong>: 인덱스 시그니처 또는 `Record&lt;string, T&gt;`.</li>
</ul>

<h2>Path 타입(선택)</h2>
<p><strong>path 타입 포함</strong>을 켜면 각 오퍼레이션마다 타입 엔벨로프가 생성됩니다:</p>
<pre><code>{`export namespace Components {
  export namespace GetPetById {
    export interface PathParameters {
      id: number;
    }
    export type Response200 = Pet;
  }
}`}</code></pre>
<p>오퍼레이션에 <code>operationId</code>가 없으면 method와 path로 합성합니다(예: <code>GetPetsId</code>).</p>

<h2>Zod 스키마(선택)</h2>
<p><strong>Zod 스키마 포함</strong>을 켜면 각 컴포넌트별로 `XSchema`를 함께 출력합니다:</p>
<pre><code>{`import { z } from 'zod';

export const PetSchema = z.object({
  id: z.number().int(),
  name: z.string(),
  status: z.enum(["available", "pending", "sold"]),
});`}</code></pre>
<p>OpenAPI 스펙이 아니라 손으로 쓴 TS 타입에서 Zod를 만들고 싶다면 <a href="/ko/tools/typescript-to-zod/">TypeScript → Zod</a>를 사용하세요.</p>
