<h2>使用方法</h2>
<ol>
  <li>将 OpenAPI 3.0.x 或 3.1.x 文档（YAML 或 JSON）粘贴到左侧面板。</li>
  <li>输入后会以 300 ms 防抖实时生成类型。</li>
  <li>可选：调整 <strong>根命名空间</strong>、把所有属性标记为可选、附加 path 操作类型、或同时输出 Zod schema。</li>
  <li>点击 <strong>复制</strong> 即可拿到完整的 TypeScript 输出。</li>
</ol>

<h2>生成内容</h2>
<ul>
  <li><strong>components/schemas</strong>：每个 schema 输出一个 `export interface` 或 `export type`，对象类型用 interface，基本/联合/枚举/交叉类型用 type alias。</li>
  <li><strong>$ref</strong>：`#/components/schemas/...` 引用会被解析为对应类型名，并按规范声明顺序输出。</li>
  <li><strong>oneOf / anyOf</strong>：转为联合类型 `A | B | C`；<strong>allOf</strong> 转为交叉类型 `A & B`。</li>
  <li><strong>enum</strong>：转为字符串或数字字面量的联合类型。</li>
  <li><strong>nullable</strong>：3.0 的 `nullable: true` 与 3.1 的 `type: ['X', 'null']` 都转为 `X | null`。</li>
  <li><strong>format 提示</strong>：`format: date-time` 会附带 ISO 8601 的 JSDoc 注释；`format: binary` 转为 `Blob`。</li>
  <li><strong>additionalProperties</strong>：转为索引签名或 `Record&lt;string, T&gt;`。</li>
</ul>

<h2>Path 类型（可选）</h2>
<p>勾选 <strong>包含 path 类型</strong> 后，每个操作都会产出一个类型封装：</p>
<pre><code>{`export namespace Components {
  export namespace GetPetById {
    export interface PathParameters {
      id: number;
    }
    export type Response200 = Pet;
  }
}`}</code></pre>
<p>如果操作没有 <code>operationId</code>，工具会按 method + path 合成一个，例如 <code>GetPetsId</code>。</p>

<h2>Zod schema（可选）</h2>
<p>勾选 <strong>包含 Zod schema</strong>，会为每个 component 输出同名 `XSchema`：</p>
<pre><code>{`import { z } from 'zod';

export const PetSchema = z.object({
  id: z.number().int(),
  name: z.string(),
  status: z.enum(["available", "pending", "sold"]),
});`}</code></pre>
<p>若你的项目是从手写 TS 类型反推 Zod，可改用 <a href="/zh/tools/typescript-to-zod/">TypeScript 转 Zod</a>。</p>
