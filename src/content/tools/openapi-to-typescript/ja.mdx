<h2>使い方</h2>
<ol>
  <li>OpenAPI 3.0.x または 3.1.x のドキュメント（YAML / JSON）を左側パネルに貼り付けます。</li>
  <li>入力中は 300 ms デバウンスでリアルタイム生成されます。</li>
  <li>必要に応じて <strong>ルート名前空間</strong> の変更、全プロパティを optional 化、path 型の追加、Zod スキーマの同時出力を切り替えられます。</li>
  <li><strong>コピー</strong> ボタンで TypeScript 出力全文をコピーします。</li>
</ol>

<h2>生成される内容</h2>
<ul>
  <li><strong>components/schemas</strong>：各スキーマが `export interface` または `export type` になります。オブジェクトは interface、プリミティブ / union / enum / intersection は type alias として出力されます。</li>
  <li><strong>$ref</strong>：`#/components/schemas/...` への参照は型名で解決され、仕様の宣言順を維持します。</li>
  <li><strong>oneOf / anyOf</strong>：union 型 `A | B | C`。<strong>allOf</strong> は intersection `A & B` になります。</li>
  <li><strong>enum</strong>：文字列または数値のリテラル union。</li>
  <li><strong>nullable</strong>：3.0 の `nullable: true` も 3.1 の `type: ['X', 'null']` も `X | null` に変換。</li>
  <li><strong>format ヒント</strong>：`format: date-time` には ISO 8601 の JSDoc を付与し、`format: binary` は `Blob` になります。</li>
  <li><strong>additionalProperties</strong>：インデックスシグネチャまたは `Record&lt;string, T&gt;`。</li>
</ul>

<h2>Path 型（任意）</h2>
<p><strong>path 型を含める</strong> を有効にすると、各オペレーションに型エンベロープが生成されます：</p>
<pre><code>{`export namespace Components {
  export namespace GetPetById {
    export interface PathParameters {
      id: number;
    }
    export type Response200 = Pet;
  }
}`}</code></pre>
<p><code>operationId</code> がない場合は method と path から合成します（例：<code>GetPetsId</code>）。</p>

<h2>Zod スキーマ（任意）</h2>
<p><strong>Zod スキーマを含める</strong> を有効にすると、各コンポーネントに対して `XSchema` を並行して出力します：</p>
<pre><code>{`import { z } from 'zod';

export const PetSchema = z.object({
  id: z.number().int(),
  name: z.string(),
  status: z.enum(["available", "pending", "sold"]),
});`}</code></pre>
<p>OpenAPI 仕様ではなく既存の手書き TS 型から Zod を作りたい場合は <a href="/ja/tools/typescript-to-zod/">TypeScript → Zod</a> を使ってください。</p>
