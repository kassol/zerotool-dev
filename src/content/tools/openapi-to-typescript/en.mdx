<h2>How to Use</h2>
<ol>
  <li>Paste an OpenAPI 3.0.x or 3.1.x document (YAML or JSON) into the left panel.</li>
  <li>Generation runs in real time as you type, with a 300 ms debounce.</li>
  <li>Optionally tweak the <strong>root namespace</strong>, mark every property optional, include path operation types, or emit matching Zod schemas.</li>
  <li>Click <strong>Copy</strong> to grab the full TypeScript output.</li>
</ol>

<h2>What gets generated</h2>
<ul>
  <li><strong>components/schemas</strong> become one `export interface` or `export type` per schema. Object schemas become interfaces; primitive, union, enum, and intersection schemas become type aliases.</li>
  <li><strong>$ref</strong> targets inside `#/components/schemas/...` are resolved and emitted by name, preserving the declaration order from your spec.</li>
  <li><strong>oneOf</strong> / <strong>anyOf</strong> become union types (`A | B | C`). <strong>allOf</strong> becomes an intersection (`A & B`).</li>
  <li><strong>enum</strong> becomes a string or numeric literal union.</li>
  <li><strong>nullable</strong> in 3.0 and `type: ['X', 'null']` in 3.1 both become `X | null`.</li>
  <li><strong>format hints</strong> add JSDoc comments — `format: date-time` notes ISO 8601, `format: binary` becomes `Blob`.</li>
  <li><strong>additionalProperties</strong> become index signatures or `Record&lt;string, T&gt;`.</li>
</ul>

<h2>Path types (optional)</h2>
<p>With <strong>Include path types</strong> enabled, each operation produces a typed envelope:</p>
<pre><code>{`export namespace Components {
  export namespace GetPetById {
    export interface PathParameters {
      id: number;
    }
    export type Response200 = Pet;
  }
}`}</code></pre>
<p>If an operation has no <code>operationId</code>, the tool synthesises one from method + path, e.g. <code>GetPetsId</code>.</p>

<h2>Zod schemas (optional)</h2>
<p>Toggle <strong>Include Zod schemas</strong> to emit a parallel `XSchema` for every component:</p>
<pre><code>{`import { z } from 'zod';

export const PetSchema = z.object({
  id: z.number().int(),
  name: z.string(),
  status: z.enum(["available", "pending", "sold"]),
});`}</code></pre>
<p>Pair this output with <a href="/tools/typescript-to-zod/">TypeScript to Zod</a> when you want to keep schemas in sync with hand-written types instead of an OpenAPI spec.</p>
