<h2>How to Use</h2>
<ol>
  <li>Paste your X.509 certificate (PEM format) into the input box.</li>
  <li>Click <strong>Decode</strong>. The full set of certificate fields appears below.</li>
  <li>Review <strong>Validity</strong> for expiry status — expired, valid, or expiring within 30 days.</li>
  <li>Inspect <strong>Subject</strong>, <strong>Issuer</strong>, and <strong>Subject Alternative Names</strong> to confirm hostnames.</li>
  <li>Copy any <strong>fingerprint</strong> to clipboard for pinning, troubleshooting, or audit logs.</li>
</ol>

<h2>What this decoder shows</h2>
<ul>
  <li><strong>Validity period</strong> — Not Before / Not After timestamps plus a colour-coded status badge.</li>
  <li><strong>Subject and Issuer</strong> — CN, O, OU, C, L, ST, plus any extension fields (jurisdiction, business category, serial number).</li>
  <li><strong>Subject Alternative Names</strong> — DNS names, IP addresses, URIs, and email entries, grouped by type.</li>
  <li><strong>Public key</strong> — algorithm (RSA, EC, Ed25519, Ed448), key size in bits, RSA exponent, EC named curve.</li>
  <li><strong>Key Usage and Extended Key Usage</strong> — digital signature, key encipherment, server / client authentication, code signing, OCSP signing, and more.</li>
  <li><strong>Basic Constraints</strong> — whether the certificate is a CA, and its path length constraint.</li>
  <li><strong>Signature algorithm</strong> — sha256WithRSAEncryption, ecdsa-with-SHA384, Ed25519, etc.</li>
  <li><strong>Fingerprints</strong> — SHA-256, SHA-1, MD5 hex digests with colon separators.</li>
  <li><strong>Serial number and version</strong> — hex serial and X.509 version (typically v3).</li>
</ul>

<h2>How a PEM certificate decodes</h2>
<p>
  A PEM certificate is a Base64 wrapper around DER-encoded ASN.1. Once stripped and
  decoded, you get a <code>Certificate</code> SEQUENCE containing <code>tbsCertificate</code>,
  <code>signatureAlgorithm</code>, and <code>signatureValue</code>. The TBS half holds every
  field above. This tool walks the ASN.1 tree directly in JavaScript — no network round-trip,
  no server-side parser.
</p>

<h2>Typical use cases</h2>
<ul>
  <li>Verifying SAN coverage before deploying a new TLS certificate.</li>
  <li>Spot-checking expiry across staging and production endpoints.</li>
  <li>Pinning by SHA-256 fingerprint in mobile apps or HSTS-preload submissions.</li>
  <li>Confirming the signature algorithm and key strength meet a compliance policy (PCI DSS, FIPS 140-3, Mozilla Modern profile).</li>
  <li>Debugging mismatched <code>Subject</code> CN versus modern SAN-only validators.</li>
</ul>

<h2>Limits by design</h2>
<p>
  This tool decodes a single certificate. It does not validate the chain, fetch CRL/OCSP,
  perform revocation checks, or speak TLS. For chain validation use
  <code>openssl verify</code>; for live TLS inspection use <code>openssl s_client -showcerts</code>
  or <code>nmap --script ssl-cert</code>.
</p>
