faqItems:
  - question: "什么是 Zod？"
    answer: "Zod 是一个 TypeScript 优先的 schema 声明和验证库，让你定义数据结构并在运行时进行验证。广泛应用于 API 响应验证、表单验证和配置解析。"
  - question: "生成器如何推断类型？"
    answer: "生成器检查每个 JSON 值：字符串 → z.string()，布尔 → z.boolean()，整数 → z.number().int()，浮点 → z.number()，null → z.null()，数组 → z.array()，对象 → z.object()。对于对象数组，只在部分 item 中出现的 key 会被标记为 optional。"
  - question: "什么是严格模式？"
    answer: "启用严格模式后，每个 z.object() 后会追加 .strict()，Zod 会拒绝包含 schema 中未定义的多余字段的输入对象。不启用时，多余字段会被静默忽略（Zod 默认行为）。"
  - question: "生成的 schema 可以直接用于生产吗？"
    answer: "生成的 schema 是良好的起点，建议进一步审查和完善，例如为字符串和数字添加 .min()/.max() 约束，将 nullable 字段改为 .nullable().optional()，或将嵌套 schema 拆分为独立的命名常量。"
  - question: "我的 JSON 数据会发送到服务器吗？"
    answer: "不会。所有处理均在浏览器中完成，JSON 数据不会离开你的设备。"

<h2>使用方法</h2>
<ol>
  <li>将 <strong>JSON 数据</strong>粘贴到左侧面板（或点击<strong>示例</strong>）。</li>
  <li>设置<strong>根 Schema 名称</strong>（用于导出常量和类型）。</li>
  <li>可选：启用<strong>严格模式</strong>以禁止多余字段。</li>
  <li>点击<strong>生成 Zod Schema</strong>。</li>
  <li>点击<strong>复制</strong>将结果粘贴到你的项目中。</li>
</ol>

<h2>输出格式</h2>
<p>生成器输出完整的 TypeScript 代码片段，包括：</p>
<ul>
  <li><code>import &#123; z &#125; from "zod"</code> — Zod 导入</li>
  <li><code>const FooSchema = z.object(&#123;...&#125;)</code> — Schema 定义</li>
  <li><code>export type Foo = z.infer&lt;typeof FooSchema&gt;</code> — 推断的 TypeScript 类型</li>
</ul>

<h2>常见使用场景</h2>
<ul>
  <li><strong>API 验证：</strong>从 API 响应示例生成 schema，在运行时验证真实响应。</li>
  <li><strong>表单验证：</strong>将表单预期的数据结构转换为 Zod schema，用于 react-hook-form 等库。</li>
  <li><strong>配置文件：</strong>定义和验证从 JSON 文件加载的配置对象。</li>
  <li><strong>类型安全：</strong>与 TypeScript 结合，从单一 schema 同时获得运行时验证和静态类型推断。</li>
</ul>
