  - question: "Zodとは何ですか？"
    answer: "ZodはTypeScriptファーストのスキーマ宣言・バリデーションライブラリです。データの形状をスキーマで定義し、実行時にデータを検証できます。APIレスポンス検証・フォームバリデーション・設定ファイル解析に広く使われています。"
  - question: "型の推論はどう行われますか？"
    answer: "各JSON値を検査します：文字列→z.string()、真偽値→z.boolean()、整数→z.number().int()、浮動小数点→z.number()、null→z.null()、配列→z.array()、オブジェクト→z.object()。オブジェクトの配列では、一部のアイテムにしか存在しないキーはoptionalとしてマークされます。"
  - question: "ストリクトモードとは何ですか？"
    answer: "ストリクトモードを有効にすると、すべてのz.object()に.strict()が追加されます。スキーマに定義されていない余分なキーを含む入力オブジェクトはZodに拒否されます。無効時は余分なキーが無視されます（Zodのデフォルト動作）。"
  - question: "生成されたスキーマはそのまま使えますか？"

<h2>使い方</h2>
<ol>
  <li>左パネルに <strong>JSONデータ</strong> を貼り付けます（または<strong>サンプル</strong>をクリック）。</li>
  <li><strong>ルートスキーマ名</strong>を設定します（エクスポート定数と型名に使用）。</li>
  <li>必要に応じて<strong>ストリクトモード</strong>を有効にします。</li>
  <li><strong>Zodスキーマを生成</strong>をクリックします。</li>
  <li><strong>コピー</strong>をクリックしてプロジェクトに貼り付けます。</li>
</ol>

<h2>出力形式</h2>
<p>完全なTypeScriptスニペットを出力します：</p>
<ul>
  <li><code>import &#123; z &#125; from "zod"</code> — Zodインポート</li>
  <li><code>const FooSchema = z.object(&#123;...&#125;)</code> — スキーマ定義</li>
  <li><code>export type Foo = z.infer&lt;typeof FooSchema&gt;</code> — 推論されたTypeScript型</li>
</ul>

<h2>主な用途</h2>
<ul>
  <li><strong>APIバリデーション：</strong>APIレスポンス例からスキーマを生成し、実際のレスポンスを実行時に検証。</li>
  <li><strong>フォームバリデーション：</strong>フォームのデータ形状をZodスキーマに変換してreact-hook-formなどで活用。</li>
  <li><strong>設定ファイル：</strong>JSONファイルから読み込む設定オブジェクトの定義と検証。</li>
  <li><strong>型安全性：</strong>TypeScriptと組み合わせて1つのスキーマから実行時検証と静的型推論を同時に取得。</li>
</ul>
