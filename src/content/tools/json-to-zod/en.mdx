faqItems:
  - question: "What is Zod?"
    answer: "Zod is a TypeScript-first schema declaration and validation library. It lets you define the shape of your data with a schema, then validate data against that schema at runtime. It's widely used for API response validation, form validation, and configuration parsing."
  - question: "How does the generator infer types?"
    answer: "The generator inspects each JSON value: strings become z.string(), booleans become z.boolean(), integers become z.number().int(), floats become z.number(), null becomes z.null(), arrays become z.array(), and objects become z.object(). For arrays of objects, keys present in only some items are marked as optional."
  - question: "What is strict mode?"
    answer: "In strict mode, .strict() is appended to every z.object() call. This causes Zod to reject input objects that contain keys not defined in the schema. Without strict mode, extra keys are silently ignored (the default Zod behavior)."
  - question: "Does this generate production-ready schemas?"
    answer: "The generated schemas are a strong starting point. You should review and refine them — for example, adding .min()/.max() constraints on strings and numbers, converting nullable fields to .nullable().optional(), or splitting nested schemas into separate named constants."
  - question: "Is my JSON data sent anywhere?"
    answer: "No. All processing happens entirely in your browser. Your JSON data never leaves your device."

<h2>How to Use</h2>
<ol>
  <li>Paste your <strong>JSON data</strong> into the left panel (or click <strong>Example</strong>).</li>
  <li>Set the <strong>root schema name</strong> (used for the exported constant and type).</li>
  <li>Optionally enable <strong>strict mode</strong> to disallow extra keys.</li>
  <li>Click <strong>Generate Zod Schema</strong>.</li>
  <li>Click <strong>Copy</strong> to copy the result into your project.</li>
</ol>

<h2>Output Format</h2>
<p>The generator outputs a complete TypeScript snippet including:</p>
<ul>
  <li><code>import &#123; z &#125; from "zod"</code> — Zod import</li>
  <li><code>const FooSchema = z.object(&#123;...&#125;)</code> — Schema definition</li>
  <li><code>export type Foo = z.infer&lt;typeof FooSchema&gt;</code> — Inferred TypeScript type</li>
</ul>

<h2>Common Use Cases</h2>
<ul>
  <li><strong>API validation:</strong> Generate schemas from API response examples to validate real responses at runtime.</li>
  <li><strong>Form validation:</strong> Convert a form's expected data shape into Zod schemas for use with react-hook-form or similar.</li>
  <li><strong>Config files:</strong> Define and validate configuration objects loaded from JSON files.</li>
  <li><strong>Type safety:</strong> Pair with TypeScript to get both runtime validation and static type inference from a single schema.</li>
</ul>
