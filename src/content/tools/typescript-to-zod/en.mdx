<h2>How to Use</h2>
<ol>
  <li>Paste your TypeScript <code>interface</code> or <code>type</code> declarations into the left panel.</li>
  <li>The tool validates and converts in real time (300 ms debounce after typing stops).</li>
  <li>Click <strong>Generate Zod Schema</strong> to force a conversion.</li>
  <li>Click <strong>Copy</strong> to copy the Zod output to your clipboard.</li>
  <li>Paste the result into your project — the output already includes the <code>import</code> statement and <code>export type</code> inference lines.</li>
</ol>

<h2>Supported TypeScript Features</h2>
<ul>
  <li><strong>Basic types</strong>: <code>string</code>, <code>number</code>, <code>boolean</code>, <code>null</code>, <code>undefined</code>, <code>any</code>, <code>unknown</code>, <code>void</code>, <code>never</code>, <code>bigint</code>, <code>symbol</code></li>
  <li><strong>Optional properties</strong>: <code>prop?: type</code> → <code>.optional()</code></li>
  <li><strong>Union types</strong>: <code>A | B</code> → <code>z.union([A, B])</code></li>
  <li><strong>String enums</strong>: <code>"a" | "b" | "c"</code> → <code>z.enum(["a", "b", "c"])</code></li>
  <li><strong>Arrays</strong>: <code>T[]</code> / <code>Array&lt;T&gt;</code> → <code>z.array(T)</code></li>
  <li><strong>Nested objects</strong>: inline object types → <code>z.object(&#123;&#125;)</code></li>
  <li><strong>Cross-references</strong>: named types reference each other → Schema constants</li>
  <li><strong>Readonly</strong>: <code>readonly</code> modifier is accepted and ignored (Zod schemas are immutable by default)</li>
  <li><strong>Utility types</strong>: <code>Partial&lt;T&gt;</code>, <code>Record&lt;K, V&gt;</code>, <code>Promise&lt;T&gt;</code></li>
  <li><strong>Intersection types</strong>: <code>A &amp; B</code> → <code>z.intersection(A, B)</code></li>
  <li><strong>Tuples</strong>: <code>[A, B]</code> → <code>z.tuple([A, B])</code></li>
  <li><strong>Literal types</strong>: <code>"value"</code>, <code>42</code>, <code>true</code> → <code>z.literal()</code></li>
</ul>

<h2>Example</h2>
<p>Given this TypeScript input:</p>
<pre><code>{`interface Address {
  street: string;
  city: string;
  zipCode?: string;
}

export interface User {
  id: number;
  name: string;
  role: "admin" | "user" | "guest";
  age?: number;
  isActive: boolean;
  tags: string[];
  address: Address;
}

export type Status = "active" | "inactive" | "pending";`}</code></pre>
<p>The tool generates:</p>
<pre><code>{`import { z } from "zod";

export const AddressSchema = z.object({
  street: z.string(),
  city: z.string(),
  zipCode: z.string().optional(),
});
export type Address = z.infer<typeof AddressSchema>;

export const UserSchema = z.object({
  id: z.number(),
  name: z.string(),
  role: z.enum(["admin", "user", "guest"]),
  age: z.number().optional(),
  isActive: z.boolean(),
  tags: z.array(z.string()),
  address: AddressSchema,
});
export type User = z.infer<typeof UserSchema>;

export const StatusSchema = z.enum(["active", "inactive", "pending"]);
export type Status = z.infer<typeof StatusSchema>;`}</code></pre>
