<h2>How to Use</h2>
<ol>
  <li>To convert <strong>YAML → TOML</strong>: paste your YAML into the left panel — the TOML output updates automatically.</li>
  <li>To convert <strong>TOML → YAML</strong>: paste your TOML into the right panel — the YAML output updates automatically.</li>
  <li>Click <strong>Swap</strong> to exchange both panels and flip the conversion direction.</li>
  <li>Use <strong>Ctrl+Enter</strong> (or <strong>Cmd+Enter</strong> on Mac) as a keyboard shortcut from whichever panel you are editing.</li>
  <li>Click <strong>Copy</strong> on either panel to copy the result to your clipboard.</li>
</ol>

<h2>Worked Example</h2>

<h3>YAML input</h3>

```yaml
database:
  host: localhost
  port: 5432
  enabled: true
tags:
  - web
  - backend
```

<h3>Equivalent TOML output</h3>

```toml
[database]
host = "localhost"
port = 5432
enabled = true
tags = ["web", "backend"]
```

<h2>YAML vs TOML</h2>
<p>
  <strong>YAML</strong> is widely used for application configuration (Docker Compose, Kubernetes, GitHub Actions) due to its human-friendly, indentation-based syntax.
  <strong>TOML</strong> is the standard for Rust projects (<code>Cargo.toml</code>), Python packaging (<code>pyproject.toml</code>), and tools like Hugo and Zola.
  This tool converts between them for data types common to both formats.
</p>
