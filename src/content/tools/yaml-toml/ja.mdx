<h2>使い方</h2>
<ol>
  <li><strong>YAML → TOML</strong>：左パネルにYAMLを貼り付けると、TOML出力が自動的に更新されます。</li>
  <li><strong>TOML → YAML</strong>：右パネルにTOMLを貼り付けると、YAML出力が自動的に更新されます。</li>
  <li><strong>入れ替え</strong>をクリックすると、両パネルの内容が交換され変換方向が切り替わります。</li>
  <li>編集中のパネルで<strong>Ctrl+Enter</strong>（MacはCmd+Enter）のショートカットが使えます。</li>
  <li>各パネルの<strong>コピー</strong>ボタンで結果をクリップボードにコピーできます。</li>
</ol>

<h2>変換例</h2>

<h3>YAML入力</h3>

```yaml
database:
  host: localhost
  port: 5432
  enabled: true
tags:
  - web
  - backend
```

<h3>TOML出力</h3>

```toml
[database]
host = "localhost"
port = 5432
enabled = true
tags = ["web", "backend"]
```

<h2>YAMLとTOMLの違い</h2>
<p>
  <strong>YAML</strong>はインデントベースの構文で可読性が高く、Docker Compose・Kubernetes・GitHub Actionsなどのアプリケーション設定に広く使われています。
  <strong>TOML</strong>はRust（Cargo.toml）・Python（pyproject.toml）・HugoやZolaなどのツールの標準設定形式です。
  本ツールは両形式に共通するデータ型を相互変換します。
</p>
