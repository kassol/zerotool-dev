<h2>사용 방법</h2>
<ol>
  <li><strong>YAML → TOML</strong>: 왼쪽 패널에 YAML을 붙여넣고 <strong>YAML → TOML</strong>을 클릭합니다.</li>
  <li><strong>TOML → YAML</strong>: 오른쪽 패널에 TOML을 붙여넣고 <strong>TOML → YAML</strong>을 클릭합니다.</li>
  <li>편집 중인 패널에서 <strong>Ctrl+Enter</strong>(Mac은 Cmd+Enter) 단축키를 사용할 수 있습니다.</li>
  <li>각 패널의 <strong>복사</strong> 버튼으로 결과를 클립보드에 복사합니다.</li>
</ol>

<h2>변환 예시</h2>

<h3>YAML 입력</h3>

```yaml
database:
  host: localhost
  port: 5432
  enabled: true
tags:
  - web
  - backend
```

<h3>TOML 출력</h3>

```toml
[database]
host = "localhost"
port = 5432
enabled = true
tags = ["web", "backend"]
```

<h2>YAML과 TOML 비교</h2>
<p>
  <strong>YAML</strong>은 들여쓰기 기반 문법으로 가독성이 높아 Docker Compose, Kubernetes, GitHub Actions 등의 애플리케이션 설정에 널리 사용됩니다.
  <strong>TOML</strong>은 Rust(<code>Cargo.toml</code>), Python 패키징(<code>pyproject.toml</code>), Hugo, Zola 등 도구의 표준 설정 형식입니다.
  이 도구는 두 형식이 공통으로 지원하는 데이터 타입을 상호 변환합니다.
</p>
