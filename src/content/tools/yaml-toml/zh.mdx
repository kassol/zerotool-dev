<h2>使用方法</h2>
<ol>
  <li><strong>YAML → TOML</strong>：将 YAML 粘贴到左侧面板，TOML 输出自动更新。</li>
  <li><strong>TOML → YAML</strong>：将 TOML 粘贴到右侧面板，YAML 输出自动更新。</li>
  <li>点击 <strong>交换</strong> 按钮互换两侧内容并切换转换方向。</li>
  <li>可使用 <strong>Ctrl+Enter</strong>（Mac 上为 <strong>Cmd+Enter</strong>）快捷键直接转换当前编辑面板的内容。</li>
  <li>点击面板上的 <strong>Copy</strong> 按钮，将结果复制到剪贴板。</li>
</ol>

<h2>转换示例</h2>

<h3>YAML 输入</h3>

```yaml
database:
  host: localhost
  port: 5432
  enabled: true
tags:
  - web
  - backend
```

<h3>等效 TOML 输出</h3>

```toml
[database]
host = "localhost"
port = 5432
enabled = true
tags = ["web", "backend"]
```

<h2>YAML 与 TOML 对比</h2>
<p>
  <strong>YAML</strong> 以其人性化的缩进语法，广泛用于应用配置（Docker Compose、Kubernetes、GitHub Actions）。
  <strong>TOML</strong> 是 Rust 项目（<code>Cargo.toml</code>）、Python 打包（<code>pyproject.toml</code>）以及 Hugo、Zola 等工具的标准配置格式。
  本工具在两者之间对两种格式共同支持的数据类型进行转换。
</p>
