<h2>使用方法</h2>
<ol>
  <li>将 OpenAPI 规范（YAML 或 JSON）粘贴到编辑器中。</li>
  <li>点击<strong>验证</strong>或按 <kbd>Ctrl</kbd>+<kbd>Enter</kbd>。</li>
  <li>若有效，摘要面板将显示 API 名称、版本、路径数量和操作数量。</li>
  <li>若无效，每个错误将以 JSON 路径和描述的形式列出。</li>
</ol>

<h2>OpenAPI 3.x 必填字段</h2>
<ul>
  <li><code>openapi</code> — 版本字符串，如 <code>"3.0.3"</code> 或 <code>"3.1.0"</code></li>
  <li><code>info.title</code> — API 名称</li>
  <li><code>info.version</code> — API 版本，如 <code>"1.0.0"</code></li>
  <li><code>paths</code> — 路径条目映射，每个键必须以 <code>/</code> 开头</li>
</ul>

<h2>常见错误</h2>
<ul>
  <li>使用 Swagger 2.x 格式（<code>swagger: "2.0"</code>）而非 OpenAPI 3.x — 将 <code>swagger</code> 替换为 <code>openapi: "3.0.3"</code>，并将 <code>definitions</code> 更新为 <code>components/schemas</code>。</li>
  <li>操作中缺少 <code>responses</code> 对象 — 每个操作必须声明至少一个响应。</li>
  <li>路径键不以 <code>/</code> 开头 — 所有路径必须是绝对路径，如 <code>/users/&#123;id&#125;</code>。</li>
  <li>YAML 缩进错误 — 请先使用 <a href="/zh/tools/yaml-validator/">YAML 语法验证器</a> 修复语法问题。</li>
</ul>
