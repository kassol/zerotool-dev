<h2>How to Use</h2>
<ol>
  <li>Paste your OpenAPI spec (YAML or JSON) into the editor.</li>
  <li>Click <strong>Validate</strong> or press <kbd>Ctrl</kbd>+<kbd>Enter</kbd>.</li>
  <li>If valid, a summary panel shows the API title, version, path count, and operation count.</li>
  <li>If invalid, each error is listed with its JSON path and a description.</li>
</ol>

<h2>Required OpenAPI 3.x Fields</h2>
<ul>
  <li><code>openapi</code> — version string, e.g. <code>"3.0.3"</code> or <code>"3.1.0"</code></li>
  <li><code>info.title</code> — human-readable API name</li>
  <li><code>info.version</code> — API version, e.g. <code>"1.0.0"</code></li>
  <li><code>paths</code> — map of path item objects; each key must begin with <code>/</code></li>
</ul>

<h2>Common Mistakes</h2>
<ul>
  <li>Using Swagger 2.x format (<code>swagger: "2.0"</code>) instead of OpenAPI 3.x — migrate by replacing <code>swagger</code> with <code>openapi: "3.0.3"</code> and updating <code>definitions</code> to <code>components/schemas</code>.</li>
  <li>Missing <code>responses</code> object inside an operation — every operation must declare at least one response.</li>
  <li>Path keys that do not begin with <code>/</code> — all paths must be absolute, e.g. <code>/users/&#123;id&#125;</code>.</li>
  <li>YAML indentation errors — use the <a href="/tools/yaml-validator">YAML Validator</a> to fix syntax issues first.</li>
</ul>
