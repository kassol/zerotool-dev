<h2>사용 방법</h2>
<ol>
  <li>OpenAPI 스펙(YAML 또는 JSON)을 에디터에 붙여넣습니다.</li>
  <li><strong>검증</strong>을 클릭하거나 <kbd>Ctrl</kbd>+<kbd>Enter</kbd>를 누릅니다.</li>
  <li>유효하면 API 제목, 버전, 경로 수, 오퍼레이션 수가 요약 패널에 표시됩니다.</li>
  <li>유효하지 않으면 각 오류가 JSON 경로와 설명과 함께 나열됩니다.</li>
</ol>

<h2>OpenAPI 3.x 필수 필드</h2>
<ul>
  <li><code>openapi</code> — 버전 문자열 (예: <code>"3.0.3"</code> 또는 <code>"3.1.0"</code>)</li>
  <li><code>info.title</code> — API 이름</li>
  <li><code>info.version</code> — API 버전 (예: <code>"1.0.0"</code>)</li>
  <li><code>paths</code> — 경로 항목 맵; 각 키는 <code>/</code>로 시작해야 함</li>
</ul>

<h2>자주 발생하는 실수</h2>
<ul>
  <li>OpenAPI 3.x 대신 Swagger 2.x 형식(<code>swagger: "2.0"</code>) 사용 — <code>swagger</code>를 <code>openapi: "3.0.3"</code>으로 변경하고 <code>definitions</code>를 <code>components/schemas</code>로 업데이트하세요.</li>
  <li>오퍼레이션 내에 <code>responses</code> 객체 없음 — 모든 오퍼레이션은 최소 하나의 응답을 선언해야 합니다.</li>
  <li><code>/</code>로 시작하지 않는 경로 키 — 모든 경로는 절대 경로여야 합니다 (예: <code>/users/&#123;id&#125;</code>).</li>
  <li>YAML 들여쓰기 오류 — 먼저 <a href="/ko/tools/yaml-validator/">YAML 유효성 검사기</a>로 구문 문제를 해결하세요.</li>
</ul>
