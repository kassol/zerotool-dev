    answer: "RSA-OAEP is used for asymmetric encryption (encrypt data with public key, decrypt with private key). RSASSA-PKCS1-v1_5 is used for digital signing (sign with private key, verify with public key)."
  - question: "Which key size should I choose?"
    answer: "2048-bit is the current industry minimum and is sufficient for most use cases. Choose 4096-bit for long-lived keys or when regulations require higher security margins. 4096-bit generation takes noticeably longer."
  - question: "What is the PEM format?"
    answer: "PEM (Privacy Enhanced Mail) is a Base64-encoded DER certificate wrapped in -----BEGIN/END----- headers. It is the most common format for TLS certificates, SSH keys, and API keys."
  - question: "What is the JWK format?"
    answer: "JWK (JSON Web Key) is a JSON representation of cryptographic keys defined by RFC 7517. It is commonly used in OAuth 2.0, OIDC, and JWT authentication flows."

<h2>How to Use</h2>
<ol>
  <li>Select a <strong>Key Size</strong> (2048-bit or 4096-bit).</li>
  <li>Select an <strong>Algorithm</strong>: RSA-OAEP for encryption use cases, RSASSA-PKCS1-v1_5 for signing.</li>
  <li>Select an output <strong>Format</strong>: PEM or JWK.</li>
  <li>Click <strong>Generate Key Pair</strong> and wait a moment for generation to complete.</li>
  <li>Copy or download the public and private keys separately.</li>
</ol>

<h2>Security Notes</h2>
<ul>
  <li>Keep your <strong>private key secret</strong>. Anyone with the private key can decrypt messages or forge signatures.</li>
  <li>Share only your <strong>public key</strong> — it is designed to be public.</li>
  <li>This tool generates keys for testing and development. For production use, manage keys with a dedicated secret manager (AWS KMS, HashiCorp Vault, etc.).</li>
</ul>
