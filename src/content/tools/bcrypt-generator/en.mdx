<h2>How to Use</h2>

<h3>Generate Mode</h3>
<ol>
  <li>Enter the plaintext password in the <strong>Password</strong> field.</li>
  <li>Adjust the <strong>Cost Factor</strong> slider (default: 12). The estimated processing time updates in real time.</li>
  <li>Click <strong>Generate Hash</strong>. The UI stays responsive while the Web Worker computes the result.</li>
  <li>Click <strong>Copy</strong> to copy the hash to your clipboard.</li>
</ol>

<h3>Verify Mode</h3>
<ol>
  <li>Switch to the <strong>Verify</strong> tab.</li>
  <li>Enter the plaintext password and the stored bcrypt hash.</li>
  <li>Click <strong>Verify</strong>. The result shows <strong>Match</strong> or <strong>No Match</strong>.</li>
</ol>

<h2>Security Notes</h2>
<ul>
  <li>Never use MD5, SHA-1, or unsalted SHA-256 to hash passwords — they are too fast and vulnerable to precomputed rainbow tables.</li>
  <li>Bcrypt hashes include the cost factor in the output string, so stored hashes remain verifiable even after you raise the cost factor for new registrations.</li>
  <li>This tool uses the <a href="https://github.com/dcodeIO/bcrypt.js" target="_blank" rel="noopener">bcryptjs</a> library — a pure JavaScript port compatible with all bcrypt implementations.</li>
</ul>
