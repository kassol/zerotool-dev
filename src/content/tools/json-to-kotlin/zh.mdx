faqItems:
  - question: "这个工具生成什么？"
    answer: "生成带有 @Serializable 注解的 Kotlin data class 声明（使用 kotlinx.serialization）。每个嵌套对象都会成为独立的命名 data class。"
  - question: "如何处理可空字段？"
    answer: "如果 JSON 字段的值为 null，对应的 Kotlin 类型将标记为可空（如 String?）。数组中某些元素缺少的字段也会标记为可空，默认值为 null。"
  - question: "各类型的默认值是什么？"
    answer: "String 默认为空字符串 \"\"，Int 为 0，Double 为 0.0，Boolean 为 false，List 为 emptyList()，可空类型为 null，嵌套 data class 调用零参数构造函数。"
  - question: "为什么会添加 @SerialName 注解？"
    answer: "当 JSON 键无法直接用作有效的 Kotlin 标识符（如含连字符或空格），或驼峰转换改变了名称时，@SerialName 会保留原始 JSON 键名。"
  - question: "数据会发送到服务器吗？"
    answer: "不会。所有转换都在浏览器中运行，数据不离开本地。"

<h2>使用方法</h2>
<ol>
  <li>将 JSON 粘贴或输入到左侧面板，工具会实时验证。</li>
  <li>可选：设置<strong>根类名称</strong>（默认：<code>RootObject</code>）。</li>
  <li>点击<strong>生成 Kotlin</strong> 或等待实时转换。</li>
  <li>点击<strong>复制</strong>将输出复制到剪贴板。</li>
</ol>

<h2>类型映射</h2>
<ul>
  <li><code>null</code> → <code>Any?</code></li>
  <li>字符串 → <code>String</code></li>
  <li>整数 → <code>Int</code></li>
  <li>浮点数 → <code>Double</code></li>
  <li>布尔值 → <code>Boolean</code></li>
  <li>数组 → <code>List&lt;T&gt;</code></li>
  <li>嵌套对象 → 独立的 <code>data class</code></li>
</ul>
