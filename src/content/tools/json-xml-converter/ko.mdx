<h2>사용 방법</h2>
<ol>
  <li><strong>JSON → XML</strong>: 왼쪽 패널에 JSON을 붙여넣고 <strong>JSON → XML</strong> 버튼을 클릭하거나, 입력하면 실시간으로 변환됩니다.</li>
  <li><strong>XML → JSON</strong>: 오른쪽 패널에 XML을 붙여넣고 <strong>XML → JSON</strong> 버튼을 클릭하세요.</li>
  <li><strong>루트 요소</strong> 입력란에서 XML 출력의 루트 요소 이름을 커스터마이징하세요.</li>
  <li><strong>보기 좋게 출력</strong>을 토글하여 들여쓰기 여부를 제어하세요.</li>
  <li>편집 중인 패널에서 <strong>Ctrl+Enter</strong>(Mac: <strong>Cmd+Enter</strong>)로 변환할 수 있습니다.</li>
</ol>

<h2>변환 예시</h2>

<h3>단순 객체</h3>
<p>입력 JSON:</p>

```json
{"name": "Alice", "age": 30, "active": true}
```

<p>출력 XML (루트 요소: <code>root</code>):</p>

```xml
<?xml version="1.0" encoding="UTF-8"?>
<root>
  <name>Alice</name>
  <age>30</age>
  <active>true</active>
</root>
```

<h3>중첩 객체</h3>

```json
{"server": {"host": "localhost", "port": 8080}}
```

```xml
<?xml version="1.0" encoding="UTF-8"?>
<root>
  <server>
    <host>localhost</host>
    <port>8080</port>
  </server>
</root>
```

<h3>배열</h3>

```json
{"tags": ["js", "xml", "api"]}
```

```xml
<?xml version="1.0" encoding="UTF-8"?>
<root>
  <tags>js</tags>
  <tags>xml</tags>
  <tags>api</tags>
</root>
```

<h2>JSON vs XML</h2>
<p>
  <strong>JSON</strong>은 REST API와 웹 서비스의 표준 데이터 형식으로 간결하고 가독성이 높으며 모든 언어에서 기본 지원됩니다.
  <strong>XML</strong>은 엔터프라이즈 시스템, SOAP API, 설정 파일(Maven, Spring), 문서 형식(SVG, XHTML, RSS)에서 여전히 널리 사용됩니다.
  이 도구는 서로 다른 데이터 형식을 사용하는 시스템 간 통합을 도와줍니다.
</p>
