  - question: "HMAC이란 무엇입니까?"
    answer: "HMAC(해시 기반 메시지 인증 코드)는 암호화 해시 함수와 비밀 키를 결합한 메시지 인증 코드입니다. 데이터 무결성과 메시지의 진위성을 동시에 검증하는 데 사용됩니다."

  - question: "어떤 알고리즘을 선택해야 합니까?"
    answer: "SHA-256은 가장 널리 사용되며 대부분의 목적에 적합합니다. SHA-384와 SHA-512는 더 큰 보안 마진을 제공하지만 출력이 더 깁니다. 세 가지 모두 HMAC 사용에 안전한 것으로 간주됩니다."
  - question: "Hex와 Base64 출력의 차이는 무엇입니까?"

<h2>사용 방법</h2>
<ol>
  <li>인증할 <strong>메시지</strong>를 입력합니다.</li>
  <li><strong>비밀 키</strong>를 입력합니다.</li>
  <li><strong>알고리즘</strong>(SHA-256, SHA-384, SHA-512)을 선택합니다.</li>
  <li>출력 형식(<strong>Hex</strong> 또는 <strong>Base64</strong>)을 선택합니다.</li>
  <li><strong>Generate HMAC</strong>을 클릭하고 결과를 복사합니다.</li>
</ol>

<h2>주요 사용 사례</h2>
<ul>
  <li><strong>웹훅 검증:</strong> 웹훅 페이로드가 신뢰할 수 있는 발신자에 의해 서명되었는지 확인.</li>
  <li><strong>API 요청 서명:</strong> 비밀 키 자체를 전송하지 않고 API 요청의 진위성 증명.</li>
  <li><strong>데이터 무결성:</strong> 메시지에 HMAC를 첨부하여 수신자가 변조되지 않았음을 확인.</li>
  <li><strong>세션 토큰:</strong> 변조 방지 세션 식별자 생성.</li>
</ul>
