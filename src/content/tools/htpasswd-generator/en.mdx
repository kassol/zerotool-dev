<h2>How to Use</h2>
<ol>
  <li>Enter a <strong>Username</strong> and <strong>Password</strong>, or click <strong>Generate</strong> to create a random password.</li>
  <li>Select an <strong>Algorithm</strong> (bcrypt is recommended).</li>
  <li>If using bcrypt, adjust the <strong>cost factor</strong> (higher = more secure but slower).</li>
  <li>Click <strong>Generate htpasswd line</strong>.</li>
  <li>Copy the result and add it to your <code>.htpasswd</code> file.</li>
</ol>

<h2>Algorithm Comparison</h2>

| Algorithm | Format prefix | Security | Notes |
|-----------|--------------|----------|-------|
| bcrypt | `$2y$` | Excellent | Recommended for all new deployments |
| MD5 (APR1) | `$apr1$` | Moderate | Apache-specific MD5 variant with salt |
| SHA1 | `{SHA}` | Weak | No salt; vulnerable to rainbow tables |
| Plaintext | (none) | None | For testing only — never use in production |

<h2>Apache Configuration Example</h2>

```apache
# .htaccess
AuthType Basic
AuthName "Restricted Area"
AuthUserFile /path/to/.htpasswd
Require valid-user
```

<h2>Batch Mode</h2>
<p>Enter multiple <code>user:password</code> pairs (one per line) in the Batch mode section and click <strong>Generate batch</strong> to produce an entire <code>.htpasswd</code> file at once.</p>

<h2>Security Notes</h2>
<ul>
  <li>Always store <code>.htpasswd</code> outside your document root so it cannot be served directly.</li>
  <li>Prefer bcrypt with cost ≥ 10 for production servers.</li>
  <li>Regenerate hashes if a password is compromised — there is no way to reverse a bcrypt hash.</li>
</ul>
