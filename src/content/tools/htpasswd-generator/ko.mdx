<h2>사용 방법</h2>
<ol>
  <li><strong>사용자 이름</strong>과 <strong>비밀번호</strong>를 입력하거나, <strong>생성</strong>을 클릭하여 임의의 비밀번호를 만듭니다.</li>
  <li><strong>알고리즘</strong>을 선택합니다(bcrypt 권장).</li>
  <li>bcrypt를 사용하는 경우 <strong>비용 인수</strong>를 조정합니다(높을수록 안전하지만 느림).</li>
  <li><strong>htpasswd 줄 생성</strong>을 클릭합니다.</li>
  <li>결과를 복사하여 <code>.htpasswd</code> 파일에 추가합니다.</li>
</ol>

<h2>알고리즘 비교</h2>

| 알고리즘 | 형식 접두사 | 보안 | 설명 |
|---------|-----------|------|------|
| bcrypt | `$2y$` | 우수 | 모든 새 배포에 권장 |
| MD5 (APR1) | `$apr1$` | 보통 | 솔트가 있는 Apache 전용 MD5 변형 |
| SHA1 | `{SHA}` | 약함 | 솔트 없음, 레인보우 테이블에 취약 |
| 평문 | (없음) | 없음 | 테스트 전용 — 프로덕션에서는 절대 사용 금지 |

<h2>Apache 구성 예시</h2>

```apache
# .htaccess
AuthType Basic
AuthName "제한 구역"
AuthUserFile /path/to/.htpasswd
Require valid-user
```

<h2>일괄 모드</h2>
<p>일괄 모드 섹션에 여러 <code>user:password</code> 쌍을 한 줄에 하나씩 입력하고 <strong>일괄 생성</strong>을 클릭하면 전체 <code>.htpasswd</code> 파일을 한 번에 생성할 수 있습니다.</p>

<h2>보안 참고사항</h2>
<ul>
  <li><code>.htpasswd</code> 파일은 항상 문서 루트 외부에 저장하여 직접 접근이 불가능하도록 하세요.</li>
  <li>프로덕션 서버에는 비용 인수 10 이상의 bcrypt를 사용하세요.</li>
  <li>비밀번호가 유출된 경우 즉시 해시를 재생성하세요 — bcrypt 해시는 역산할 수 없습니다.</li>
</ul>
