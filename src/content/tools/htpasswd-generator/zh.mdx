<h2>使用方法</h2>
<ol>
  <li>输入<strong>用户名</strong>和<strong>密码</strong>，或点击<strong>随机生成</strong>创建随机密码。</li>
  <li>选择<strong>算法</strong>（推荐 bcrypt）。</li>
  <li>如果使用 bcrypt，可调整<strong>成本因子</strong>（越高越安全，但越慢）。</li>
  <li>点击<strong>生成 htpasswd 行</strong>。</li>
  <li>复制结果并添加到 <code>.htpasswd</code> 文件。</li>
</ol>

<h2>算法对比</h2>

| 算法 | 格式前缀 | 安全性 | 说明 |
|------|---------|--------|------|
| bcrypt | `$2y$` | 极佳 | 推荐用于所有新部署 |
| MD5 (APR1) | `$apr1$` | 中等 | Apache 特有的带盐 MD5 变体 |
| SHA1 | `{SHA}` | 弱 | 无盐，易受彩虹表攻击 |
| 明文 | （无）| 无 | 仅用于测试，生产环境禁用 |

<h2>Apache 配置示例</h2>

```apache
# .htaccess
AuthType Basic
AuthName "受限区域"
AuthUserFile /path/to/.htpasswd
Require valid-user
```

<h2>批量模式</h2>
<p>在批量模式区域输入多个 <code>user:password</code> 对（每行一对），点击<strong>批量生成</strong>即可一次生成完整的 <code>.htpasswd</code> 文件。</p>

<h2>安全建议</h2>
<ul>
  <li>始终将 <code>.htpasswd</code> 文件存储在文档根目录之外，防止被直接访问。</li>
  <li>生产服务器推荐使用成本因子 ≥ 10 的 bcrypt。</li>
  <li>密码泄露后立即重新生成哈希值——bcrypt 哈希无法逆向还原。</li>
</ul>
