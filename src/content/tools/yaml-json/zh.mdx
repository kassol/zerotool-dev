<h2>使用方法</h2>
<ol>
  <li><strong>YAML → JSON</strong>：将 YAML 数据粘贴到左侧面板，点击 <strong>YAML → JSON</strong> 按钮。</li>
  <li><strong>JSON → YAML</strong>：将 JSON 数据粘贴到右侧面板，点击 <strong>JSON → YAML</strong> 按钮。</li>
  <li>可使用 <strong>Ctrl+Enter</strong>（Mac 上为 <strong>Cmd+Enter</strong>）快捷键直接转换当前面板的内容。</li>
  <li>点击面板上的 <strong>Copy</strong> 按钮，将结果复制到剪贴板。</li>
</ol>

<h2>转换示例</h2>

<h3>简单对象</h3>
<p>输入 YAML：</p>

```yaml
name: Alice
age: 30
active: true
```

<p>输出 JSON 包含 name、age 和 active 字段。</p>

<h3>嵌套结构与数组</h3>
<p>输入 YAML：</p>

```yaml
server:
  host: localhost
  port: 8080
  features:
    - auth
    - logging
    - cache
```

<p>输出 JSON 包含嵌套的 server 对象，含 features 数组。</p>

<h3>往返转换</h3>
<p>将 JSON 输出粘贴回右侧面板，点击 <strong>JSON → YAML</strong> 即可往返转换数据。注意，YAML 注释和格式偏好在往返转换中不会保留。</p>

<h2>YAML 与 JSON 对比</h2>
<p>
  <strong>YAML</strong>（YAML Ain't Markup Language）是一种人类友好的数据序列化格式，广泛用于配置文件（Docker Compose、Kubernetes、Ansible、GitHub Actions）。
  <strong>JSON</strong>（JavaScript Object Notation）是 API 和 Web 服务中最常用的数据交换格式。
  两者可以表示相同的数据结构——本工具在它们之间无损转换（YAML 独有的注释和锚点除外）。
</p>
