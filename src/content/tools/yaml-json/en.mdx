<h2>How to Use</h2>
<ol>
  <li>To convert <strong>YAML → JSON</strong>: paste your YAML into the left panel and click <strong>YAML → JSON</strong>.</li>
  <li>To convert <strong>JSON → YAML</strong>: paste your JSON into the right panel and click <strong>JSON → YAML</strong>.</li>
  <li>Use <strong>Ctrl+Enter</strong> (or <strong>Cmd+Enter</strong> on Mac) as a keyboard shortcut to convert from whichever panel you're typing in.</li>
  <li>Click <strong>Copy</strong> on either panel to copy the result to your clipboard.</li>
</ol>

<h2>Worked Examples</h2>

<h3>Simple object</h3>
<p>Input YAML:</p>

```yaml
name: Alice
age: 30
active: true
```

<p>Output JSON contains name, age, and active fields.</p>

<h3>Nested structure with arrays</h3>
<p>Input YAML:</p>

```yaml
server:
  host: localhost
  port: 8080
  features:
    - auth
    - logging
    - cache
```

<p>Output JSON contains nested server object with features array.</p>

<h3>Round-trip conversion</h3>
<p>Paste the JSON output back into the right panel and click <strong>JSON → YAML</strong> to round-trip your data. Note that YAML comments and formatting preferences are not preserved in round-trips.</p>

<h2>YAML vs JSON</h2>
<p>
  <strong>YAML</strong> (YAML Ain't Markup Language) is a human-friendly data serialization format popular for configuration files (Docker Compose, Kubernetes, Ansible, GitHub Actions).
  <strong>JSON</strong> (JavaScript Object Notation) is the most widely used data interchange format for APIs and web services.
  Both can represent the same data structures — this tool converts between them losslessly (except for YAML-only features like comments and anchors).
</p>
