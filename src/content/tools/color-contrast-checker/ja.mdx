<h2>使い方</h2>
<ol>
  <li>左に <strong>前景色 HEX</strong>（テキスト・記号の色）、右に <strong>背景色 HEX</strong> を入力。</li>
  <li>ネイティブカラーピッカー・直接入力・デザインツールからのペーストに対応。3 桁省略形（<code>#abc</code>）は自動的に 6 桁に展開。</li>
  <li>コントラスト比はリアルタイム更新。例: <code>12.6 : 1</code>、AA / AAA 合否バッジ付き。</li>
  <li>3 つのステータスカードで <strong>本文</strong>・<strong>大きな文字</strong>・<strong>UI コンポーネント</strong> の AA / AAA 結果を表示。</li>
  <li>本文 AA（4.5 : 1）が不合格の場合、修正案カードが現れ、<strong>修正案を適用</strong> をクリックすると色相を保ったまま明度を調整。</li>
</ol>

<h2>WCAG コントラスト比の仕組み</h2>
<p>WCAG 2.x のコントラスト比は 2 色の<em>相対輝度</em>から計算します。各 sRGB チャンネルを区分関数で gamma 復号し、<code>0.2126·R + 0.7152·G + 0.0722·B</code> で重み付けして輝度を求め、比は <code>(L1 + 0.05) / (L2 + 0.05)</code>（L1 は明るい方）。値は 1（コントラストなし）〜21（純黒×純白）の範囲。</p>

<h2>本文・大きな文字・UI の 3 階層</h2>
<p>WCAG 2 はユースケースごとに目標値を分けています：</p>
<ul>
  <li><strong>本文（達成基準 1.4.3）</strong>—— AA ≥ <code>4.5 : 1</code>、AAA ≥ <code>7 : 1</code>。18&nbsp;px 未満通常または 14&nbsp;px 未満太字のテキストが対象。</li>
  <li><strong>大きな文字</strong>—— AA ≥ <code>3 : 1</code>、AAA ≥ <code>4.5 : 1</code>。18.66&nbsp;px 太字以上または 24&nbsp;px 通常以上のテキストが対象。</li>
  <li><strong>UI コンポーネントと図形（達成基準 1.4.11）</strong>—— ≥ <code>3 : 1</code>。枠線・フォーカスリング・アイコン・グラフ線などの非テキスト視覚要素を対象。WCAG 2 はここでは AAA を規定していません。</li>
</ul>

<h2>「色相を保ったまま明度を調整」する理由</h2>
<p>ブランドカラーやデザイントークンは輝度ではなく色相のアイデンティティで選ばれることが多いため、コントラスト不足のときに黒や白へ切り替えると視覚言語が崩れます。本ツールは前景の OKLCH 明度軸（知覚的に均一）を歩み、原色相と彩度を保持したまま、現在の背景に対して <code>4.5 : 1</code> を最初に満たす明度で停止します。結果として元の色に最も近く本文 AA を通過する色が得られます。色相が sRGB 色域内で目標に届かない場合は、合格する黒または白にフォールバックします。</p>

<h2>プライバシー</h2>
