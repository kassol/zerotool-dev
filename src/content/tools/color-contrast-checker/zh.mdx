<h2>使用方法</h2>
<ol>
  <li>左侧填 <strong>前景色 HEX</strong>（文字或符号的颜色），右侧填 <strong>背景色 HEX</strong>。</li>
  <li>可使用原生取色器、直接输入或从设计工具粘贴。三位简写（<code>#abc</code>）会自动扩展为六位。</li>
  <li>对比度数值实时更新——例如 <code>12.6 : 1</code>，并显示 AA / AAA 通过徽章。</li>
  <li>三张状态卡分别按 <strong>正文</strong>、<strong>大号文字</strong>、<strong>UI 组件</strong> 给出 AA、AAA 结果。</li>
  <li>当正文 AA（4.5 : 1）不通过时，自动出现修正建议卡，<strong>应用修正</strong> 一键完成色相不变的亮度调整。</li>
</ol>

<h2>WCAG 对比度公式</h2>
<p>WCAG 2.x 对比度基于两种颜色的<em>相对亮度</em>计算。每个 sRGB 通道经分段 gamma 解码后，按 <code>0.2126·R + 0.7152·G + 0.0722·B</code> 加权得到亮度，对比度公式为 <code>(L1 + 0.05) / (L2 + 0.05)</code>，其中 L1 为较亮一方。结果取值范围 1（无对比）到 21（纯黑配纯白）。</p>

<h2>三档场景：正文 / 大号 / UI</h2>
<p>WCAG 2 按使用场景分档：</p>
<ul>
  <li><strong>正文（成功标准 1.4.3）</strong>——AA ≥ <code>4.5 : 1</code>，AAA ≥ <code>7 : 1</code>。涵盖小于 18&nbsp;px 常规字号或小于 14&nbsp;px 粗体的文字。</li>
  <li><strong>大号文字</strong>——AA ≥ <code>3 : 1</code>，AAA ≥ <code>4.5 : 1</code>。涵盖 18.66&nbsp;px 粗体及以上或 24&nbsp;px 常规及以上的文字。</li>
  <li><strong>UI 组件与图形（成功标准 1.4.11）</strong>——≥ <code>3 : 1</code>。涵盖边框、聚焦环、图标、图表线条等非文字视觉元素。WCAG 2 在此处未规定 AAA 阈值。</li>
</ul>

<h2>为什么是「保留色相、调整亮度」</h2>
<p>品牌色与设计 token 通常按色相身份选定，而非按亮度。当对比不达标时，直接换成纯黑或纯白会破坏视觉语言。本工具沿前景的 OKLCH 亮度轴（感知均匀）行进，保留原始色相与饱和度，在第一个达到 <code>4.5 : 1</code> 的亮度处停下——结果是离原色最近、且能通过正文 AA 的颜色。若该色相在 sRGB 色域内无法达成目标，工具回退到通过的纯黑或纯白。</p>
