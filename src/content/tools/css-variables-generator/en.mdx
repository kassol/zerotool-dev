<h2>How to Use</h2>
<ol>
  <li>Set the <strong>Variable Prefix</strong> (default <code>--</code>, change to <code>--brand-</code> etc.).</li>
  <li>Edit the pre-filled tokens in each group, or add new ones with <strong>+ Add Token</strong>.</li>
  <li>For color tokens, use the color picker swatch or type a hex value directly.</li>
  <li>The <strong>Generated CSS</strong> block updates in real time.</li>
  <li>Click <strong>Copy</strong> to copy the <code>:root &#123; ... &#125;</code> block to your clipboard.</li>
</ol>

<h2>What Are Design Tokens?</h2>
<p>
  Design tokens are the single source of truth for your design decisions — color palettes,
  type scales, spacing rhythms, and corner radii. By storing them as CSS custom properties
  in a <code>:root</code> block, you make every value accessible throughout your entire
  stylesheet. Changing a token in one place cascades to every component that uses it.
</p>

<h2>Token Groups</h2>
<ul>
  <li><strong>Colors</strong> — brand palette, background, text, and state colors with a color picker.</li>
  <li><strong>Typography</strong> — font family stacks and base/scale font sizes.</li>
  <li><strong>Spacing</strong> — a consistent spacing scale (4 px, 8 px, 16 px, etc.).</li>
  <li><strong>Border Radius</strong> — corner radius values from subtle to pill-shaped.</li>
</ul>
