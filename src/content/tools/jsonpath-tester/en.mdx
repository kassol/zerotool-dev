faqItems:
  - question: "What is JSONPath?"
    answer: "JSONPath is a query language for JSON, analogous to XPath for XML. It uses a path expression to select nodes from a JSON document. JSONPath is widely used in APIs, testing frameworks, and tools like Kubernetes, AWS Step Functions, and Postman."
  - question: "What does $ mean in a JSONPath expression?"
    answer: "$ refers to the root element of the JSON document. Every valid JSONPath expression must start with $. On its own, $ selects the entire document. You chain selectors after it, such as $.store.book[0].title."
  - question: "How do I select all items in an array?"
    answer: "Use the wildcard selector [*] after the array's key. For example, $.store.book[*] returns all book objects in the array. You can further drill into each item: $.store.book[*].author returns every author value."
  - question: "How do filter expressions work?"
    answer: "Filter expressions use the syntax [?(@.property operator value)]. The @ symbol refers to the current element. For example, $.store.book[?(@.price < 10)] returns all books with a price less than 10. Supported operators include ==, !=, <, >, <=, >=."
  - question: "What is recursive descent (..) ?"
    answer: "The .. operator recursively searches all descendants of the current node. For example, $..author finds every author field anywhere in the JSON tree, regardless of nesting depth."

<h2>JSONPath Syntax Reference</h2>
<ul>
  <li><code>$</code> — root element</li>
  <li><code>.key</code> — child property (dot notation)</li>
  <li><code>['key']</code> — child property (bracket notation)</li>
  <li><code>[n]</code> — array element at index n</li>
  <li><code>[*]</code> — all array elements or object properties</li>
  <li><code>..key</code> — recursive descent — finds key anywhere in the tree</li>
  <li><code>[?(@.prop op value)]</code> — filter expression</li>
</ul>
