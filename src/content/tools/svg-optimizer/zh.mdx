<h2>使用方式</h2>
<ol>
  <li>粘贴 SVG 代码，或切到「上传文件」标签页拖放 <code>.svg</code> 文件。</li>
  <li>勾选需要的插件。默认配置对齐 SVGO 的 <em>preset-default</em>；主面板 8 个 toggle 覆盖最常见的优化决策。</li>
  <li>点击<strong>压缩 SVG</strong>。优化前 / 优化后会同时显示字节数、节省百分比和实际渲染效果，方便立刻发现视觉回归。</li>
  <li>点<strong>复制结果</strong>把代码贴回项目，或<strong>下载 .svg</strong>保存清理后的文件。</li>
</ol>

<h2>哪些内容会被去掉</h2>
<p>
  SVGO 会去掉编辑器元数据、注释、隐藏元素、默认属性以及其他不影响渲染的标记，同时重写 path 数据、合并相邻 path、缩短颜色值。常见图标体积一般缩小 40–70%，主要节省来自 <code>convertPathData</code>、<code>mergePaths</code> 和 <code>cleanupNumericValues</code>。
</p>

<h2>两个高风险 toggle</h2>
<p>
  <strong>去 viewBox</strong> 和 <strong>去 width/height</strong> 默认关闭是有原因的：没有 <code>viewBox</code> 的 SVG 会失去响应式缩放能力，没有 <code>width</code> / <code>height</code> 的 SVG 会从父元素继承尺寸 — 对独立下载的图标来说基本不是你想要的结果。只有在完全掌控嵌入上下文时才打开（比如 sprite sheet 的父级 <code>&lt;use&gt;</code> 已经指定尺寸）。
</p>

<h2>什么时候压缩</h2>
<ul>
  <li><strong>生产构建：</strong>用 <code>vite-plugin-svgo</code> 之类把 SVGO 装进打包流程；本工具用于无 CLI 时快速核对结果。</li>
  <li><strong>手写或导出的 SVG：</strong>Figma、Illustrator、Sketch 都会留下大量编辑器元数据和无引用的 <code>id</code> 属性，SVGO 能清得很干净。</li>
  <li><strong>HTML 中内联的 SVG：</strong>每一字节都会推迟首屏渲染，越小越好。</li>
</ul>

<h2>按设计单文件</h2>
<p>
  本工具一次优化一份 SVG。批量处理、自定义插件、CI 集成请用 SVGO 命令行 — <code>npm install -g svgo</code> 之后跑 <code>svgo input.svg -o output.svg</code> 就能处理整个目录或 glob。在这里调好 toggle 组合后，把对应的 <code>svgo.config.js</code> 片段从 SVGO 文档里复制到构建里就能复用。
</p>
