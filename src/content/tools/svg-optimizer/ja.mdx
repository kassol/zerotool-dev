<h2>使い方</h2>
<ol>
  <li>SVG マークアップを貼り付け、または「ファイルをアップロード」タブで <code>.svg</code> をドロップします。</li>
  <li>必要なプラグインを切り替えます。既定値は SVGO の <em>preset-default</em> に揃え、主トグル 8 件で最頻出の判断をカバーします。</li>
  <li><strong>SVG を最適化</strong>を押下。最適化前 / 後でバイト数・削減率・実際の描画結果を並べて表示し、視覚的なリグレッションを即座に確認できます。</li>
  <li><strong>出力をコピー</strong>でコードベースに即貼付、または<strong>.svg をダウンロード</strong>でファイル保存。</li>
</ol>

<h2>削除される内容</h2>
<p>
  SVGO はエディタのメタデータ、コメント、非表示要素、既定属性など描画に影響しないマークアップを削除し、パスデータを書き直し、隣接パスを統合し、色記法を短縮します。一般的なアイコンは 40–70% の削減が見込め、その大部分は <code>convertPathData</code>・<code>mergePaths</code>・<code>cleanupNumericValues</code> によるものです。
</p>

<h2>注意すべき 2 つのトグル</h2>
<p>
  <strong>viewBox を削除</strong>と<strong>width/height を削除</strong>が既定でオフなのには理由があります。<code>viewBox</code> がない SVG はレスポンシブにスケールできず、<code>width</code> / <code>height</code> がない SVG は親要素から描画ボックスを継承します — ダウンロード可能なアイコンとして配布する場合はまず望ましくない結果です。埋め込みコンテキストを完全に制御している場合（親 <code>&lt;use&gt;</code> でサイズを指定したスプライトシートなど）に限り有効化してください。
</p>

<h2>最適化のタイミング</h2>
<ul>
  <li><strong>本番ビルド：</strong><code>vite-plugin-svgo</code> 等でバンドラ側に SVGO を組み込みます。本ツールは CLI を起動せず結果を即確認したい場合に便利です。</li>
  <li><strong>手書き / エクスポートされた SVG：</strong>Figma・Illustrator・Sketch はエディタのメタデータと未使用 <code>id</code> 属性を大量に残します。SVGO はそれらをきれいに除去します。</li>
  <li><strong>HTML へのインライン SVG：</strong>マークアップが直接埋め込まれる場合、1 KB ごとに初回描画が遅れます — 小さければ小さいほど良好です。</li>
</ul>

<h2>仕様：1 ファイル単位</h2>
<p>
  本ツールは 1 ファイルずつ最適化します。バッチ最適化、カスタムプラグイン記述、CI 統合は SVGO コマンドラインに任せましょう — <code>npm install -g svgo</code> 後、<code>svgo input.svg -o output.svg</code> でディレクトリや glob を一括処理できます。本ツールでトグル構成を確定したら、SVGO 公式ドキュメントから対応する <code>svgo.config.js</code> をコピーしてビルドに組み込めます。
</p>

<p>
  最適化後に PNG へ変換、または React へ抽出したい場合は <a href="/ja/tools/svg-to-png-converter/">SVG → PNG 変換</a> と <a href="/ja/tools/svg-to-jsx/">SVG → JSX</a> を続けて使用してください。
</p>
