<h2>사용 방법</h2>
<ol>
  <li>SVG 마크업을 붙여넣거나, 「파일 업로드」 탭으로 전환해 <code>.svg</code> 파일을 드롭합니다.</li>
  <li>필요한 플러그인을 토글합니다. 기본값은 SVGO의 <em>preset-default</em>에 맞춰져 있고, 주 토글 8개가 가장 자주 내리는 결정을 커버합니다.</li>
  <li><strong>SVG 최적화</strong>를 클릭합니다. 최적화 전 / 후 바이트 수, 절감 비율, 실제 렌더링 결과를 함께 표시해 시각적 회귀를 즉시 발견할 수 있습니다.</li>
  <li><strong>결과 복사</strong>로 코드베이스에 바로 붙여넣거나, <strong>.svg 다운로드</strong>로 정리된 파일을 저장합니다.</li>
</ol>

<h2>제거되는 항목</h2>
<p>
  SVGO는 에디터 메타데이터, 주석, 숨겨진 요소, 기본 속성 등 렌더링에 영향을 주지 않는 마크업을 제거하고, 경로 데이터를 다시 쓰고, 인접 경로를 통합하며, 색상 표기를 단축합니다. 일반적인 아이콘은 40–70% 줄어들고, 절감의 대부분은 <code>convertPathData</code>, <code>mergePaths</code>, <code>cleanupNumericValues</code>에서 나옵니다.
</p>

<h2>위험한 두 토글</h2>
<p>
  <strong>viewBox 제거</strong>와 <strong>width/height 제거</strong>가 기본 비활성화된 데에는 이유가 있습니다. <code>viewBox</code>가 없는 SVG는 반응형 스케일이 불가능하고, <code>width</code> / <code>height</code>가 없는 SVG는 부모 요소의 렌더링 박스를 상속받습니다 — 다운로드용 아이콘에서는 거의 원하지 않는 결과입니다. 임베딩 컨텍스트를 완전히 제어할 때(예: 부모 <code>&lt;use&gt;</code>가 크기를 지정한 스프라이트 시트)에만 켜세요.
</p>

<h2>언제 최적화하나요</h2>
<ul>
  <li><strong>프로덕션 빌드:</strong> <code>vite-plugin-svgo</code> 같은 도구로 번들러에 SVGO를 통합하세요. 본 도구는 CLI를 띄우지 않고 결과를 빠르게 확인할 때 유용합니다.</li>
  <li><strong>수작업 / 익스포트한 SVG:</strong> Figma, Illustrator, Sketch 모두 에디터 메타데이터와 미참조 <code>id</code> 속성을 대량으로 남기고, SVGO는 이를 깨끗이 제거합니다.</li>
  <li><strong>HTML에 인라인된 SVG:</strong> 마크업이 직접 임베드되는 경우 1 KB마다 첫 페인트가 지연됩니다 — 작을수록 좋습니다.</li>
</ul>

<h2>설계상 단일 파일</h2>
<p>
  본 도구는 한 번에 하나의 SVG만 최적화합니다. 배치 최적화, 사용자 정의 플러그인 작성, CI 통합은 SVGO 명령행 도구에 맡기세요 — <code>npm install -g svgo</code> 후 <code>svgo input.svg -o output.svg</code>로 디렉터리나 glob를 처리할 수 있습니다. 본 도구에서 토글 조합을 결정한 뒤, SVGO 공식 문서에서 대응하는 <code>svgo.config.js</code> 스니펫을 복사해 빌드에 적용하세요.
</p>
