<h2>How to use</h2>
<ol>
  <li>Paste your SVG markup, or switch to the <strong>Upload File</strong> tab and drop a <code>.svg</code> file.</li>
  <li>Toggle the plugins you want. Defaults match SVGO's <em>preset-default</em>; the eight primary toggles cover the most common decisions.</li>
  <li>Click <strong>Optimize SVG</strong>. The before / after preview shows byte counts, the saved percentage, and rendered visuals so you can spot regressions immediately.</li>
  <li>Use <strong>Copy output</strong> for a quick paste into your codebase, or <strong>Download .svg</strong> to save the cleaned file.</li>
</ol>

<h2>What gets removed</h2>
<p>
  SVGO removes editor metadata, comments, hidden elements, default attributes, and other markup that does not affect rendering. It also rewrites path data, merges adjacent paths, and shortens color values. Real-world icons typically shrink by 40–70%, with the bulk of the savings coming from <code>convertPathData</code>, <code>mergePaths</code>, and <code>cleanupNumericValues</code>.
</p>

<h2>The two risky toggles</h2>
<p>
  <strong>Remove viewBox</strong> and <strong>Remove width/height</strong> are off by default for a reason: an SVG without <code>viewBox</code> loses its responsive scaling, and one without <code>width</code>/<code>height</code> attributes inherits the rendering box from its parent — which is rarely what you want for downloadable icons. Turn them on only when you fully control the embedding context (for example, sprite sheets where the parent <code>&lt;use&gt;</code> sets the size).
</p>

<h2>When to optimize</h2>
<ul>
  <li><strong>Production builds:</strong> ship icons through your bundler with a <code>vite-plugin-svgo</code> or similar step. Use this tool to spot-check the output without spinning up a CLI.</li>
  <li><strong>Hand-authored or exported SVGs:</strong> Figma, Illustrator, and Sketch all leave editor metadata and unused <code>id</code> attributes that SVGO strips cleanly.</li>
  <li><strong>Inline SVGs in HTML:</strong> if you embed the markup directly, every kilobyte counts — the document blocks first paint until parsed.</li>
</ul>

<h2>Single-file by design</h2>
<p>
  This tool optimizes one SVG at a time. Batch optimization, custom plugin authoring, and CI pipelines belong to the SVGO command-line tool — install it with <code>npm install -g svgo</code> and run <code>svgo input.svg -o output.svg</code> against folders or globs. Once you find the right toggle combination here, copy the equivalent <code>svgo.config.js</code> snippet from the SVGO docs and ship the same configuration into your build.
</p>
