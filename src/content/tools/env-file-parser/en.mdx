<h2>How to Use</h2>
<ol>
  <li>Paste your <code>.env</code> file contents into the text area.</li>
  <li>Click <strong>Parse</strong> to visualize all key-value pairs in a table.</li>
  <li>Review the <strong>Notes</strong> column — warnings highlight empty values, duplicate keys, or non-standard names; errors flag lines that can't be parsed.</li>
  <li>Click <strong>Export JSON</strong> to download the parsed data as a <code>env.json</code> file.</li>
</ol>

<h2>Example Input</h2>

```
# Database config
DB_HOST=localhost
DB_PORT=5432
DB_NAME=myapp
DB_PASSWORD="s3cr3t"
# API keys
API_KEY=
STRIPE_KEY=sk_live_abc123
# Duplicate example
APP_ENV=development
APP_ENV=production
```

<p>The parser will show <code>DB_PASSWORD</code> with quotes stripped, flag <code>API_KEY</code> as having an empty value, and warn that <code>APP_ENV</code> is a duplicate key.</p>

<h2>Common .env Pitfalls Detected</h2>
<ul>
  <li><strong>Missing = sign</strong> — the line cannot be parsed as a key-value pair.</li>
  <li><strong>Empty value</strong> — the key exists but the value is blank; may cause runtime errors depending on your app.</li>
  <li><strong>Duplicate key</strong> — the same key appears more than once; behavior depends on which dotenv library is used (first or last wins).</li>
  <li><strong>Non-standard key name</strong> — keys should start with a letter or underscore and contain only letters, digits, and underscores.</li>
</ul>
