<h2>使用方法</h2>
<ol>
  <li>将 <code>.env</code> 文件内容粘贴到文本框中。</li>
  <li>点击 <strong>Parse</strong> 按钮，以表格形式查看所有键值对。</li>
  <li>查看 <strong>Notes</strong> 列——警告标注空值、重复键或非标准键名；错误标注无法解析的行。</li>
  <li>点击 <strong>Export JSON</strong> 将解析结果下载为 <code>env.json</code> 文件。</li>
</ol>

<h2>示例输入</h2>
```
# 数据库配置
DB_HOST=localhost
DB_PORT=5432
DB_NAME=myapp
DB_PASSWORD="s3cr3t"
# API 密钥
API_KEY=
STRIPE_KEY=sk_live_abc123
# 重复键示例
APP_ENV=development
APP_ENV=production
```

<p>解析器会去除 <code>DB_PASSWORD</code> 的引号，将 <code>API_KEY</code> 标记为空值，并对重复出现的 <code>APP_ENV</code> 键发出警告。</p>

<h2>常见 .env 问题检测</h2>
<ul>
  <li><strong>缺少等号</strong>——该行无法解析为键值对。</li>
  <li><strong>空值</strong>——键存在但值为空，可能导致运行时错误。</li>
  <li><strong>重复键</strong>——同一键出现多次，取哪个值取决于所使用的 dotenv 库。</li>
  <li><strong>非标准键名</strong>——键名应以字母或下划线开头，仅包含字母、数字和下划线。</li>
</ul>
