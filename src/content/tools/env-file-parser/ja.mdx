<h2>使い方</h2>
<ol>
  <li><code>.env</code> ファイルの内容をテキストエリアに貼り付けます。</li>
  <li><strong>Parse</strong> ボタンをクリックすると、キーと値のペアがテーブル形式で表示されます。</li>
  <li><strong>Notes</strong> 列を確認してください。警告は空の値・重複キー・非標準キー名を示し、エラーは解析不可の行を示します。</li>
  <li><strong>Export JSON</strong> をクリックすると、解析結果を <code>env.json</code> としてダウンロードできます。</li>
</ol>

<h2>入力例</h2>
```
# データベース設定
DB_HOST=localhost
DB_PORT=5432
DB_NAME=myapp
DB_PASSWORD="s3cr3t"
# API キー
API_KEY=
STRIPE_KEY=sk_live_abc123
# 重複キーの例
APP_ENV=development
APP_ENV=production
```

<p>パーサーは <code>DB_PASSWORD</code> のクォートを除去し、<code>API_KEY</code> を空値として警告し、重複する <code>APP_ENV</code> キーに対して警告を表示します。</p>

<h2>検出される一般的な .env の問題</h2>
<ul>
  <li><strong>= 記号がない</strong>——キーと値のペアとして解析できない行。</li>
  <li><strong>空の値</strong>——キーは存在するが値が空。アプリによっては実行時エラーになる可能性があります。</li>
  <li><strong>重複キー</strong>——同じキーが複数回出現。どの値が使われるかは dotenv ライブラリによって異なります。</li>
  <li><strong>非標準キー名</strong>——キー名は英字またはアンダースコアで始まり、英字・数字・アンダースコアのみで構成する必要があります。</li>
</ul>
