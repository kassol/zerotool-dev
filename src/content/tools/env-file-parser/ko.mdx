<h2>사용 방법</h2>
<ol>
  <li><code>.env</code> 파일 내용을 텍스트 영역에 붙여넣습니다.</li>
  <li><strong>Parse</strong> 버튼을 클릭하면 모든 키-값 쌍이 테이블로 표시됩니다.</li>
  <li><strong>Notes</strong> 열을 확인하세요 — 경고는 빈 값, 중복 키, 비표준 키 이름을 나타내고, 오류는 파싱할 수 없는 줄을 나타냅니다.</li>
  <li><strong>Export JSON</strong>을 클릭하면 파싱 결과를 <code>env.json</code> 파일로 다운로드할 수 있습니다.</li>
</ol>

<h2>입력 예시</h2>
```
# 데이터베이스 설정
DB_HOST=localhost
DB_PORT=5432
DB_NAME=myapp
DB_PASSWORD="s3cr3t"
# API 키
API_KEY=
STRIPE_KEY=sk_live_abc123
# 중복 키 예시
APP_ENV=development
APP_ENV=production
```

<p>파서는 <code>DB_PASSWORD</code>의 따옴표를 제거하고, <code>API_KEY</code>를 빈 값으로 경고하며, 중복된 <code>APP_ENV</code> 키에 대해 경고를 표시합니다.</p>

<h2>감지되는 일반적인 .env 오류</h2>
<ul>
  <li><strong>등호 누락</strong> — 키-값 쌍으로 파싱할 수 없는 줄.</li>
  <li><strong>빈 값</strong> — 키는 있지만 값이 비어 있음. 앱에 따라 런타임 오류가 발생할 수 있습니다.</li>
  <li><strong>중복 키</strong> — 동일한 키가 두 번 이상 등장. 어떤 값이 사용될지는 dotenv 라이브러리에 따라 다릅니다.</li>
  <li><strong>비표준 키 이름</strong> — 키 이름은 문자 또는 밑줄로 시작하고 문자, 숫자, 밑줄만 포함해야 합니다.</li>
</ul>
