<h2>사용 방법</h2>
<ol>
  <li>JSONL / NDJSON을 왼쪽 패널에 붙여넣거나 로컬 <code>.jsonl</code>, <code>.ndjson</code>, <code>.json</code>, <code>.txt</code> 파일을 엽니다.</li>
  <li><strong>검증</strong>을 클릭해 각 줄을 검사하고 파싱 오류가 있는 줄 번호를 확인합니다.</li>
  <li><strong>JSONL → JSON</strong>을 클릭해 유효한 레코드로 JSON 배열을 만듭니다.</li>
  <li>오른쪽 패널에 JSON 배열을 붙여넣고 <strong>JSON → JSONL</strong>을 클릭해 compact JSON Lines 출력을 생성합니다.</li>
  <li><strong>.json 저장</strong> 또는 <strong>.jsonl 저장</strong>으로 변환 결과를 저장합니다.</li>
</ol>

<h2>예시</h2>
<p>입력 JSONL:</p>

```json
{"id":1,"event":"signup","user":"alice"}
{"id":2,"event":"purchase","user":"bob","amount":29.99}
```

<p>출력 JSON 배열:</p>

```json
[
  {
    "id": 1,
    "event": "signup",
    "user": "alice"
  },
  {
    "id": 2,
    "event": "purchase",
    "user": "bob",
    "amount": 29.99
  }
]
```

<h2>JSONL과 JSON 배열</h2>
<p>
  JSON 배열은 모든 레코드를 하나의 큰 문서 안에 감쌉니다. JSONL은 각 레코드를 독립적인 JSON 값으로 한 줄에 씁니다.
  이 줄 중심 형식은 로그, 스트리밍 내보내기, 명령줄 처리, 대형 데이터셋에 잘 맞습니다.
</p>

<h2>검증 방식</h2>
<p>
  검증기는 빈 줄을 제외한 각 줄을 <code>JSON.parse()</code>로 파싱합니다. 성공한 줄은 유효 레코드로 집계됩니다.
  실패한 줄은 문제 패널에 표시되며 줄 번호, 파서 메시지, 원본 줄의 짧은 미리보기를 확인할 수 있습니다.
</p>
