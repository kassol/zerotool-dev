<h2>How to Use</h2>
<ol>
  <li>Paste JSONL / NDJSON into the left panel, or open a local <code>.jsonl</code>, <code>.ndjson</code>, <code>.json</code>, or <code>.txt</code> file.</li>
  <li>Click <strong>Validate</strong> to check each line and see exact line numbers for parse errors.</li>
  <li>Click <strong>JSONL → JSON</strong> to build a JSON array from the valid records.</li>
  <li>Paste a JSON array into the right panel and click <strong>JSON → JSONL</strong> to create compact JSON Lines output.</li>
  <li>Use <strong>Download .json</strong> or <strong>Download .jsonl</strong> to save the converted result.</li>
</ol>

<h2>Worked Example</h2>
<p>Input JSONL:</p>

```json
{"id":1,"event":"signup","user":"alice"}
{"id":2,"event":"purchase","user":"bob","amount":29.99}
```

<p>Output JSON array:</p>

```json
[
  {
    "id": 1,
    "event": "signup",
    "user": "alice"
  },
  {
    "id": 2,
    "event": "purchase",
    "user": "bob",
    "amount": 29.99
  }
]
```

<h2>JSONL vs JSON Arrays</h2>
<p>
  A JSON array wraps every record inside one large document. JSONL writes each record as a standalone JSON value on its own line.
  That line-oriented shape works well for logs, streaming exports, command-line processing, and large datasets where each record can be handled independently.
</p>

<h2>Validation Details</h2>
<p>
  The validator parses every non-empty line with <code>JSON.parse()</code>. Successful lines are counted as valid records.
  Failed lines appear in the issue panel with the line number, parser message, and a short preview of the original line.
</p>
