<h2>使用方法</h2>
<ol>
  <li>把 JSONL / NDJSON 粘贴到左侧面板，或打开本地 <code>.jsonl</code>、<code>.ndjson</code>、<code>.json</code>、<code>.txt</code> 文件。</li>
  <li>点击 <strong>校验</strong>，逐行检查内容并看到解析错误所在行号。</li>
  <li>点击 <strong>JSONL → JSON</strong>，把有效记录转换成 JSON 数组。</li>
  <li>把 JSON 数组粘贴到右侧面板，点击 <strong>JSON → JSONL</strong>，生成紧凑的 JSON Lines 输出。</li>
  <li>使用 <strong>下载 .json</strong> 或 <strong>下载 .jsonl</strong> 保存转换结果。</li>
</ol>

<h2>示例</h2>
<p>输入 JSONL：</p>

```json
{"id":1,"event":"signup","user":"alice"}
{"id":2,"event":"purchase","user":"bob","amount":29.99}
```

<p>输出 JSON 数组：</p>

```json
[
  {
    "id": 1,
    "event": "signup",
    "user": "alice"
  },
  {
    "id": 2,
    "event": "purchase",
    "user": "bob",
    "amount": 29.99
  }
]
```

<h2>JSONL 与 JSON 数组的区别</h2>
<p>
  JSON 数组把所有记录包在一个大文档里。JSONL 把每条记录写成独立的一行 JSON 值。
  这种按行组织的格式适合日志、流式导出、命令行处理和大数据集，每条记录都可以独立处理。
</p>

<h2>校验细节</h2>
<p>
  校验器会用 <code>JSON.parse()</code> 解析每一个非空行。解析成功的行会计入有效记录。
  解析失败的行会显示在问题面板中，包含行号、解析消息和原始行预览。
</p>
