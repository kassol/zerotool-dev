<h2>使い方</h2>
<ol>
  <li>JSONL / NDJSON を左パネルに貼り付けるか、ローカルの <code>.jsonl</code>、<code>.ndjson</code>、<code>.json</code>、<code>.txt</code> ファイルを開きます。</li>
  <li><strong>検証</strong> をクリックし、各行をチェックして解析エラーの行番号を確認します。</li>
  <li><strong>JSONL → JSON</strong> をクリックし、有効なレコードから JSON 配列を作成します。</li>
  <li>右パネルに JSON 配列を貼り付け、<strong>JSON → JSONL</strong> をクリックしてコンパクトな JSON Lines を生成します。</li>
  <li><strong>.json を保存</strong> または <strong>.jsonl を保存</strong> で変換結果を保存します。</li>
</ol>

<h2>変換例</h2>
<p>入力 JSONL:</p>

```json
{"id":1,"event":"signup","user":"alice"}
{"id":2,"event":"purchase","user":"bob","amount":29.99}
```

<p>出力 JSON 配列:</p>

```json
[
  {
    "id": 1,
    "event": "signup",
    "user": "alice"
  },
  {
    "id": 2,
    "event": "purchase",
    "user": "bob",
    "amount": 29.99
  }
]
```

<h2>JSONL と JSON 配列</h2>
<p>
  JSON 配列はすべてのレコードを 1 つの大きなドキュメントに包みます。JSONL は各レコードを独立した JSON 値として 1 行に書きます。
  この行指向の形は、ログ、ストリーミングエクスポート、コマンドライン処理、大規模データセットに向いています。
</p>

<h2>検証の仕組み</h2>
<p>
  検証器は空行以外の各行を <code>JSON.parse()</code> で解析します。成功した行は有効レコードとしてカウントされます。
  失敗した行は問題パネルに表示され、行番号、パーサーメッセージ、元の行の短いプレビューを確認できます。
</p>
