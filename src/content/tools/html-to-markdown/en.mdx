faqItems:
  - question: "What HTML elements are supported?"
    answer: "All common HTML elements: h1–h6, p, strong/b, em/i, a, img, code, pre+code (with language class), ul, ol, li, blockquote, hr, table (GFM format), and br. Unknown elements fall back to their text content."
  - question: "How are code blocks handled?"
    answer: "Inline <code> becomes backtick-delimited code. A <pre><code class=\"language-javascript\"> block becomes a fenced code block with the language tag preserved: ```javascript. If no language class is present, a plain ``` fence is used."
  - question: "Are GFM tables supported?"
    answer: "Yes. HTML tables with thead and tbody are converted to GitHub Flavored Markdown (GFM) tables with the standard pipe syntax and a separator row."
  - question: "What happens with invalid or incomplete HTML?"
    answer: "The browser's built-in HTML parser is used for parsing, so it applies the same tolerance as any browser. Malformed HTML is corrected automatically where possible, and conversion proceeds on the resulting DOM."
  - question: "Can I convert a full HTML page?"
    answer: "Yes, but only the content inside <body> will be meaningfully converted. Metadata like <head>, <style>, and <script> tags are stripped automatically by the converter."
  - question: "Is my HTML sent to a server?"
    answer: "No. All conversion happens entirely in your browser using the turndown.js library. Nothing is transmitted over the network."

<h2>Common Use Cases</h2>
<ul>
  <li><strong>CMS migration</strong>: Export WordPress or Drupal content as HTML, then convert it to Markdown for a static site generator like Hugo or Astro.</li>
  <li><strong>README preparation</strong>: Scrape a web page and clean it up as Markdown for a GitHub README or documentation site.</li>
  <li><strong>LLM input</strong>: Convert a web page to Markdown to reduce token count and noise before feeding it to a language model.</li>
  <li><strong>Note-taking</strong>: Convert email HTML or rich-text snippets to clean Markdown for Obsidian, Notion, or similar tools.</li>
</ul>
