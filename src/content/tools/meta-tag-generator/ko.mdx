    answer: "1200x630 px(1.91:1)이 Facebook, LinkedIn, Discord 공통 안전값입니다. Twitter의 summary_large_image는 2:1을 기대하며 1200x600도 무난합니다. 파일은 5 MB 이하의 PNG 또는 JPEG로 유지하세요. 대부분의 플랫폼은 og:image:width와 og:image:height를 존중하여 노출합니다."

<h2>사용 방법</h2>
<ol>
  <li><strong>제목</strong>, <strong>설명</strong>, <strong>대표 URL</strong> — 핵심 필드 3개를 입력합니다.</li>
  <li><strong>og:image URL</strong>을 추가해 소셜 미리보기 이미지를 정확히 노출합니다. 1200x630 PNG / JPEG가 어디서나 잘 작동합니다.</li>
  <li><strong>Twitter 카드 유형</strong>을 선택합니다. 메인 이미지가 있는 글은 <code>summary_large_image</code>, 콤팩트 표시는 <code>summary</code>.</li>
  <li>(선택) <strong>Schema.org @type</strong>을 선택하면 최소한의 JSON-LD 블록을 함께 출력합니다.</li>
  <li>오른쪽 미리보기 탭을 전환해 Google SERP / Facebook / Twitter(X) / Discord에서 어떻게 펼쳐지는지 확인합니다.</li>
  <li>생성된 HTML을 복사해 페이지 <code>&lt;head&gt;</code> 안에 붙여 넣습니다.</li>
</ol>

<h2>생성되는 항목</h2>
<ul>
  <li><strong>기본 SEO 메타</strong> — <code>title</code>, <code>description</code>, <code>canonical</code>, <code>author</code>, <code>keywords</code>, <code>theme-color</code>, <code>viewport</code>, <code>robots</code>.</li>
  <li><strong>Open Graph</strong> — <code>og:type</code>, <code>og:title</code>, <code>og:description</code>, <code>og:url</code>, <code>og:site_name</code>, <code>og:image</code>(width / height / alt 포함), <code>og:locale</code>.</li>
  <li><strong>Twitter 카드</strong> — <code>twitter:card</code>, <code>twitter:site</code>, <code>twitter:creator</code>, <code>twitter:title</code>, <code>twitter:description</code>, <code>twitter:image</code>.</li>
  <li><strong>Schema.org JSON-LD</strong> — 최소한의 <code>WebPage</code>, <code>Article</code>, <code>Product</code>, <code>Organization</code> 블록(name / description / url / image).</li>
</ul>
