    answer: "Open Graph（og:）是 Facebook 提出的社交预览协议；LinkedIn、Slack、Discord、iMessage 等几乎所有平台现在都读它。Twitter（X）则有自己的 twitter:* 标签和 card 类型控制。两者同时存在时，Twitter 优先用自己的标签，缺失字段才回退到 Open Graph，所以两套都写是常规做法。"

    answer: "1200×630 px（1.91:1）是 Facebook、LinkedIn、Discord 通用的安全默认值；Twitter 的 summary_large_image 期望 2:1，1200×600 也能正常显示。文件控制在 5 MB 以内，PNG 或 JPEG 即可。多数平台会读取 og:image:width 与 og:image:height 来分发。"

<h2>使用方法</h2>
<ol>
  <li>填写 <strong>标题</strong>、<strong>描述</strong> 与 <strong>规范 URL</strong>——三个核心字段。</li>
  <li>填入 <strong>og:image URL</strong> 让社交预览图正确显示，1200×630 PNG 或 JPEG 通用。</li>
  <li>选择 <strong>Twitter 卡片类型</strong>：<code>summary_large_image</code> 用于带主图的帖子，<code>summary</code> 用于紧凑卡片。</li>
  <li>（可选）选 <strong>Schema.org @type</strong>，输出一段最小 JSON-LD。</li>
  <li>切换右侧预览的 4 个 tab，验证页面在 Google 搜索、Facebook、Twitter（X）、Discord 的展开效果。</li>
  <li>复制生成的 HTML，粘贴到页面的 <code>&lt;head&gt;</code> 中。</li>
</ol>

<h2>生成内容包含</h2>
<ul>
  <li><strong>基础 SEO meta</strong>——<code>title</code>、<code>description</code>、<code>canonical</code>、<code>author</code>、<code>keywords</code>、<code>theme-color</code>、<code>viewport</code>、<code>robots</code>。</li>
  <li><strong>Open Graph</strong>——<code>og:type</code>、<code>og:title</code>、<code>og:description</code>、<code>og:url</code>、<code>og:site_name</code>、<code>og:image</code>（含 width / height / alt）、<code>og:locale</code>。</li>
  <li><strong>Twitter Card</strong>——<code>twitter:card</code>、<code>twitter:site</code>、<code>twitter:creator</code>、<code>twitter:title</code>、<code>twitter:description</code>、<code>twitter:image</code>。</li>
  <li><strong>Schema.org JSON-LD</strong>——最小的 <code>WebPage</code>、<code>Article</code>、<code>Product</code> 或 <code>Organization</code> 块（name / description / url / image）。</li>
</ul>
