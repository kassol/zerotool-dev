faqItems:
  - question: "서브넷이란 무엇인가요?"
    answer: "서브넷(서브네트워크)은 IP 네트워크의 논리적 세분화입니다. 서브네팅은 대규모 네트워크를 더 작고 관리하기 쉬운 세그먼트로 나누어 성능과 보안을 향상시킵니다."
  - question: "CIDR 표기법이란 무엇인가요?"
    answer: "CIDR(Classless Inter-Domain Routing) 표기법은 IP 주소와 프리픽스 길이를 함께 표현합니다(예: 192.168.1.0/24). 슬래시 뒤의 숫자가 프리픽스 길이로, 네트워크 부분에 할당된 비트 수를 나타냅니다."
  - question: "서브넷 마스크는 어떻게 계산하나요?"
    answer: "서브넷 마스크는 프리픽스 길이에서 도출됩니다. 처음 N 비트를 1로, 나머지를 0으로 설정합니다. /24의 경우 255.255.255.0(24개의 1 뒤에 8개의 0)이 됩니다."
  - question: "와일드카드 마스크란 무엇인가요?"
    answer: "와일드카드 마스크는 서브넷 마스크의 비트 반전입니다. ACL 및 라우팅 프로토콜(예: OSPF)에서 주소의 어떤 비트가 중요한지 지정하는 데 사용됩니다."
  - question: "/24 서브넷의 사용 가능한 호스트 수는?"
    answer: "/24 서브넷에는 256개의 주소가 있습니다. 그 중 두 개가 예약됩니다(네트워크 주소와 브로드캐스트 주소). 따라서 사용 가능한 호스트 주소는 254개입니다."
  - question: "데이터가 서버로 전송되나요?"

<h2>사용 방법</h2>
<ol>
  <li>입력 필드에 CIDR 표기법이 포함된 IP 주소(예: <code>192.168.1.0/24</code>)를 입력하거나, IP 주소만 입력하고 드롭다운에서 프리픽스 길이를 선택합니다.</li>
  <li>결과가 즉시 표시됩니다: 네트워크 주소, 브로드캐스트 주소, 서브넷 마스크, 와일드카드 마스크, 첫/마지막 사용 가능한 호스트, 사용 가능한 호스트 총 수.</li>
  <li><strong>복사</strong>를 클릭하면 모든 결과를 일반 텍스트로 클립보드에 복사할 수 있습니다.</li>
</ol>

<h2>출력 필드 설명</h2>

| 필드 | 설명 |
|------|------|
| 네트워크 주소 | 서브넷의 첫 번째 주소 — 네트워크 자체를 식별 |
| 브로드캐스트 주소 | 마지막 주소 — 서브넷의 모든 호스트에 전송 |
| 서브넷 마스크 | 프리픽스의 점으로 구분된 십진수 형식(예: 255.255.255.0) |
| 와일드카드 마스크 | 서브넷 마스크의 비트 반전 |
| 첫 번째 사용 가능한 호스트 | 네트워크 주소 + 1 |
| 마지막 사용 가능한 호스트 | 브로드캐스트 주소 − 1 |
| 사용 가능한 호스트 수 | 총 주소 수 − 2(네트워크 + 브로드캐스트) |
| CIDR 표기법 | 표준 네트워크/프리픽스 형식 |
