<h2>How to Use</h2>
<h3>Schema → Sample JSON (default mode)</h3>
<ol>
  <li>Paste your <code>.proto</code> schema in the left panel.</li>
  <li>The tool auto-detects all message types and populates the dropdown.</li>
  <li>Select a message type and click <strong>Generate JSON</strong>.</li>
  <li>A sample JSON with default field values appears on the right.</li>
</ol>

<h3>Binary → JSON mode</h3>
<ol>
  <li>Switch to the <strong>Binary → JSON</strong> mode using the toggle.</li>
  <li>Paste your <code>.proto</code> schema in the top textarea.</li>
  <li>Paste your encoded binary data (hex or base64) in the second textarea.</li>
  <li>Select the message type and click <strong>Decode Binary</strong>.</li>
</ol>

## Proto3 Syntax Quick Reference

```proto
syntax = "proto3";

package myapp;

message User {
  string id = 1;
  string name = 2;
  int32 age = 3;
  repeated string roles = 4;
  Address address = 5;
  UserStatus status = 6;
}

message Address {
  string street = 1;
  string city = 2;
  string country_code = 3;
}

enum UserStatus {
  UNKNOWN = 0;
  ACTIVE = 1;
  INACTIVE = 2;
}
```

<h2>Field Types Reference</h2>
<ul>
  <li><strong>Scalar types:</strong> <code>double</code>, <code>float</code>, <code>int32</code>, <code>int64</code>, <code>uint32</code>, <code>uint64</code>, <code>bool</code>, <code>string</code>, <code>bytes</code></li>
  <li><strong>Repeated fields:</strong> prefix any field with <code>repeated</code> — maps to JSON arrays</li>
  <li><strong>Nested messages:</strong> use another message type as a field type</li>
  <li><strong>Enums:</strong> define with <code>enum</code>, first value must be 0</li>
  <li><strong>oneof:</strong> only one field in a group can be set at a time</li>
</ul>
