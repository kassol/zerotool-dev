<h2>使い方</h2>
<h3>Schema → サンプルJSON（デフォルトモード）</h3>
<ol>
  <li>左パネルに<code>.proto</code>スキーマを貼り付けます。</li>
  <li>ツールが全メッセージタイプを自動検出してドロップダウンに表示します。</li>
  <li>メッセージタイプを選択し、<strong>JSONを生成</strong>をクリックします。</li>
  <li>デフォルト値を持つサンプルJSONが右側に表示されます。</li>
</ol>

<h3>バイナリ → JSONモード</h3>
<ol>
  <li>トグルで<strong>バイナリ → JSON</strong>モードに切り替えます。</li>
  <li>上のテキストエリアに<code>.proto</code>スキーマを貼り付けます。</li>
  <li>2番目のテキストエリアにバイナリデータ（hexまたはbase64）を貼り付けます。</li>
  <li>メッセージタイプを選択し、<strong>バイナリをデコード</strong>をクリックします。</li>
</ol>

## Proto3構文クイックリファレンス

```proto
syntax = "proto3";

package myapp;

message User {
  string id = 1;
  string name = 2;
  int32 age = 3;
  repeated string roles = 4;
  Address address = 5;
  UserStatus status = 6;
}

message Address {
  string street = 1;
  string city = 2;
  string country_code = 3;
}

enum UserStatus {
  UNKNOWN = 0;
  ACTIVE = 1;
  INACTIVE = 2;
}
```

<h2>フィールドタイプ一覧</h2>
<ul>
  <li><strong>スカラー型：</strong><code>double</code>、<code>float</code>、<code>int32</code>、<code>int64</code>、<code>uint32</code>、<code>uint64</code>、<code>bool</code>、<code>string</code>、<code>bytes</code></li>
  <li><strong>繰り返しフィールド：</strong>フィールドの前に<code>repeated</code>を付ける — JSON配列に対応</li>
  <li><strong>ネストメッセージ：</strong>別のメッセージタイプをフィールド型として使用</li>
  <li><strong>enum：</strong><code>enum</code>で定義、最初の値は必ず0</li>
  <li><strong>oneof：</strong>グループ内の1つのフィールドのみ同時に設定可能</li>
</ul>

<h2>プライバシー</h2>
<p>処理はすべてクライアント側で行われます。スキーマとバイナリデータはアップロードも保存もされません。</p>
