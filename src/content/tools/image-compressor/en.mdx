<h2>How to Use</h2>
<ol>
  <li>Pick an <strong>output format</strong>: Keep original, JPEG, PNG, or WebP.</li>
  <li>Drag the <strong>Quality</strong> slider — 75 is a balanced default for photographic content.</li>
  <li>Optionally enable <strong>Resize long edge to</strong> and enter a target pixel value such as <code>1920</code>.</li>
  <li>Drop your images on the drop zone, or click to open the file picker.</li>
  <li>Each result card shows the original size, output size, dimensions, and a percentage delta.</li>
  <li>Click <strong>Download</strong> on a card, or <strong>Download All</strong> to save the batch.</li>
</ol>

<h2>Quality vs File Size</h2>
<p>
  Quality controls the lossy compression rate for JPEG and WebP outputs.
  Numbers above 90 produce nearly transparent compression — file size stays high.
  Numbers below 60 introduce visible artefacts on smooth gradients and skin tones.
  Quality 75 is the working default used by most web build tools because it keeps photographs visually clean while cutting file size by 60–80% against an uncompressed JPEG straight from a phone camera.
</p>

<h2>Format Cheat Sheet</h2>
<ul>
  <li><strong>JPEG</strong> — photographs, hero shots, anything with continuous tonal gradients. No transparency.</li>
  <li><strong>PNG</strong> — UI screenshots, diagrams, anything with hard edges or transparency. Lossless.</li>
  <li><strong>WebP</strong> — modern browsers everywhere; usually 25–35% smaller than JPEG or PNG at equal quality. Supports transparency.</li>
  <li><strong>Keep original</strong> — useful when you just want to resize without converting formats.</li>
</ul>

<h2>Resize the Long Edge</h2>
<p>
  Most web layouts max out around 1920 pixels wide. Retina (2× DPR) screens benefit from 2560–3840.
  When you set the long edge limit, the tool scales the shorter edge proportionally so aspect ratio is preserved.
  For thumbnails or social previews, try 1024 or 1440 px on the long edge.
</p>

<h2>Privacy: Nothing Is Uploaded</h2>
<p>
  This tool uses <code>createImageBitmap</code> and the Canvas 2D API to decode and re-encode images entirely in your browser.
  The compressed file is constructed locally by <code>canvas.convertToBlob</code> (or <code>canvas.toBlob</code> as a fallback) and served to you via a temporary blob URL.
  There is no server endpoint involved — try it in airplane mode if you want to confirm.
</p>

<h2>Limits</h2>
<ul>
  <li>Maximum 50 MB per file (mobile browsers run out of memory on much larger inputs).</li>
  <li>Animated GIFs are flattened to the first frame by design — for animated GIF re-encoding, use <code>ffmpeg</code> locally.</li>
  <li>EXIF orientation is auto-applied: photos taken sideways come out the right way up.</li>
</ul>
