<h2>使い方</h2>
<ol>
  <li><strong>出力フォーマット</strong>を選択:元のまま、JPEG、PNG、WebP。</li>
  <li><strong>品質</strong>スライダーを調整。写真コンテンツのデフォルトは 75 が定番。</li>
  <li>必要に応じて<strong>長辺をリサイズ</strong>を有効化し、ピクセル値(例:<code>1920</code>)を入力。</li>
  <li>画像をドロップゾーンに投下するか、クリックでファイル選択。</li>
  <li>各結果カードに元サイズ・出力サイズ・寸法変化・節約パーセントが表示されます。</li>
  <li>カード単位の <strong>ダウンロード</strong>、または <strong>すべてダウンロード</strong> でバッチ保存。</li>
</ol>

<h2>品質とファイルサイズのトレードオフ</h2>
<p>
  品質スライダーは JPEG / WebP 出力の非可逆圧縮率を制御します。
  90 を超えると圧縮の差はほとんど見えず、サイズの減少も限定的です。
  60 を下回ると、滑らかなグラデーションや肌色領域にバンドが現れます。
  多くの Web ビルドツールが採用する 75 は、写真の見た目を保ちつつスマホ原本のサイズを 60〜80% カットできる平衡点です。
</p>

<h2>フォーマットチートシート</h2>
<ul>
  <li><strong>JPEG</strong> — 写真・ヒーロー画像など、連続諧調の画像。透明は非対応。</li>
  <li><strong>PNG</strong> — UI スクリーンショット、ダイアグラム、ハードエッジや透明を含む画像。ロスレス。</li>
  <li><strong>WebP</strong> — 現代ブラウザ全対応。同等品質で JPEG/PNG より 25〜35% 軽い。透明にも対応。</li>
  <li><strong>元のまま</strong> — リサイズだけしてフォーマットは変えたくない場合に。</li>
</ul>

<h2>長辺リサイズの戦略</h2>
<p>
  一般的な Web レイアウトは長辺 1920px が上限。Retina(2× DPR)スクリーン向けには 2560〜3840px が目安。
  長辺を指定すると、ツールがアスペクト比を維持したまま短辺を比例縮小します。
  サムネイルや SNS プレビューには 1024 や 1440px を試してみてください。
</p>

<h2>プライバシー:アップロードゼロ</h2>
<p>
  本ツールは <code>createImageBitmap</code> と Canvas 2D API を使い、ブラウザ内でデコードと再エンコードを完結させます。
  圧縮結果は <code>canvas.convertToBlob</code>(またはフォールバックの <code>canvas.toBlob</code>)でローカル生成され、
  一時的な blob URL でダウンロードできます。サーバーエンドポイントを一切経由しないため、機内モードでも全工程が動きます。
</p>

<h2>制限</h2>
<ul>
  <li>1 ファイル最大 50 MB(モバイルブラウザはそれ以上の入力でメモリ枯渇)。</li>
  <li>アニメ GIF は設計上、最初のフレームに平坦化されます。アニメ再エンコードは <code>ffmpeg</code> をローカルで使ってください。</li>
  <li>EXIF 回転は自動適用:横向きで撮った写真も正位置で圧縮されます。</li>
</ul>
