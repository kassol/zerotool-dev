<h2>使用步骤</h2>
<ol>
  <li>选择 <strong>输出格式</strong>:保持原格式、JPEG、PNG 或 WebP。</li>
  <li>拖动 <strong>质量</strong> 滑块——照片类内容默认 75 是常用平衡点。</li>
  <li>可选启用 <strong>长边缩放至</strong>,填入目标像素(例如 <code>1920</code>)。</li>
  <li>把图片拖到拖拽框,或点击调出文件选择器。</li>
  <li>每张结果卡片会显示原始体积、输出体积、尺寸变化以及节省百分比。</li>
  <li>点单张卡片的 <strong>下载</strong>,或点 <strong>全部下载</strong> 一次性保存整批。</li>
</ol>

<h2>质量与体积的权衡</h2>
<p>
  质量滑块控制 JPEG 和 WebP 输出的有损压缩程度。90 以上压缩几乎不可见,但文件体积下降也有限;60 以下平滑渐变和肤色区域会出现可见的色块。
  大多数 Web 构建工具默认 75,因为它能在视觉上保持照片干净,同时把手机原图的体积砍掉 60–80%。
</p>

<h2>格式速查</h2>
<ul>
  <li><strong>JPEG</strong> ——照片、Banner、连续色调的图像。不支持透明。</li>
  <li><strong>PNG</strong> ——UI 截图、流程图、有硬边或透明的图像。无损。</li>
  <li><strong>WebP</strong> ——现代浏览器都支持,等质量下通常比 JPEG/PNG 小 25–35%。支持透明。</li>
  <li><strong>保持原格式</strong> ——只想缩放尺寸而不换格式时用。</li>
</ul>

<h2>长边缩放策略</h2>
<p>
  常见的 Web 布局宽度上限是 1920px。视网膜屏(2 倍 DPR)适配建议 2560–3840px。
  设置长边后,工具会按比例同步缩短另一边,保证宽高比不变。
  做缩略图或社交分享图,试试 1024 或 1440px。
</p>

<h2>隐私:零上传</h2>
<p>
  本工具用 <code>createImageBitmap</code> 与 Canvas 2D API 在浏览器内解码与重新编码,
  压缩结果由 <code>canvas.convertToBlob</code>(或 fallback 的 <code>canvas.toBlob</code>)在本地生成,
  通过临时 blob URL 提供下载。整个流程没有服务端环节——开飞行模式也能完整跑完。
</p>

<h2>限制</h2>
<ul>
  <li>单文件最大 50 MB(移动端浏览器对更大输入会爆显存)。</li>
  <li>动图 GIF 按设计被压平到第一帧——要重新编码动图请在本地用 <code>ffmpeg</code>。</li>
  <li>自动按 EXIF 旋转图像:横着拍的照片会被翻正再压缩。</li>
</ul>
