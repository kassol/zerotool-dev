<h2>How to Use</h2>
<ol>
  <li>To convert <strong>YAML → JSON</strong>: paste your YAML into the left panel and click <strong>YAML → JSON</strong>.</li>
  <li>To convert <strong>JSON → YAML</strong>: paste your JSON into the right panel and click <strong>JSON → YAML</strong>.</li>
  <li>Use <strong>Ctrl+Enter</strong> (or <strong>Cmd+Enter</strong> on Mac) as a keyboard shortcut.</li>
  <li>Click <strong>Copy</strong> on either panel to copy the result to your clipboard.</li>
</ol>

<h2>Worked Examples</h2>

<h3>Simple object</h3>
<p>Input YAML:</p>

```yaml
name: Alice
age: 30
active: true
```

<p>Output JSON has name, age, and active fields with correct types.</p>

<h3>Nested structure with arrays</h3>
<p>Input YAML:</p>

```yaml
server:
  host: localhost
  port: 8080
  features:
    - auth
    - logging
    - cache
```

<p>Output JSON contains nested server object with features array.</p>

<h2>YAML vs JSON</h2>
<p>
  <strong>YAML</strong> is the dominant format for configuration files — Docker Compose, Kubernetes manifests, GitHub Actions workflows, Ansible playbooks, and Helm charts all use it.
  <strong>JSON</strong> is the standard interchange format for REST APIs and web services.
  Both represent the same data structures. This converter lets you move between them when your tools expect a different format.
</p>
