<h2>사용 방법</h2>
<ol>
  <li><strong>YAML → JSON</strong>: YAML을 왼쪽 패널에 붙여넣고 <strong>YAML → JSON</strong>을 클릭합니다.</li>
  <li><strong>JSON → YAML</strong>: JSON을 오른쪽 패널에 붙여넣고 <strong>JSON → YAML</strong>을 클릭합니다.</li>
  <li><strong>Ctrl+Enter</strong>(Mac은 Cmd+Enter)로 빠르게 변환할 수 있습니다.</li>
  <li>패널의 <strong>복사</strong>를 클릭하여 결과를 클립보드에 복사합니다.</li>
</ol>

<h2>변환 예시</h2>

<h3>간단한 객체</h3>
<p>입력 YAML:</p>

```yaml
name: Alice
age: 30
active: true
```

<p>출력 JSON은 올바른 타입의 name, age, active 필드를 포함합니다.</p>

<h3>중첩 구조와 배열</h3>
<p>입력 YAML:</p>

```yaml
server:
  host: localhost
  port: 8080
  features:
    - auth
    - logging
    - cache
```

<p>출력 JSON은 features 배열을 가진 중첩된 server 객체를 포함합니다.</p>

<h2>YAML과 JSON에 대하여</h2>
<p>
  <strong>YAML</strong>은 설정 파일의 주요 형식입니다 — Docker Compose, Kubernetes 매니페스트, GitHub Actions 워크플로, Ansible 플레이북, Helm 차트에서 널리 사용됩니다.
  <strong>JSON</strong>은 REST API와 웹 서비스의 표준 교환 형식입니다.
  두 형식 모두 동일한 데이터 구조를 표현할 수 있습니다. 도구가 다른 형식을 필요로 할 때 이 변환기를 사용하십시오.
</p>
