<h2>사용 방법</h2>
<ol>
  <li>캡처 패드를 클릭하면 첫 로드 시 자동으로 키보드 포커스를 가져갑니다.</li>
  <li>아무 키나 누르세요. 수정자 키, 펑션 키, 화살표 키 모두 동작합니다.</li>
  <li>event.key, event.code 및 폐지된 keyCode / which / charCode 값을 확인하세요.</li>
  <li>Shift / Ctrl / Alt / Meta를 누르고 있으면 해당 수정자 키 pill이 켜집니다.</li>
  <li>생성된 JS 스니펫을 복사하여 자신의 코드에 그대로 사용할 수 있습니다.</li>
</ol>

<h2>event.key vs event.code vs keyCode</h2>
<p>
  프론트엔드 키보드 버그의 대부분은 이 세 가지 속성을 혼용한 데서 비롯됩니다. 참조표:
</p>

| 속성 | 레이아웃 영향 | 권장 여부 | QWERTY에서 A 누를 때 값 |
|---|---|---|---|
| event.key | 있음(현재 레이아웃 반영) | 권장 — 시맨틱 판정 | "a" 또는 Shift 시 "A" |
| event.code | 없음(물리 위치) | 권장 — 게임 / 단축키 바인딩 | "KeyA" |
| keyCode | 없음 | 비권장 — 폐지됨 | 65 |
| which | 없음 | 비권장 — 폐지됨 | 65 |
| charCode | 없음 | 비권장 — keydown 시 항상 0 | 0 |

<h2>주요 사용 사례</h2>
<ul>
  <li><strong>키보드 핸들러 디버깅</strong> — 실제 event.key 값을 if 조건에 붙여 오타를 잡습니다.</li>
  <li><strong>게임 컨트롤 구현</strong> — event.code를 사용하면 AZERTY 레이아웃에서도 WASD가 작동합니다.</li>
  <li><strong>커맨드 팔레트 단축키</strong> — event.key를 e.ctrlKey / e.metaKey와 조합합니다.</li>
  <li><strong>레거시 코드 감사</strong> — keyCode / which를 읽어 옛 핸들러를 event.key로 이전합니다.</li>
  <li><strong>IME 입력 상태 판별</strong> — e.isComposing = true는 한글/일본어 IME가 조합 중임을 의미하므로 단일 키 핸들러는 즉시 종료해야 합니다.</li>
</ul>

<h2>preventDefault 동작</h2>
<p>
  캡처 패드는 Tab, Space, Backspace, 화살표 키, F1-F12에 대해서만 <code>preventDefault()</code>를 호출합니다 —
  브라우저가 네비게이션/페이지 동작으로 소비하는 키들입니다. Ctrl+W / Cmd+Q 같은 시스템 단축키는
  가로채지 않으며 브라우저 기본 동작이 유지됩니다.
</p>
