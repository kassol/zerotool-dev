<h2>使用方法</h2>
<ol>
  <li>点击捕获区域——首次进入页面时会自动获取键盘焦点。</li>
  <li>按下任意键，修饰键、功能键、方向键均可。</li>
  <li>查看 event.key、event.code 以及已弃用的 keyCode / which / charCode。</li>
  <li>按住 Shift / Ctrl / Alt / Meta 时对应修饰键 pill 会亮起。</li>
  <li>复制生成的 JS 代码片段，直接用于你自己的代码。</li>
</ol>

<h2>event.key、event.code 与 keyCode 对照</h2>
<p>
  前端键盘逻辑 bug 大多来自混用这三个属性。对照表如下：
</p>

| 属性 | 受布局影响 | 推荐使用 | QWERTY 按 A 时的取值 |
|---|---|---|---|
| event.key | 是（反映当前布局） | 推荐——语义判断 | "a" 或按 Shift 时 "A" |
| event.code | 否（物理位置） | 推荐——游戏/快捷键绑定 | "KeyA" |
| keyCode | 否 | 不推荐——已弃用 | 65 |
| which | 否 | 不推荐——已弃用 | 65 |
| charCode | 否 | 不推荐——keydown 时恒为 0 | 0 |

<h2>常见用途</h2>
<ul>
  <li><strong>调试键盘事件处理</strong>——把 event.key 实际值粘到 if 条件里，排查拼写错误。</li>
  <li><strong>实现游戏控制</strong>——优先用 event.code，让 AZERTY 布局也能用 WASD 走位。</li>
  <li><strong>命令面板快捷键</strong>——组合 event.key 与 e.ctrlKey / e.metaKey。</li>
  <li><strong>审计遗留代码</strong>——读出 keyCode / which，把老代码迁移到 event.key。</li>
  <li><strong>识别 IME 输入</strong>——e.isComposing = true 表示中文/日文输入法正在组字，单键处理器应直接返回。</li>
</ul>

<h2>关于 preventDefault</h2>
<p>
  捕获区域仅对 Tab、Space、Backspace、方向键、F1-F12 调用 <code>preventDefault()</code>——这些是浏览器会消费的导航/页面键。
  Ctrl+W / Cmd+Q 等系统级快捷键不拦截，保持浏览器默认行为。
</p>
