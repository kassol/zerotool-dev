<h2>使い方</h2>
<ol>
  <li>キャプチャパッドをクリックすると、初回読み込み時に自動でキーボードフォーカスを取得する。</li>
  <li>任意のキーを押す。修飾キー・ファンクションキー・矢印キーすべて動作する。</li>
  <li>event.key、event.code および非推奨の keyCode / which / charCode を確認。</li>
  <li>Shift / Ctrl / Alt / Meta を押下中は対応する修飾キーピルが点灯する。</li>
  <li>生成された JS スニペットをコピーして自分のコードにそのまま使える。</li>
</ol>

<h2>event.key vs event.code vs keyCode</h2>
<p>
  フロントエンドのキーボードバグの多くはこの3つの混在が原因。リファレンス対応表：
</p>

| プロパティ | レイアウト依存 | 推奨 | QWERTY で A を押した時の値 |
|---|---|---|---|
| event.key | あり（現在のレイアウト反映） | 推奨——セマンティック判定 | "a" または Shift で "A" |
| event.code | なし（物理位置） | 推奨——ゲーム・ショートカット | "KeyA" |
| keyCode | なし | 非推奨 | 65 |
| which | なし | 非推奨 | 65 |
| charCode | なし | 非推奨——keydown 時は常に 0 | 0 |

<h2>よくある用途</h2>
<ul>
  <li><strong>キーボードハンドラのデバッグ</strong>——実際の event.key を if 条件に貼ってタイポを直す。</li>
  <li><strong>ゲームコントロール構築</strong>——event.code を使えば AZERTY でも WASD が機能する。</li>
  <li><strong>コマンドパレット実装</strong>——event.key と e.ctrlKey / e.metaKey を組み合わせる。</li>
  <li><strong>レガシーコード監査</strong>——keyCode / which を読み、旧ハンドラを event.key に移行する。</li>
  <li><strong>IME 入力中の判定</strong>——e.isComposing = true は IME 変換中。単一キーハンドラは早期 return すべき。</li>
</ul>

<h2>preventDefault の挙動</h2>
<p>
  キャプチャパッドは Tab、Space、Backspace、矢印キー、F1-F12 のみ <code>preventDefault()</code> を呼び出す——
  ブラウザがナビゲーション・ページ操作で消費するキー。Ctrl+W / Cmd+Q などのシステムショートカットは
  傍受せずブラウザのデフォルト動作を維持する。
</p>
