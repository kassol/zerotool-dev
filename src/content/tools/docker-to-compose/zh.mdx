<h2>使用方法</h2>
<ol>
  <li>将 <strong>docker run 命令</strong>粘贴到左侧面板（或点击<strong>示例</strong>）。</li>
  <li>点击<strong>转换</strong>。</li>
  <li>在右侧面板查看生成的 <strong>docker-compose.yml</strong>。</li>
  <li>点击<strong>复制</strong>，将内容保存为项目中的 <code>docker-compose.yml</code>。</li>
</ol>

<h2>支持的参数</h2>
<ul>
  <li><code>--name NAME</code> — 服务名称</li>
  <li><code>-p / --publish HOST:CONTAINER</code> — 端口映射</li>
  <li><code>-v / --volume HOST:CONTAINER[:mode]</code> — 卷挂载</li>
  <li><code>-e / --env KEY=VALUE</code> — 环境变量</li>
  <li><code>--network NETWORK</code> — 网络</li>
  <li><code>--restart POLICY</code> — 重启策略（no、always、unless-stopped、on-failure）</li>
  <li><code>--cap-add CAP</code> — 添加 Linux 能力</li>
  <li><code>--memory MEM</code> — 内存限制（如 512m、1g）</li>
  <li><code>--cpus N</code> — CPU 限制</li>
</ul>

<h2>常见使用场景</h2>
<ul>
  <li><strong>迁移：</strong>将一次性 docker run 命令转换为可复现的 compose 配置。</li>
  <li><strong>文档化：</strong>将 README 或文档中的 run 命令转换为 compose 格式供团队使用。</li>
  <li><strong>多服务部署：</strong>分别生成各服务块，合并为完整的 compose 文件。</li>
  <li><strong>CI/CD：</strong>用声明式 compose 文件替代脆弱的 docker run shell 脚本。</li>
</ul>
