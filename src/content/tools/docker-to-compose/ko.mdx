<h2>사용 방법</h2>
<ol>
  <li>왼쪽 패널에 <strong>docker run 명령어</strong>를 붙여넣습니다 (또는 <strong>예시</strong> 클릭).</li>
  <li><strong>변환</strong>을 클릭합니다.</li>
  <li>오른쪽 패널에서 생성된 <strong>docker-compose.yml</strong>을 확인합니다.</li>
  <li><strong>복사</strong>를 클릭하여 <code>docker-compose.yml</code>로 저장합니다.</li>
</ol>

<h2>지원 플래그</h2>
<ul>
  <li><code>--name NAME</code> — 서비스 이름</li>
  <li><code>-p / --publish HOST:CONTAINER</code> — 포트 매핑</li>
  <li><code>-v / --volume HOST:CONTAINER[:mode]</code> — 볼륨 마운트</li>
  <li><code>-e / --env KEY=VALUE</code> — 환경변수</li>
  <li><code>--network NETWORK</code> — 네트워크</li>
  <li><code>--restart POLICY</code> — 재시작 정책 (no, always, unless-stopped, on-failure)</li>
  <li><code>--cap-add CAP</code> — 리눅스 권한 추가</li>
  <li><code>--memory MEM</code> — 메모리 제한 (예: 512m, 1g)</li>
  <li><code>--cpus N</code> — CPU 제한</li>
</ul>

<h2>주요 사용 사례</h2>
<ul>
  <li><strong>마이그레이션:</strong> 일회성 docker run 명령어를 재현 가능한 compose 구성으로 변환.</li>
  <li><strong>문서화:</strong> README의 run 명령어를 팀용 compose 형식으로 변환.</li>
  <li><strong>멀티 서비스:</strong> 각 서비스 블록을 생성하여 하나의 compose 파일로 통합.</li>
  <li><strong>CI/CD:</strong> docker run 쉘 스크립트를 선언적 compose 파일로 대체.</li>
</ul>
