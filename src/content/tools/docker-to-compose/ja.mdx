<h2>使い方</h2>
<ol>
  <li>左パネルに <strong>docker runコマンド</strong> を貼り付けます（または<strong>サンプル</strong>をクリック）。</li>
  <li><strong>変換</strong> をクリックします。</li>
  <li>右パネルで生成された <strong>docker-compose.yml</strong> を確認します。</li>
  <li><strong>コピー</strong> をクリックして <code>docker-compose.yml</code> として保存します。</li>
</ol>

<h2>対応フラグ</h2>
<ul>
  <li><code>--name NAME</code> — サービス名</li>
  <li><code>-p / --publish HOST:CONTAINER</code> — ポートマッピング</li>
  <li><code>-v / --volume HOST:CONTAINER[:mode]</code> — ボリュームマウント</li>
  <li><code>-e / --env KEY=VALUE</code> — 環境変数</li>
  <li><code>--network NETWORK</code> — ネットワーク</li>
  <li><code>--restart POLICY</code> — 再起動ポリシー（no・always・unless-stopped・on-failure）</li>
  <li><code>--cap-add CAP</code> — ケーパビリティ追加</li>
  <li><code>--memory MEM</code> — メモリ制限（例：512m、1g）</li>
  <li><code>--cpus N</code> — CPU制限</li>
</ul>

<h2>主な用途</h2>
<ul>
  <li><strong>移行：</strong>都度実行のdocker runを再現可能なcomposeに変換。</li>
  <li><strong>ドキュメント化：</strong>READMEのrunコマンドをチーム用のcompose形式に変換。</li>
  <li><strong>マルチサービス：</strong>各サービスブロックを個別に生成して1つのcomposeに統合。</li>
  <li><strong>CI/CD：</strong>docker runのシェルスクリプトを宣言型composeファイルに置き換え。</li>
</ul>
