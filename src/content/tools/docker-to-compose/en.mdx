<h2>How to Use</h2>
<ol>
  <li>Paste your <strong>docker run command</strong> into the left panel (or click <strong>Example</strong>).</li>
  <li>Click <strong>Convert</strong>.</li>
  <li>Review the generated <strong>docker-compose.yml</strong> in the right panel.</li>
  <li>Click <strong>Copy</strong> and save it as <code>docker-compose.yml</code> in your project.</li>
</ol>

<h2>Supported Flags</h2>
<ul>
  <li><code>--name NAME</code> — Service name</li>
  <li><code>-p / --publish HOST:CONTAINER</code> — Port mappings</li>
  <li><code>-v / --volume HOST:CONTAINER[:mode]</code> — Volume mounts</li>
  <li><code>-e / --env KEY=VALUE</code> — Environment variables</li>
  <li><code>--network NETWORK</code> — Network</li>
  <li><code>--restart POLICY</code> — Restart policy (no, always, unless-stopped, on-failure)</li>
  <li><code>--cap-add CAP</code> — Capability additions</li>
  <li><code>--memory MEM</code> — Memory limit (e.g. 512m, 1g)</li>
  <li><code>--cpus N</code> — CPU limit</li>
</ul>

<h2>Common Use Cases</h2>
<ul>
  <li><strong>Migration:</strong> Turn one-off docker run commands into reproducible compose configurations.</li>
  <li><strong>Documentation:</strong> Convert run commands from README files or docs into compose format for your team.</li>
  <li><strong>Multi-service setup:</strong> Generate individual service blocks and combine them into a single compose file.</li>
  <li><strong>CI/CD:</strong> Replace fragile shell scripts of docker run commands with a declarative compose file.</li>
</ul>
