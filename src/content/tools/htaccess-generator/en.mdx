<h2>What Each Section Does</h2>
<ul>
  <li><strong>Force HTTPS:</strong> Adds a RewriteRule that issues a 301 redirect from any <code>http://</code> request to <code>https://</code>. Requires <code>mod_rewrite</code>.</li>
  <li><strong>WWW Redirect:</strong> Enforces a single canonical form of your domain — either always with <code>www.</code> or always without it — preventing duplicate-content issues.</li>
  <li><strong>Directory Index:</strong> Sets the files Apache will serve when a directory URL is requested. Listing multiple files defines a fallback order.</li>
  <li><strong>Browser Caching:</strong> Uses <code>mod_expires</code> to tell browsers how long to cache images, CSS/JS, and fonts. Reduces repeated requests and speeds up page loads.</li>
  <li><strong>Security:</strong> Disables directory listing (<code>Options -Indexes</code>), blocks direct access to <code>.htaccess</code> and <code>.env</code>, and optionally adds XSS / clickjacking protection headers.</li>
  <li><strong>Custom Redirect:</strong> Creates a single-path redirect using the Apache <code>Redirect</code> directive. Choose 301 (permanent) or 302 (temporary).</li>
</ul>
