<h2>사용 방법</h2>
<ol>
  <li>체크박스를 사용하여 각 섹션을 켜거나 끕니다.</li>
  <li>각 섹션 내에서 세부 옵션(캐시 기간, www 설정 등)을 구성합니다.</li>
  <li>하단의 <code>.htaccess</code> 출력창이 실시간으로 업데이트됩니다.</li>
  <li><strong>Copy</strong>를 클릭하여 <code>.htaccess</code> 파일에 붙여넣으세요.</li>
</ol>

<h2>각 섹션 설명</h2>
<ul>
  <li><strong>HTTPS 강제:</strong> 모든 <code>http://</code> 요청을 <code>https://</code>로 301 리디렉션하는 RewriteRule을 추가합니다. 서버에서 <code>mod_rewrite</code>가 활성화되어 있어야 합니다.</li>
  <li><strong>WWW 리디렉션:</strong> 도메인 접근 형식을 통일합니다. 항상 <code>www.</code>를 붙이거나 제거하여 중복 콘텐츠 문제를 방지합니다.</li>
  <li><strong>디렉터리 인덱스:</strong> 디렉터리 URL 요청 시 Apache가 반환할 기본 파일을 설정합니다. 여러 파일을 지정하면 우선순위 순서로 처리됩니다.</li>
  <li><strong>브라우저 캐싱:</strong> <code>mod_expires</code>를 사용하여 이미지, CSS/JS, 폰트의 캐시 만료 기간을 브라우저에 전달합니다. 반복 요청을 줄이고 페이지 로딩 속도를 높입니다.</li>
  <li><strong>보안:</strong> 디렉터리 목록을 비활성화(<code>Options -Indexes</code>)하고, <code>.htaccess</code>와 <code>.env</code> 파일에 대한 직접 접근을 차단합니다. XSS 및 클릭재킹 방지 헤더도 선택적으로 추가할 수 있습니다.</li>
  <li><strong>커스텀 리디렉션:</strong> Apache <code>Redirect</code> 디렉티브를 사용하여 특정 경로에 대한 리디렉션을 생성합니다. 301(영구) 또는 302(임시)를 선택할 수 있습니다.</li>
</ul>
