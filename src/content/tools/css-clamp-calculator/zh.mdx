<h2>使用方法</h2>

<ol>
  <li>选择需要做成流体数值的 CSS 属性。</li>
  <li>输入最小与最大视口宽度，单位为像素。</li>
  <li>输入两个视口端点所对应的属性值。</li>
  <li>选择 <code>rem</code> 或 <code>px</code> 输出；使用 rem 时，可设置换算所用的根字体大小。</li>
  <li>查看曲线与采样值，拖动预览滑块，再复制完整声明或单独的 <code>clamp()</code> 数值。</li>
</ol>

<p>默认设置为 320px 到 1440px、16px 到 32px，生成结果如下：</p>

<pre><code>font-size: clamp(1rem, 0.7143rem + 1.4286vw, 2rem);</code></pre>

<h2>生成表达式的含义</h2>

<p><code>clamp(最小值, 首选值, 最大值)</code> 会尽量采用首选值，同时确保结果不越过上下限。工具依据两个“视口宽度—属性值”端点计算直线：</p>

<pre><code>斜率 = (结束值 - 起始值) / (最大视口 - 最小视口)
截距 = 起始值 - 斜率 × 最小视口
首选值 = 截距 + (斜率 × 100)vw</code></pre>

<p>视口小于起点时，属性停在对应端点；位于两个端点之间时，数值线性变化；超过终点后，另一侧边界接管。</p>

<h2>流体排版与间距</h2>

<p>当断点切换会造成明显跳变时，流体数值很实用，例如标题字号、区块内边距、网格间距和内容宽度。对于必须严格固定的尺寸，例如 1px 边框，则通常没有必要使用 clamp。</p>

<p>字号使用 rem 边界，可以保留它与根字体大小的关系。完成后仍应在浏览器 200% 缩放、窄屏和较长翻译文本下测试；公式正确并不等于组件一定不会溢出。</p>

<h2>反向范围与固定值</h2>

<p>结束值可以小于起始值。此时首选表达式会包含负的 vw 项，使属性随视口变宽而减小。工具只会将 <code>clamp()</code> 的上下边界按正确顺序排列，不会改变直线方向。</p>

<p>如果两个端点相同，就不需要流体计算，输出会简化为 <code>1rem</code> 这样的固定长度。负值仅对 <code>margin</code> 和自定义属性开放，因为其他预设属性通常不接受或不适合负长度。</p>
