<h2>How to use the calculator</h2>

<ol>
  <li>Choose the CSS property you want to make fluid.</li>
  <li>Enter the minimum and maximum viewport widths in pixels.</li>
  <li>Enter the property value at each viewport endpoint.</li>
  <li>Choose <code>rem</code> or <code>px</code> output. If you use rem, set the root font size used for conversion.</li>
  <li>Inspect the curve, move the preview slider, and copy either the complete declaration or only the <code>clamp()</code> value.</li>
</ol>

<p>With the defaults—320px to 1440px and 16px to 32px—the result is:</p>

<pre><code>font-size: clamp(1rem, 0.7143rem + 1.4286vw, 2rem);</code></pre>

<h2>What the generated expression means</h2>

<p><code>clamp(minimum, preferred, maximum)</code> asks the browser to use the preferred value while keeping it between the two bounds. The calculator derives the preferred value from the line through your two viewport/value points:</p>

<pre><code>slope = (end value - start value) / (max viewport - min viewport)
intercept = start value - slope × min viewport
preferred = intercept + (slope × 100)vw</code></pre>

<p>Below the minimum viewport, the lower or upper bound holds the value at its endpoint. Between the endpoints, the value changes linearly. Beyond the maximum viewport, the opposite bound takes over.</p>

<h2>Fluid typography and spacing</h2>

<p>Fluid values are useful when a sudden breakpoint would create a visible jump. Typical targets include headings, section padding, grid gaps, and content widths. They are less useful for values that must align to a strict fixed system, such as one-pixel borders.</p>

<p>For font sizes, rem bounds preserve a relationship with the root font size. Still test the finished layout at 200% browser zoom and with longer translated text. A mathematically valid expression can still overflow a real component.</p>

<h2>Reverse and constant ranges</h2>

<p>The end value may be smaller than the start value. In that case the preferred expression uses a negative vw term, so the property decreases as the viewport widens. The calculator sorts only the <code>clamp()</code> bounds; it preserves the direction of the line.</p>

<p>If both endpoint values match, no fluid calculation is needed. The output becomes a single fixed length such as <code>1rem</code>. Negative values are accepted for <code>margin</code> and the custom property, where negative lengths can be meaningful, but rejected for the other presets.</p>
