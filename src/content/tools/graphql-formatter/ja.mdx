<h2>使い方</h2>
<ol>
  <li>GraphQL query、mutation、subscription、SDL スキーマを入力欄に貼り付けるか、<strong>サンプルを読み込む</strong> から例を起点にしてください。</li>
  <li><strong>整形</strong> で美化、<strong>圧縮</strong> で短縮、<strong>検証</strong> で構文のみ確認できます。</li>
  <li>インデントは 2 または 4 スペースを切り替え可能。デフォルト 2 は graphql-js のネイティブ出力と同じです。</li>
  <li><strong>コピー</strong> や <strong>.graphql をダウンロード</strong> で結果を持ち帰り。<kbd>Cmd/Ctrl + Enter</kbd> でキーボードから整形できます。</li>
</ol>

<h2>実例</h2>

<h3>整形前</h3>

```graphql
query GetUser($id:ID!,$withPosts:Boolean!){user(id:$id){id name email posts @include(if:$withPosts){...PostSummary}}} fragment PostSummary on Post{id title publishedAt}
```

<h3>整形後</h3>

```graphql
query GetUser($id: ID!, $withPosts: Boolean!) {
  user(id: $id) {
    id
    name
    email
    posts @include(if: $withPosts) {
      ...PostSummary
    }
  }
}

fragment PostSummary on Post {
  id
  title
  publishedAt
}
```

<h3>SDL スキーマ</h3>

```graphql
type Post {
  id: ID!
  title: String!
  publishedAt: DateTime
  author: User!
}

type Query {
  post(id: ID!): Post
  posts(authorId: ID, limit: Int = 20): [Post!]!
}
```

<h2>なぜ GraphQL を整形するのか</h2>
<p>一貫した GraphQL の整形は、コードレビュー、スキーマ diff、<code>.graphql</code> ファイルの保管に直結します。永続化クエリを使えない環境では minify によって通信量を減らせます。整形されたクエリは PR 上で読みやすく、クエリを単体で検証することで fragment 名のタイプミスをリクエストを送る前に発見できます。</p>

<h2>内部の仕組み</h2>
<p>本ツールはブラウザ内で <code>graphql-js</code>（GraphQL Foundation 管理の公式リファレンス実装）を動かします。<code>parse()</code> がテキストから AST を構築し、最初の構文エラーを行・列で報告。<code>print()</code> が AST を規範的な GraphQL に再シリアライズ。<code>stripIgnoredCharacters()</code> が spec で ignored とされるトークンをすべて除去して圧縮版を返します。ネットワーク往復ゼロ、データは端末内に留まります。</p>
