<h2>使用步骤</h2>
<ol>
  <li>把 GraphQL query、mutation、subscription 或 SDL 架构粘进输入框，或点击 <strong>载入示例</strong> 从样例开始。</li>
  <li>点击 <strong>格式化</strong> 美化、<strong>压缩</strong> 缩短，或 <strong>校验</strong> 仅检查语法。</li>
  <li>在 2 / 4 空格之间切换缩进，默认 2 空格与 graphql-js 原生输出一致。</li>
  <li>点 <strong>复制</strong> 或 <strong>下载 .graphql</strong> 带走结果。按 <kbd>Cmd/Ctrl + Enter</kbd> 可用键盘触发格式化。</li>
</ol>

<h2>实际示例</h2>

<h3>格式化前</h3>

```graphql
query GetUser($id:ID!,$withPosts:Boolean!){user(id:$id){id name email posts @include(if:$withPosts){...PostSummary}}} fragment PostSummary on Post{id title publishedAt}
```

<h3>格式化后</h3>

```graphql
query GetUser($id: ID!, $withPosts: Boolean!) {
  user(id: $id) {
    id
    name
    email
    posts @include(if: $withPosts) {
      ...PostSummary
    }
  }
}

fragment PostSummary on Post {
  id
  title
  publishedAt
}
```

<h3>SDL 架构</h3>

```graphql
type Post {
  id: ID!
  title: String!
  publishedAt: DateTime
  author: User!
}

type Query {
  post(id: ID!): Post
  posts(authorId: ID, limit: Int = 20): [Post!]!
}
```

<h2>为什么要格式化 GraphQL？</h2>
<p>一致的 GraphQL 格式直接影响 code review、schema diff 与 <code>.graphql</code> 文件入库。压缩后的 query 在没有 persisted query 时能减少线上传输体积；美化后的 query 在 PR 里更易扫读。把 query 单独拿出来做校验，可以在不发请求的情况下秒级定位 fragment 拼写错误。</p>

<h2>实现原理</h2>
<p>本工具在浏览器中跑 <code>graphql-js</code>（GraphQL Foundation 维护的参考实现）。<code>parse()</code> 从文本构建 AST 并在首个语法错误处报出行列号；<code>print()</code> 把 AST 序列化为规范 GraphQL；<code>stripIgnoredCharacters()</code> 按 spec 移除所有 ignored token，得到压缩形式。整个过程零网络往返，数据完全留在本地。</p>
