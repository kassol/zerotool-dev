<h2>How to Use</h2>
<ol>
  <li>Check the data fields you need (e.g. Full Name, Email, Phone).</li>
  <li>Set the number of records (1–100, default 10).</li>
  <li>Choose output format: <strong>JSON</strong> or <strong>CSV</strong>.</li>
  <li>Click <strong>Generate</strong>.</li>
  <li>Click <strong>Copy</strong> to copy to clipboard, or <strong>Download</strong> to save the file.</li>
</ol>

<h2>Use Cases</h2>
<p>
  Fake data is essential at every stage of software development. Seed a dev database without
  polluting it with real user records. Mock an API response for frontend development.
  Populate a spreadsheet for QA testing or a UI demo that needs to look realistic.
</p>
<p>
  All generation happens client-side with no dependencies — names, addresses, and other
  values are drawn from curated word lists and combined randomly. UUID generation uses
  the browser's <code>crypto.randomUUID()</code> API for cryptographic quality.
</p>
