<h2>사용 방법</h2>
<ol>
  <li>필요한 데이터 필드에 체크합니다 (예: 전체 이름, 이메일, 전화번호).</li>
  <li>레코드 수를 설정합니다 (1~100, 기본값 10).</li>
  <li>출력 형식을 선택합니다: <strong>JSON</strong> 또는 <strong>CSV</strong>.</li>
  <li><strong>생성</strong>을 클릭합니다.</li>
  <li><strong>복사</strong>로 클립보드에 복사하거나, <strong>다운로드</strong>로 파일을 저장합니다.</li>
</ol>

<h2>활용 사례</h2>
<p>
  가짜 데이터는 소프트웨어 개발의 모든 단계에서 필수적입니다. 실제 사용자 데이터를 오염시키지 않고 개발 데이터베이스를 채우거나, 프론트엔드 개발용 API 응답을 모킹하거나, QA 테스트와 현실적으로 보여야 하는 UI 데모용 스프레드시트를 채울 수 있습니다.
</p>
<p>
  모든 생성은 클라이언트 사이드에서 완결되며 의존성이 없습니다. 이름, 주소 등의 값은 엄선된 단어 목록에서 무작위로 조합됩니다. UUID 생성은 브라우저의 <code>crypto.randomUUID()</code> API를 사용하여 암호화 수준의 무작위성을 보장합니다.
</p>
