<h2>共有前に EXIF を除去すべき理由</h2>

<p>多くのスマートフォンやカメラは、撮影のたびに正確な GPS 座標、カメラのシリアル番号、タイムスタンプを写真に静かに書き込みます。EXIF を取り除かずに掲示板・ブログ・自動除去のないチャットへ送ると、無料ツールだけで自宅・職場・宿泊先を数メートル単位で特定されかねません。</p>

<p>このツールでは、写真がどんな情報を持っているかをすばやく確認し、メタデータを除去したクリーンなコピーをダウンロードできます。すべての処理は今開いているブラウザタブ内で完結します。</p>

<h2>使い方</h2>
<ol>
  <li>JPEG 写真をアップロード領域にドラッグ＆ドロップ、クリックでファイルを選択、または <code>Ctrl + V</code> で貼り付けます。</li>
  <li>解析された EXIF を確認します：カメラ本体・レンズ、露出設定、撮影日時、GPS 座標（Google マップリンク付き）、ファイルを編集したソフトウェア。</li>
  <li><strong>メタデータ除去済み JPEG をダウンロード</strong>をクリックして、EXIF / XMP / IPTC マーカーを除去したコピーを保存します。画像データはバイト単位で同一で、ピクセルは再エンコードされません。</li>
  <li>元のメタデータを残しておきたい場合は、<strong>EXIF JSON をコピー</strong>で解析結果を控えておけます。</li>
</ol>

<h2>除去対象のセグメント</h2>

<p>除去処理は JPEG のマーカーストリームをスキャンし、個人情報を持ちやすいセグメントを取り除きます：</p>

<ul>
  <li><strong>APP1</strong> — EXIF（カメラ、GPS、タイムスタンプ）と XMP（Adobe メタデータ、編集履歴）</li>
  <li><strong>APP13</strong> — Photoshop / IPTC（キャプション、著作権、連絡先）</li>
  <li><strong>APP14</strong> — Adobe カラー変換マーカー</li>
</ul>

<p>ブラウザが画像を正しくデコードするために必要なセグメントはそのまま残します：SOI、JFIF（APP0）、ICC カラープロファイル（APP2）、SOS、圧縮画像データ、EOI。色味は元画像と同じです。</p>

<h2>表示される EXIF フィールド</h2>

<ul>
  <li><strong>カメラ</strong>：メーカー、モデル、レンズモデル — 例：Apple iPhone 15 Pro、iPhone back camera 6.86mm f/1.78</li>
  <li><strong>日時</strong>：撮影日時、最終更新日時</li>
  <li><strong>露出</strong>：絞り（F 値）、シャッタースピード、ISO、焦点距離</li>
  <li><strong>GPS</strong>：緯度、経度、高度、Google マップ直リンク</li>
  <li><strong>ソフトウェア</strong>：書き出し時に使われた OS や編集アプリ</li>
</ul>

<h2>プライバシーに関する補足</h2>

<ul>
  <li>このページは写真をアップロードしません。使用中にブラウザのネットワークタブを開いて確認できます。</li>
  <li>Google マップリンクはローカルで作られ、クリックするまでリクエストは送られません。</li>
  <li>元のメタデータを記録として残したい場合は、<strong>EXIF JSON をコピー</strong>を先に行ってからクリーンなコピーをダウンロードしてください。</li>
</ul>
