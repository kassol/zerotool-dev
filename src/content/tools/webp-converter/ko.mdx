<h2>사용 방법</h2>
<ol>
  <li>변환 방향 선택: <strong>PNG/JPG/GIF → WebP</strong> 또는 <strong>WebP → PNG</strong>.</li>
  <li><strong>품질</strong> 슬라이더 조정 (WebP 출력 시에만) — 85가 권장 기본값입니다.</li>
  <li>이미지를 드롭 영역에 끌어다 놓거나 클릭하여 파일을 선택합니다.</li>
  <li>각 파일에 썸네일, 원본 크기, 변환 후 크기, 절약 비율이 표시됩니다.</li>
  <li>파일 옆의 <strong>다운로드</strong>를 클릭하거나 <strong>모두 다운로드</strong>로 일괄 저장합니다.</li>
</ol>

<h2>WebP 변환이 유용한 경우</h2>
<ul>
  <li><strong>웹 성능 최적화:</strong> 사이트의 JPEG/PNG 에셋을 WebP로 교체해 페이지 용량을 줄이고 Core Web Vitals 점수를 개선합니다.</li>
  <li><strong>이메일 마케팅:</strong> WebP를 지원하는 이메일 클라이언트에서 작은 이미지가 더 빠르게 로드됩니다.</li>
  <li><strong>앱 개발:</strong> 모바일 및 웹 앱에서 에셋 번들 크기를 줄이는 데 활용합니다.</li>
  <li><strong>CDN 비용 절감:</strong> 파일이 작을수록 CDN 청구 대역폭이 줄어듭니다.</li>
</ul>

<h2>WebP vs JPEG vs PNG</h2>
<p>
  WebP는 손실·무손실 압축과 함께 투명도(PNG와 유사)와 애니메이션(GIF와 유사)도 지원합니다.
  사진 콘텐츠의 경우 품질 80의 WebP 손실 압축은 JPEG 품질 90과 비슷한 시각적 결과를 절반 크기로 제공합니다.
  투명 그래픽의 경우 WebP 무손실이 PNG보다 평균 20~30% 작습니다.
</p>
