<h2>使用方法</h2>
<ol>
  <li>选择转换方向：<strong>PNG/JPG/GIF → WebP</strong> 或 <strong>WebP → PNG</strong>。</li>
  <li>调整<strong>质量</strong>滑块（仅限 WebP 输出）—— 85 是推荐默认值。</li>
  <li>将图片拖入拖放区域，或点击选择文件。</li>
  <li>每个文件将显示缩略图、原始大小、转换后大小和节省百分比。</li>
  <li>点击文件旁边的<strong>下载</strong>，或使用<strong>全部下载</strong>批量保存。</li>
</ol>

<h2>何时将图片转换为 WebP</h2>
<ul>
  <li><strong>Web 性能优化：</strong>替换网站上的 JPEG/PNG 资源，减轻页面体积，提升 Core Web Vitals 评分。</li>
  <li><strong>邮件营销：</strong>更小的图片在支持 WebP 的邮件客户端中加载更快。</li>
  <li><strong>应用开发：</strong>在移动端和 Web 应用中使用 WebP 以减小资源包体积。</li>
  <li><strong>降低 CDN 成本：</strong>文件更小意味着 CDN 计费的出口带宽更少。</li>
</ul>

<h2>WebP vs JPEG vs PNG</h2>
<p>
  WebP 同时支持有损和无损压缩，以及透明度（类似 PNG）和动画（类似 GIF）。
  对于摄影内容，质量 80 的 WebP 有损压缩通常能达到 JPEG 质量 90 的视觉效果，文件大小却约为其一半。
  对于带透明度的图形，WebP 无损平均比 PNG 小 20–30%。
</p>
