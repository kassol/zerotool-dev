faqItems:
  - question: "什么是 RFC 6902 JSON Patch？"
    answer: "RFC 6902 定义了一种 JSON 文档格式，用于表达对 JSON 文档的一系列操作。每个操作是一个包含 'op'（add/remove/replace/move/copy/test）、'path'（RFC 6901 JSON Pointer）和可选 'value' 的对象。"
  - question: "我的 JSON 数据会被发送到服务器吗？"
    answer: "不会。比较完全在浏览器中通过 JavaScript 完成，数据不会离开本机。"
  - question: "可以比较大型 JSON 文件吗？"
    answer: "可以，但包含数千个键或深层嵌套数组的超大对象可能需要片刻处理时间。所有计算均在浏览器内完成。"
  - question: "可视化差异是如何工作的？"
    answer: "工具执行深层递归比较。只在「After」中存在的键显示为添加（+）；只在「Before」中存在的键显示为删除（−）；值发生变化的键显示为修改（~），同时展示旧值和新值。"
  - question: "这与文本对比工具有什么不同？"
    answer: "文本对比工具逐行比较纯文本。JSON 差异工具理解 JSON 结构，对值进行语义比较而非文本比较，并输出机器可读的 JSON Patch 操作。"

<h2>使用方法</h2>
<ol>
  <li>将原始 JSON 粘贴到<strong>「修改前」</strong>面板。</li>
  <li>将修改后的 JSON 粘贴到<strong>「修改后」</strong>面板。</li>
  <li>点击<strong>比较</strong>执行差异对比。</li>
  <li>在<strong>可视化差异</strong>中查看人类可读的变更。</li>
  <li>复制 <strong>JSON Patch</strong> 用于你的应用程序或 API。</li>
</ol>

<h2>使用场景</h2>
<ul>
  <li>审查不同版本之间的 API 响应变更。</li>
  <li>为 PATCH HTTP 接口生成 patch 操作。</li>
  <li>审计配置文件变更。</li>
  <li>调试应用中意外的数据变更。</li>
</ul>
