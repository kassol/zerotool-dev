faqItems:
  - question: "What is RFC 6902 JSON Patch?"
    answer: "RFC 6902 defines a JSON document format for expressing a sequence of operations to apply to a JSON document. Each operation is an object with 'op' (add/remove/replace/move/copy/test), 'path' (JSON Pointer per RFC 6901), and optionally 'value'."
  - question: "Is my JSON data sent to any server?"
    answer: "No. The comparison is done entirely in your browser using JavaScript. Nothing leaves your machine."
  - question: "Can I compare large JSON files?"
    answer: "Yes, but very large objects (thousands of keys or deeply nested arrays) may take a moment to process. All computation is in-browser."
  - question: "How does the visual diff work?"
    answer: "The tool performs a deep recursive comparison. Keys present only in 'After' are shown as additions (+). Keys present only in 'Before' are shown as removals (−). Keys with changed values are shown as changes (~) with both the old and new values."
  - question: "How is this different from the Diff Checker tool?"
    answer: "The Diff Checker tool compares plain text line-by-line. This JSON Diff tool understands JSON structure — it compares values semantically, not textually, and outputs machine-readable JSON Patch operations."

<h2>How to Use</h2>
<ol>
  <li>Paste the original JSON into the <strong>Before</strong> panel.</li>
  <li>Paste the updated JSON into the <strong>After</strong> panel.</li>
  <li>Click <strong>Compare</strong> to run the diff.</li>
  <li>Review the <strong>Visual Diff</strong> for human-readable changes.</li>
  <li>Copy the <strong>JSON Patch</strong> to use in your application or API.</li>
</ol>

<h2>Use Cases</h2>
<ul>
  <li>Review API response changes between versions.</li>
  <li>Generate patch operations for PATCH HTTP endpoints.</li>
  <li>Audit configuration file changes.</li>
  <li>Debug unexpected data mutations in your application.</li>
</ul>
