<h2>How to Use</h2>
<ol>
  <li>Paste your HTML — a fragment or a full document with doctype — into the input panel.</li>
  <li>Click <strong>Minify</strong> to strip comments and collapse whitespace, or <strong>Beautify</strong> to re-indent the markup.</li>
  <li>For Beautify, choose 2 spaces, 4 spaces, or Tab from the indent dropdown.</li>
  <li>The status bar shows the savings after a minify (e.g. <code>12.4 KB → 8.1 KB (−34.6%)</code>).</li>
  <li>Click <strong>Copy</strong> on the output panel to grab the result.</li>
</ol>

<h2>Worked Examples</h2>
<h3>Before minify</h3>

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <!-- page metadata -->
    <title>Hello</title>
  </head>
  <body>
    <h1>Hello World</h1>
    <p>Some <strong>bold</strong> text.</p>
  </body>
</html>
```

<h3>After minify</h3>

```html
<!doctype html><html lang="en"><head><title>Hello</title></head><body><h1>Hello World</h1><p>Some <strong>bold</strong> text.</p></body></html>
```

<h2>What Gets Minified</h2>
<ul>
  <li>Whitespace between tags (inter-tag whitespace) is collapsed.</li>
  <li>HTML comments are removed — except IE conditional comments (<code>&lt;!--[if IE]&gt;</code>).</li>
  <li>Boolean attribute values like <code>disabled=""</code> are simplified to <code>disabled</code>.</li>
  <li>Trailing whitespace inside text nodes is trimmed; internal runs collapse to single spaces.</li>
</ul>

<h2>What Stays Verbatim</h2>
<ul>
  <li>Content inside <code>&lt;script&gt;</code>, <code>&lt;style&gt;</code>, <code>&lt;pre&gt;</code>, <code>&lt;textarea&gt;</code>, <code>&lt;code&gt;</code>, <code>&lt;samp&gt;</code>, <code>&lt;kbd&gt;</code>.</li>
  <li>Attribute values, including quoted whitespace and inline event handlers.</li>
  <li>The <code>&lt;!DOCTYPE&gt;</code> declaration.</li>
</ul>

<h2>When to Minify Production HTML</h2>
<p>Most modern build pipelines minify HTML as part of the bundler step (Vite, Webpack, Astro, Next.js). Use this tool when you need a one-off minify — pasting from a CMS export, comparing before/after sizes, or sanity-checking a hand-written template. Beautify is the daily driver: paste minified HTML from production, get a readable copy to debug or diff.</p>
