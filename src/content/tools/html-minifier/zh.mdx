<h2>使用方法</h2>
<ol>
  <li>把 HTML（片段或带 doctype 的完整文档）粘贴到输入框。</li>
  <li>点击 <strong>压缩</strong> 去注释折叠空白，或点击 <strong>美化</strong> 重新缩进。</li>
  <li>美化时可在缩进下拉选择 2 空格、4 空格或 Tab。</li>
  <li>压缩后状态栏会显示节省情况（例如 <code>12.4 KB → 8.1 KB (−34.6%)</code>）。</li>
  <li>点击输出面板的 <strong>复制</strong> 按钮即可取走结果。</li>
</ol>

<h2>示例</h2>
<h3>压缩前</h3>

```html
<!DOCTYPE html>
<html lang="zh">
  <head>
    <!-- 页面元信息 -->
    <title>你好</title>
  </head>
  <body>
    <h1>Hello World</h1>
    <p>一段 <strong>加粗</strong> 文本。</p>
  </body>
</html>
```

<h3>压缩后</h3>

```html
<!doctype html><html lang="zh"><head><title>你好</title></head><body><h1>Hello World</h1><p>一段 <strong>加粗</strong> 文本。</p></body></html>
```

<h2>会被压缩的内容</h2>
<ul>
  <li>标签之间的空白（inter-tag whitespace）会折叠。</li>
  <li>HTML 注释被删除——IE 条件注释（<code>&lt;!--[if IE]&gt;</code>）除外。</li>
  <li><code>disabled=""</code> 这类布尔属性会简化为 <code>disabled</code>。</li>
  <li>文本节点首尾空白被裁掉，内部多个空白折叠为单个空格。</li>
</ul>

<h2>会原样保留的内容</h2>
<ul>
  <li><code>&lt;script&gt;</code>、<code>&lt;style&gt;</code>、<code>&lt;pre&gt;</code>、<code>&lt;textarea&gt;</code>、<code>&lt;code&gt;</code>、<code>&lt;samp&gt;</code>、<code>&lt;kbd&gt;</code> 内的内容。</li>
  <li>属性值，包括带引号的空白和内联事件处理函数。</li>
  <li><code>&lt;!DOCTYPE&gt;</code> 声明。</li>
</ul>

<h2>什么时候手动压缩 HTML</h2>
<p>现代构建链路（Vite、Webpack、Astro、Next.js）通常会在 bundle 阶段自动压缩 HTML。这个工具适合一次性场景——从 CMS 导出的代码、想对比压缩前后体积、或检查手写模板。美化则是日常使用：把生产环境的一行式 HTML 粘进来，拿到可读副本用于调试或 diff。</p>
