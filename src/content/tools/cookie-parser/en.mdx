<h2>How to Use</h2>
<ol>
  <li>Paste a raw Cookie header string (e.g. from browser DevTools) into the text area.</li>
  <li>Click <strong>Parse</strong> to display all cookies in a table with Name, Value, and Flags columns.</li>
  <li>Toggle <strong>URL Decode values</strong> to control whether percent-encoded characters are decoded.</li>
  <li>Switch to the <strong>JSON</strong> tab to see the cookies as a formatted JSON object.</li>
  <li>Click <strong>Copy JSON</strong> to copy the JSON object to your clipboard.</li>
</ol>

<h2>Example Input</h2>

```
session=abc123; user=admin; theme=dark; Path=/; HttpOnly; Secure; SameSite=Lax
```

<p>The parser will show <code>session</code>, <code>user</code>, and <code>theme</code> as cookie rows, with <code>Path=/</code>, <code>HttpOnly</code>, <code>Secure</code>, and <code>SameSite=Lax</code> displayed as flags.</p>

<h2>URL Encoding Example</h2>

```
q=hello%20world; token=abc%3D123
```

<p>With URL Decode enabled: <code>q</code> → <code>hello world</code>, <code>token</code> → <code>abc=123</code>. With it disabled, the raw encoded strings are shown.</p>

<h2>Common Cookie Flags</h2>
<ul>
  <li><strong>HttpOnly</strong> — prevents JavaScript from accessing the cookie via <code>document.cookie</code>, reducing XSS risk.</li>
  <li><strong>Secure</strong> — cookie is only sent over HTTPS connections.</li>
  <li><strong>SameSite</strong> — controls cross-site cookie sending (Strict, Lax, or None).</li>
  <li><strong>Path</strong> — limits the cookie to a specific URL path on the server.</li>
  <li><strong>Domain</strong> — specifies which domains the cookie is sent to.</li>
  <li><strong>Expires</strong> — sets an absolute expiry date for the cookie.</li>
  <li><strong>Max-Age</strong> — sets a relative expiry in seconds from now.</li>
</ul>
