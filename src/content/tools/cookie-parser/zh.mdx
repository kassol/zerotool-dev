<h2>使用方法</h2>
<ol>
  <li>将原始 Cookie 请求头字符串（例如从浏览器开发工具复制）粘贴到文本框中。</li>
  <li>点击<strong>解析</strong>，在表格中查看所有 Cookie 的名称、值和标志列。</li>
  <li>切换 <strong>URL 解码</strong>开关控制是否解码百分号编码字符。</li>
  <li>切换到 <strong>JSON</strong> 选项卡，以格式化 JSON 对象形式查看 Cookie。</li>
  <li>点击<strong>复制 JSON</strong>将 JSON 对象复制到剪贴板。</li>
</ol>

<h2>示例输入</h2>

```
session=abc123; user=admin; theme=dark; Path=/; HttpOnly; Secure; SameSite=Lax
```

<p>解析器将显示 <code>session</code>、<code>user</code> 和 <code>theme</code> 三行 Cookie，并将 <code>Path=/</code>、<code>HttpOnly</code>、<code>Secure</code> 和 <code>SameSite=Lax</code> 显示为标志。</p>

<h2>URL 编码示例</h2>

```
q=hello%20world; token=abc%3D123
```

<p>开启 URL 解码后：<code>q</code> → <code>hello world</code>，<code>token</code> → <code>abc=123</code>。关闭后显示原始编码字符串。</p>

<h2>常见 Cookie 标志说明</h2>
<ul>
  <li><strong>HttpOnly</strong> — 禁止 JavaScript 通过 <code>document.cookie</code> 访问该 Cookie，降低 XSS 风险。</li>
  <li><strong>Secure</strong> — Cookie 仅通过 HTTPS 连接发送。</li>
  <li><strong>SameSite</strong> — 控制跨站 Cookie 发送行为（Strict、Lax 或 None）。</li>
  <li><strong>Path</strong> — 将 Cookie 限制在服务器的特定 URL 路径下。</li>
  <li><strong>Domain</strong> — 指定 Cookie 发送到哪些域名。</li>
  <li><strong>Expires</strong> — 设置 Cookie 的绝对过期日期。</li>
  <li><strong>Max-Age</strong> — 设置从现在起多少秒后过期。</li>
</ul>
