<h2>使い方</h2>
<ol>
  <li>生のCookieヘッダー文字列（例：ブラウザのDevToolsからコピー）をテキストエリアに貼り付けます。</li>
  <li><strong>解析</strong>をクリックして、名前・値・フラグ列を含むテーブルにすべてのCookieを表示します。</li>
  <li><strong>URLデコード</strong>を切り替えてパーセントエンコードされた文字をデコードするかどうかを制御します。</li>
  <li><strong>JSON</strong>タブに切り替えると、整形されたJSONオブジェクトとしてCookieが表示されます。</li>
  <li><strong>JSONコピー</strong>をクリックしてJSONオブジェクトをクリップボードにコピーします。</li>
</ol>

<h2>入力例</h2>

```
session=abc123; user=admin; theme=dark; Path=/; HttpOnly; Secure; SameSite=Lax
```

<p>パーサーは<code>session</code>・<code>user</code>・<code>theme</code>をCookie行として表示し、<code>Path=/</code>・<code>HttpOnly</code>・<code>Secure</code>・<code>SameSite=Lax</code>をフラグとして表示します。</p>

<h2>URLエンコード例</h2>

```
q=hello%20world; token=abc%3D123
```

<p>URLデコード有効時：<code>q</code>→<code>hello world</code>、<code>token</code>→<code>abc=123</code>。無効時は元のエンコード済み文字列が表示されます。</p>

<h2>主なCookieフラグ</h2>
<ul>
  <li><strong>HttpOnly</strong> — JavaScriptが<code>document.cookie</code>経由でCookieにアクセスするのを防ぎ、XSSリスクを軽減します。</li>
  <li><strong>Secure</strong> — HTTPS接続でのみCookieが送信されます。</li>
  <li><strong>SameSite</strong> — クロスサイトCookie送信を制御します（Strict・Lax・None）。</li>
  <li><strong>Path</strong> — CookieをサーバーのURLパスに制限します。</li>
  <li><strong>Domain</strong> — Cookieを送信するドメインを指定します。</li>
  <li><strong>Expires</strong> — Cookieの絶対有効期限日を設定します。</li>
  <li><strong>Max-Age</strong> — 今から何秒後に期限切れになるかを設定します。</li>
</ul>
