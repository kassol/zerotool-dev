<h2>사용 방법</h2>
<ol>
  <li>원시 Cookie 헤더 문자열(예: 브라우저 DevTools에서 복사)을 텍스트 영역에 붙여넣습니다.</li>
  <li><strong>파싱</strong>을 클릭하여 이름, 값, 플래그 열이 있는 표에 모든 쿠키를 표시합니다.</li>
  <li><strong>URL 디코딩</strong>을 토글하여 퍼센트 인코딩된 문자를 디코딩할지 제어합니다.</li>
  <li><strong>JSON</strong> 탭으로 전환하여 형식화된 JSON 객체로 쿠키를 확인합니다.</li>
  <li><strong>JSON 복사</strong>를 클릭하여 JSON 객체를 클립보드에 복사합니다.</li>
</ol>

<h2>입력 예시</h2>

```
session=abc123; user=admin; theme=dark; Path=/; HttpOnly; Secure; SameSite=Lax
```

<p>파서는 <code>session</code>, <code>user</code>, <code>theme</code>을 쿠키 행으로 표시하고 <code>Path=/</code>, <code>HttpOnly</code>, <code>Secure</code>, <code>SameSite=Lax</code>를 플래그로 표시합니다.</p>

<h2>URL 인코딩 예시</h2>

```
q=hello%20world; token=abc%3D123
```

<p>URL 디코딩 활성화 시: <code>q</code> → <code>hello world</code>, <code>token</code> → <code>abc=123</code>. 비활성화 시 원본 인코딩된 문자열이 표시됩니다.</p>

<h2>주요 Cookie 플래그</h2>
<ul>
  <li><strong>HttpOnly</strong> — JavaScript가 <code>document.cookie</code>로 쿠키에 접근하는 것을 방지하여 XSS 위험을 줄입니다.</li>
  <li><strong>Secure</strong> — HTTPS 연결을 통해서만 쿠키가 전송됩니다.</li>
  <li><strong>SameSite</strong> — 크로스사이트 쿠키 전송을 제어합니다(Strict, Lax 또는 None).</li>
  <li><strong>Path</strong> — 쿠키를 서버의 특정 URL 경로로 제한합니다.</li>
  <li><strong>Domain</strong> — 쿠키가 전송되는 도메인을 지정합니다.</li>
  <li><strong>Expires</strong> — 쿠키의 절대 만료 날짜를 설정합니다.</li>
  <li><strong>Max-Age</strong> — 지금부터 몇 초 후에 만료되는지 설정합니다.</li>
</ul>
