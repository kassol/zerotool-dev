<h2>사용 방법</h2>
<ol>
  <li>도메인 입력 — 호스트명, 전체 URL, 끝에 점이 붙은 형태 모두 가능. 자동으로 정규화됩니다.</li>
  <li>레코드 타입 선택. <strong>ALL</strong>은 한 번 클릭으로 A, AAAA, CNAME, MX, TXT, NS, SOA, CAA 8가지를 병렬 조회합니다.</li>
  <li>리졸버 선택: <strong>Cloudflare(1.1.1.1)</strong> 또는 <strong>Google(8.8.8.8)</strong>.</li>
  <li><strong>조회</strong>를 클릭하거나 <kbd>Enter</kbd> 키를 누르세요.</li>
  <li><strong>레코드</strong>, <strong>원본</strong>(dig 스타일 출력), <strong>JSON</strong>(DoH 원본 응답) 탭을 전환하며 확인.</li>
</ol>

<h2>지원하는 레코드 타입</h2>
<ul>
  <li><strong>A</strong> — IPv4 주소.</li>
  <li><strong>AAAA</strong> — IPv6 주소.</li>
  <li><strong>CNAME</strong> — 다른 이름을 가리키는 별칭.</li>
  <li><strong>MX</strong> — 메일 교환 서버와 우선순위.</li>
  <li><strong>TXT</strong> — 임의 텍스트. SPF, DKIM, DMARC, 도메인 소유권 확인, HTTP/3 origin 등을 보관.</li>
  <li><strong>NS</strong> — 영역의 권한 있는 네임서버.</li>
  <li><strong>SOA</strong> — Start of Authority: 기본 NS, 연락처, 시리얼, 리프레시 타이머.</li>
  <li><strong>CAA</strong> — 해당 도메인에 인증서를 발급할 수 있는 CA 지정.</li>
  <li><strong>PTR</strong> — 역방향 DNS. <code>x.y.z.w.in-addr.arpa</code> 형식 이름을 직접 입력하세요. 도구는 일반 PTR 쿼리만 발행합니다.</li>
  <li><strong>SRV</strong> — 서비스 위치. priority / weight / port / target 포함.</li>
</ul>

<h2>왜 DNS-over-HTTPS인가</h2>
<p>
  브라우저는 UDP/53 소켓을 직접 열 수 없어서 브라우저 완결 도구가 클래식 DNS를 말할 수 없습니다.
  DoH는 DNS 메시지를 HTTPS로 감싸므로 일반 <code>fetch()</code> 호출만으로 공개 리졸버에 임의 레코드를
  질의할 수 있습니다. Cloudflare와 Google 모두 JSON 프로파일(Cloudflare는 <code>application/dns-json</code>,
  Google은 네이티브 JSON)을 CORS 허용으로 노출합니다. 본 도구는 그 엔드포인트를 호출하고 응답을 렌더링할 뿐입니다.
</p>

<h2>요약 필 읽는 법</h2>
<ul>
  <li><strong>NOERROR / NXDOMAIN / SERVFAIL</strong> — DNS RCODE. NXDOMAIN은 "도메인이 존재하지 않음"이라는 유효 응답, SERVFAIL은 리졸버가 답할 수 없었음을 의미.</li>
  <li><strong>N개 레코드</strong> — 응답 레코드 수. NOERROR이고 0개면 조회한 이름에 해당 타입 레코드가 없음.</li>
  <li><strong>DNSSEC ✓</strong> — AD 플래그가 설정됨. 리졸버가 서명을 검증함.</li>
  <li><strong>Resolver</strong> — 응답을 제공한 DoH 엔드포인트.</li>
  <li><strong>지연시간</strong> — <code>fetch()</code> 시작부터 JSON 파싱 완료까지의 왕복 시간.</li>
</ul>

<h2>대표적인 활용 사례</h2>
<ul>
  <li>A 또는 CNAME 변경이 특정 리졸버에 반영되었는지 확인.</li>
  <li>신규 발송 도메인 오픈 전 MX, SPF(TXT), DKIM(TXT), DMARC(TXT) 정합성 검사.</li>
  <li>두 공개 리졸버가 본 레코드 비교 — 전파 기간 동안 Cloudflare와 Google이 일치하지 않을 수 있음.</li>
  <li>인증서 발급 전 CAA를 확인해 "CAA mismatch" 실패 회피.</li>
  <li>레지스트라 이전 시 NS 위임이 올바른지 점검.</li>
</ul>

<h2>본 도구의 설계상 경계</h2>
<p>
  단발성 재귀 리졸버 클라이언트입니다. 역방향 IP 조회를 위한 ARPA 자동 변환은 하지 않으며(PTR은 in-addr.arpa 이름을 직접 입력),
  권한 서버에 직접 질의하지 않고(항상 공개 재귀에 질의), 히스토리 스냅샷, WHOIS, 차단 목록, 다중 리졸버 전파 비교도 하지 않습니다.
  전파 맵은 <a href="https://www.whatsmydns.net/">whatsmydns.net</a>, 권한 서버만 조회는 로컬에서
  <code>dig @ns1.example.com</code>을 사용하세요.
</p>
