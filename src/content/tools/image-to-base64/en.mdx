<ol>

  <li>The image preview, file info, and encoded outputs appear instantly.</li>
  <li>Click <strong>Copy</strong> next to <em>Base64 String</em> to copy the raw base64 data.</li>

<h2>Common Use Cases</h2>

<h3>Embed in HTML</h3>
<p>Use the Data URI directly in an <code>&lt;img&gt;</code> tag to inline the image without an HTTP request.</p>

<h3>Embed in CSS</h3>
<p>Use the Data URI as a CSS background-image value for icons and logos.</p>

<h3>JSON payloads</h3>
<p>Use the Base64 string to include image data in JSON API requests where binary data is not supported.</p>

<h2>Base64 Encoding Overhead</h2>
<p>
  Base64 encoding increases the data size by approximately 33% compared to the original binary file.
  For example, a 100 KB PNG becomes ~133 KB when base64-encoded.
  This is acceptable for small images (icons, logos) but is inefficient for large photos.
</p>
