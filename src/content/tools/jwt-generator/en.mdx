faqItems:
  - question: "What is a JWT?"
    answer: "A JSON Web Token (JWT) is a compact, URL-safe token format defined in RFC 7519. It consists of three Base64URL-encoded parts separated by dots: a header (algorithm and type), a payload (claims), and a signature. JWTs are commonly used for authentication and information exchange."
  - question: "Is my secret key sent to any server?"
    answer: "No. All signing is performed using the browser's built-in Web Crypto API (SubtleCrypto). Your header, payload, and secret never leave your browser."
  - question: "Which algorithm should I choose?"
    answer: "HS256 (HMAC-SHA256) is the most widely used algorithm and is suitable for most use cases. HS384 and HS512 use larger hash outputs and are appropriate when extra security margin is required. All three are symmetric algorithms — the same secret is used to both sign and verify."
  - question: "What does 'Secret is empty — token will be unsigned' mean?"
    answer: "If the secret field is left blank, the tool generates a token with an empty signature part. The resulting token is technically invalid for verification but can still be useful for inspecting the encoded header and payload structure."
  - question: "Can I verify the token generated here on jwt.io?"
    answer: "Yes. Copy the generated token and paste it into jwt.io. Enter the same secret and select the matching algorithm to verify the signature."
  - question: "What is the Base64 secret format option?"
    answer: "Some services provide secrets pre-encoded in Base64 (or Base64URL). Select the Base64 option when your secret is already in that form — the tool will decode it before signing, producing the correct signature."
