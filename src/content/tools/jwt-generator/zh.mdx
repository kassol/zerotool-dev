faqItems:
  - question: "什么是 JWT？"
    answer: "JSON Web Token（JWT）是 RFC 7519 定义的一种紧凑型 URL 安全令牌格式。它由三个 Base64URL 编码的部分组成，以点号分隔：Header（算法和类型）、Payload（声明）和 Signature（签名）。JWT 常用于身份认证和信息交换。"
  - question: "我的密钥会发送到服务器吗？"
    answer: "不会。所有签名操作均使用浏览器内置的 Web Crypto API（SubtleCrypto）完成。你的 Header、Payload 和密钥不会离开浏览器。"
  - question: "应该选择哪种算法？"
    answer: "HS256（HMAC-SHA256）是最广泛使用的算法，适合大多数场景。HS384 和 HS512 输出更大的哈希值，适用于需要更高安全余量的场景。三者均为对称算法——使用相同的密钥进行签名和验证。"
  - question: "「密钥为空 — Token 将不带签名」是什么意思？"
    answer: "如果密钥字段为空，工具会生成一个签名部分为空的 Token。该 Token 在验证时技术上无效，但可用于检查 Header 和 Payload 的编码结构。"
  - question: "可以在 jwt.io 上验证这里生成的 Token 吗？"
    answer: "可以。复制生成的 Token 粘贴到 jwt.io，输入相同的密钥并选择对应算法即可验证签名。"
  - question: "Base64 密钥格式选项是什么？"
    answer: "某些服务提供预先经过 Base64（或 Base64URL）编码的密钥。当你的密钥已是这种形式时，选择 Base64 选项——工具会在签名前先解码，从而生成正确的签名。"

<h2>使用方法</h2>
<ol>
  <li>选择签名<strong>算法</strong>（HS256、HS384 或 HS512）。</li>
  <li>如需修改 <strong>Header</strong> JSON，<code>alg</code> 字段会随算法切换自动同步。</li>
  <li>在 <strong>Payload</strong> JSON 中填写你的声明（Claims）。</li>
  <li>输入<strong>密钥</strong>（UTF-8 或 Base64 格式）。</li>
  <li>从结果框中复制生成的 Token。</li>
</ol>

<h2>常见使用场景</h2>
<ul>
  <li><strong>本地开发：</strong>无需启动后端，快速生成 Token 用于测试 API 接口或认证流程。</li>
  <li><strong>调试：</strong>了解 Header 和 Payload JSON 是如何被编码进 JWT 的。</li>
  <li><strong>文档编写：</strong>为 API 文档或 README 生成示例 Token。</li>
  <li><strong>学习：</strong>理解 JWT 的结构和签名原理。</li>
</ul>
