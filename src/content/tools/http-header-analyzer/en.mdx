<h2>How to Use</h2>
<ol>
  <li>Paste raw HTTP headers into the text area. You can include the status line or method line as the first line.</li>
  <li>Click <strong>Analyze</strong>. The tool detects request vs response automatically and groups headers by category.</li>
  <li>Switch between <strong>Categorized</strong>, <strong>Raw</strong>, and <strong>JSON</strong> tabs to see different views.</li>
  <li>Read inline hints on security headers — warnings for misconfiguration, notes for hardening opportunities.</li>
  <li>Use <strong>Copy JSON</strong> to export the parsed result for further automation.</li>
</ol>

<h2>Example: Response with security headers</h2>

```
HTTP/1.1 200 OK
Content-Type: application/json; charset=utf-8
Strict-Transport-Security: max-age=63072000; includeSubDomains; preload
Content-Security-Policy: default-src 'self'; script-src 'self'
X-Content-Type-Options: nosniff
Referrer-Policy: strict-origin-when-cross-origin
Cache-Control: no-store
Set-Cookie: session=abc123; HttpOnly; Secure; SameSite=Strict; Path=/
```

<p>The analyzer groups Strict-Transport-Security, Content-Security-Policy, X-Content-Type-Options, and Referrer-Policy under <strong>Security</strong>. Cache-Control lands in <strong>Caching</strong>, Content-Type in <strong>Content</strong>, and Set-Cookie in <strong>Cookie</strong>. Each card includes a one-line description and any hints — for example, a Set-Cookie without HttpOnly triggers a warning.</p>

<h2>Example: CORS preflight</h2>

```
OPTIONS /api/v1/orders HTTP/1.1
Host: api.example.com
Origin: https://app.example.com
Access-Control-Request-Method: POST
Access-Control-Request-Headers: Authorization, Content-Type
```

<p>This is recognized as a request. The Access-Control-Request-* pair indicates a preflight; the analyzer surfaces the Origin and lists the requested method and headers. Use it alongside the response side to debug CORS handshakes end-to-end.</p>

<h2>Categories at a glance</h2>
<ul>
  <li><strong>Status</strong> — the first line (status line or method line).</li>
  <li><strong>Security</strong> — HSTS, CSP, X-Frame-Options, COOP/COEP/CORP, Permissions-Policy, fetch metadata.</li>
  <li><strong>Caching</strong> — Cache-Control, ETag, Vary, Expires, Last-Modified, conditional headers.</li>
  <li><strong>Content</strong> — Content-Type, Content-Encoding, Accept-\*, Content-Disposition.</li>
  <li><strong>CORS</strong> — Access-Control-\* and Origin.</li>
  <li><strong>Auth</strong> — Authorization, WWW-Authenticate, Proxy-Authenticate.</li>
  <li><strong>Cookie</strong> — Cookie and Set-Cookie.</li>
  <li><strong>Transport / Range / Proxy / General / Custom</strong> — the rest, with X-\* unknown headers in Custom.</li>
</ul>

<h2>Compliance hints</h2>
<p>Hints are conservative — they highlight obvious misconfiguration, not subjective style. Examples:</p>
<ul>
  <li>HSTS max-age below 1 year, or missing includeSubDomains / preload directives.</li>
  <li>CSP containing <code>'unsafe-inline'</code> or <code>'unsafe-eval'</code>.</li>
  <li>Set-Cookie without HttpOnly or Secure flags.</li>
  <li>Access-Control-Allow-Credentials: true paired with a wildcard origin (spec-illegal).</li>
  <li>Cache-Control combining no-store with max-age (dead weight).</li>
</ul>
