<h2>使い方</h2>
<ol>
  <li>生のHTTPヘッダーをテキストエリアに貼り付けます。最初の行にステータス行またはメソッド行を含めてください。</li>
  <li><strong>分析</strong>をクリック。ツールがリクエスト/レスポンスを自動判別し、カテゴリ別にグルーピングして表示します。</li>
  <li><strong>カテゴリ別</strong>、<strong>生データ</strong>、<strong>JSON</strong>タブで表示を切り替えます。</li>
  <li>セキュリティヘッダーのインラインヒントを確認 — 設定ミスは警告、強化機会は注意として表示されます。</li>
  <li><strong>JSONコピー</strong>で解析結果をエクスポートし、後続の自動化処理に利用できます。</li>
</ol>

<h2>例: セキュリティヘッダー付きレスポンス</h2>

```
HTTP/1.1 200 OK
Content-Type: application/json; charset=utf-8
Strict-Transport-Security: max-age=63072000; includeSubDomains; preload
Content-Security-Policy: default-src 'self'; script-src 'self'
X-Content-Type-Options: nosniff
Referrer-Policy: strict-origin-when-cross-origin
Cache-Control: no-store
Set-Cookie: session=abc123; HttpOnly; Secure; SameSite=Strict; Path=/
```

<p>アナライザーはStrict-Transport-Security、Content-Security-Policy、X-Content-Type-Options、Referrer-Policyを<strong>セキュリティ</strong>に分類します。Cache-Controlは<strong>キャッシュ</strong>、Content-Typeは<strong>コンテンツ</strong>、Set-Cookieは<strong>Cookie</strong>に入ります。各カードには一行解説とヒント(例: HttpOnlyなしのSet-Cookieは警告表示)が付きます。</p>

<h2>例: CORSプリフライト</h2>

```
OPTIONS /api/v1/orders HTTP/1.1
Host: api.example.com
Origin: https://app.example.com
Access-Control-Request-Method: POST
Access-Control-Request-Headers: Authorization, Content-Type
```

<p>これはリクエストとして認識されます。Access-Control-Request-*ペアはプリフライトの印。アナライザーはOriginを強調し、要求メソッドとヘッダーをリスト化します。レスポンス側と組み合わせれば、CORSハンドシェイクを端から端まで追跡できます。</p>

<h2>カテゴリ一覧</h2>
<ul>
  <li><strong>ステータス</strong> — 最初の行(ステータス行またはメソッド行)。</li>
  <li><strong>セキュリティ</strong> — HSTS、CSP、X-Frame-Options、COOP/COEP/CORP、Permissions-Policy、Fetch metadata。</li>
  <li><strong>キャッシュ</strong> — Cache-Control、ETag、Vary、Expires、Last-Modified、条件付きリクエストヘッダー。</li>
  <li><strong>コンテンツ</strong> — Content-Type、Content-Encoding、Accept-\*、Content-Disposition。</li>
  <li><strong>CORS</strong> — Access-Control-\*とOrigin。</li>
  <li><strong>認証</strong> — Authorization、WWW-Authenticate、Proxy-Authenticate。</li>
  <li><strong>Cookie</strong> — CookieとSet-Cookie。</li>
  <li><strong>転送 / レンジ / プロキシ / 一般 / カスタム</strong> — その他のカテゴリ。X-\*未知ヘッダーはカスタムに入ります。</li>
</ul>

<h2>合規ヒントの基準</h2>
<p>ヒントは保守的で、明らかな設定ミスのみを指摘し、主観的なスタイル評価はしません。例:</p>
<ul>
  <li>HSTS max-ageが1年未満、またはincludeSubDomains / preload不足。</li>
  <li>CSPに<code>'unsafe-inline'</code>または<code>'unsafe-eval'</code>が含まれる。</li>
  <li>Set-CookieにHttpOnlyまたはSecureが欠落。</li>
  <li>Access-Control-Allow-Credentials: trueとワイルドカードOriginの組み合わせ(仕様違反)。</li>
  <li>Cache-Controlにno-storeとmax-ageが共存(max-age実質無効)。</li>
</ul>
