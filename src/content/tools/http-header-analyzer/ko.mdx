<h2>사용 방법</h2>
<ol>
  <li>원시 HTTP 헤더를 텍스트 영역에 붙여넣습니다. 첫 줄은 상태 줄 또는 메서드 줄이 될 수 있습니다.</li>
  <li><strong>분석</strong>을 클릭하세요. 도구가 요청 / 응답을 자동 감지하고 카테고리별로 그룹화합니다.</li>
  <li><strong>분류별</strong>, <strong>원본</strong>, <strong>JSON</strong> 탭을 전환하며 보기를 바꿉니다.</li>
  <li>보안 헤더 카드의 인라인 힌트를 확인하세요 — 잘못된 구성은 경고, 강화 권장은 참고로 표시됩니다.</li>
  <li><strong>JSON 복사</strong>로 분석 결과를 내보내 후속 자동화에 활용합니다.</li>
</ol>

<h2>예시: 보안 헤더가 포함된 응답</h2>

```
HTTP/1.1 200 OK
Content-Type: application/json; charset=utf-8
Strict-Transport-Security: max-age=63072000; includeSubDomains; preload
Content-Security-Policy: default-src 'self'; script-src 'self'
X-Content-Type-Options: nosniff
Referrer-Policy: strict-origin-when-cross-origin
Cache-Control: no-store
Set-Cookie: session=abc123; HttpOnly; Secure; SameSite=Strict; Path=/
```

<p>분석기는 Strict-Transport-Security, Content-Security-Policy, X-Content-Type-Options, Referrer-Policy를 <strong>보안</strong>으로 분류합니다. Cache-Control은 <strong>캐싱</strong>, Content-Type은 <strong>콘텐츠</strong>, Set-Cookie는 <strong>쿠키</strong>로 분류됩니다. 각 카드에는 한 줄 설명과 힌트(예: HttpOnly 없는 Set-Cookie는 경고)가 포함됩니다.</p>

<h2>예시: CORS 프리플라이트</h2>

```
OPTIONS /api/v1/orders HTTP/1.1
Host: api.example.com
Origin: https://app.example.com
Access-Control-Request-Method: POST
Access-Control-Request-Headers: Authorization, Content-Type
```

<p>요청으로 인식됩니다. Access-Control-Request-* 쌍은 프리플라이트 표식입니다. 분석기는 Origin을 강조하고 요청된 메서드와 헤더를 나열합니다. 응답 측과 함께 사용하면 CORS 핸드셰이크를 끝에서 끝까지 추적할 수 있습니다.</p>

<h2>카테고리 한눈에 보기</h2>
<ul>
  <li><strong>상태</strong> — 첫 줄(상태 줄 또는 메서드 줄).</li>
  <li><strong>보안</strong> — HSTS, CSP, X-Frame-Options, COOP/COEP/CORP, Permissions-Policy, Fetch metadata.</li>
  <li><strong>캐싱</strong> — Cache-Control, ETag, Vary, Expires, Last-Modified, 조건부 요청 헤더.</li>
  <li><strong>콘텐츠</strong> — Content-Type, Content-Encoding, Accept-\*, Content-Disposition.</li>
  <li><strong>CORS</strong> — Access-Control-\*와 Origin.</li>
  <li><strong>인증</strong> — Authorization, WWW-Authenticate, Proxy-Authenticate.</li>
  <li><strong>쿠키</strong> — Cookie와 Set-Cookie.</li>
  <li><strong>전송 / 범위 / 프록시 / 일반 / 사용자 정의</strong> — 나머지 카테고리. X-\* 미지 헤더는 사용자 정의로 분류됩니다.</li>
</ul>

<h2>컴플라이언스 힌트 기준</h2>
<p>힌트는 보수적으로, 명백한 설정 오류만 표시하며 주관적 스타일 평가는 하지 않습니다. 예:</p>
<ul>
  <li>HSTS max-age가 1년 미만이거나 includeSubDomains / preload 누락.</li>
  <li>CSP에 <code>'unsafe-inline'</code> 또는 <code>'unsafe-eval'</code> 포함.</li>
  <li>Set-Cookie에 HttpOnly 또는 Secure 누락.</li>
  <li>Access-Control-Allow-Credentials: true와 와일드카드 Origin 조합(스펙 위반).</li>
  <li>Cache-Control에 no-store와 max-age 공존(max-age 사실상 무효).</li>
</ul>
