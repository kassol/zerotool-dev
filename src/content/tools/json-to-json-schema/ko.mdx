<h2>사용 방법</h2>
<ol>
  <li>왼쪽 <strong>JSON 입력</strong> 창에 JSON을 붙여넣습니다.</li>
  <li>오른쪽 <strong>JSON Schema</strong> 창이 자동으로 업데이트됩니다.</li>
  <li><strong>복사</strong>를 클릭하여 스키마를 클립보드에 복사합니다.</li>
  <li>필요에 따라 스키마를 조정합니다 — 제약 추가, 설명 추가, 필수 필드 변경.</li>
</ol>

<h2>출력 이해하기</h2>
<p>
  생성된 스키마에는 항상 draft-07을 가리키는 <code>$schema</code>가 포함됩니다.
  객체 속성은 JSON 키에서 추론되고, <code>required</code> 배열에는 null이 아닌 모든 키가 나열됩니다.
  배열의 <code>items</code>는 실제 요소에서 타입을 추론합니다.
</p>

<h2>생성 후 개선 포인트</h2>
<p>
  생성된 스키마는 출발점입니다. 일반적인 개선 사항으로는 필드에 <code>description</code> 추가,
  <code>minLength</code>나 <code>pattern</code>으로 문자열 제약 강화,
  선택적 속성을 <code>required</code>에서 제거,
  알려진 값 집합에 <code>enum</code> 추가 등이 있습니다.
</p>
