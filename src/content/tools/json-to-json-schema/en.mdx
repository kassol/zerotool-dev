<h2>How to Use</h2>
<ol>
  <li>Paste your JSON into the <strong>JSON Input</strong> pane on the left.</li>
  <li>The <strong>JSON Schema</strong> pane on the right updates automatically.</li>
  <li>Click <strong>Copy</strong> to copy the schema to your clipboard.</li>
  <li>Refine the schema as needed — add constraints, descriptions, or adjust required fields.</li>
</ol>

<h2>Understanding the Output</h2>
<p>
  The generated schema always includes <code>$schema</code> pointing to draft-07.
  Object properties are inferred from the JSON keys.
  The <code>required</code> array lists every non-null key found.
  Array items use an inferred schema based on the elements present.
</p>

<h2>Next Steps After Generation</h2>
<p>
  A generated schema is a starting point. Common refinements include: adding
  <code>description</code> fields for documentation, tightening string constraints
  with <code>minLength</code> or <code>pattern</code>, marking optional properties by
  removing them from <code>required</code>, and adding <code>enum</code> for fields
  with a known set of values.
</p>
