<h2>How to Use</h2>

<ol>
  <li>Paste your Markdown into the input area.</li>
  <li>Pick the <strong>Anchor style</strong> matching your target platform — GitHub / Hugo, GitLab, Jekyll, or Bitbucket.</li>
  <li>Adjust <strong>Min</strong> and <strong>Max</strong> heading levels to control which headings appear in the TOC.</li>
  <li>Toggle <strong>Include H1</strong> off if your document title is rendered separately.</li>
  <li>Copy the generated TOC, or enable <strong>Marker mode</strong> to get the full Markdown with the TOC inserted between <code>&lt;!-- toc --&gt;</code> markers.</li>
</ol>

<h2>Anchor Styles</h2>

<p>The tool ships with four anchor styles. Pick the one that matches your renderer:</p>

<ul>
  <li><strong>GitHub / Hugo</strong> — lowercase, strip punctuation, hyphenate spaces, keep Unicode letters. Hugo (since v0.62 with goldmark) defaults to the same rules.</li>
  <li><strong>GitLab</strong> — same as GitHub plus consecutive hyphens collapsed into one, matching GitLab Flavored Markdown.</li>
  <li><strong>Jekyll (kramdown classic)</strong> — strips non-ASCII characters before slugifying. Use this only if your Jekyll site has not switched to a Unicode-friendly slugger.</li>
  <li><strong>Bitbucket</strong> — every anchor is prefixed with <code>markdown-</code> and duplicates use <code>_N</code> instead of <code>-N</code>.</li>
</ul>

<h2>Marker Mode</h2>

<p>
  Marker mode keeps your TOC fresh without touching the rest of the document.
  Wrap the spot you want the TOC to live with two HTML comments:
</p>

<pre><code>&lt;!-- toc --&gt;
&lt;!-- /toc --&gt;</code></pre>

<p>
  Run the generator any time and the block between the two comments is replaced
  with a new TOC. If your document has no markers yet, the tool inserts a fresh
  block right before the first heading and you can commit the result.
</p>

<h2>Worked Examples</h2>

<h3>Simple TOC, GitHub style</h3>

<p>Input:</p>

```markdown
# API Reference

## Authentication

### OAuth Flow

## Endpoints

### GET /users
### POST /users
```

<p>Output (GitHub anchors, 2-space indent):</p>

```markdown
- [API Reference](#api-reference)
  - [Authentication](#authentication)
    - [OAuth Flow](#oauth-flow)
  - [Endpoints](#endpoints)
    - [GET /users](#get-users)
    - [POST /users](#post-users)
```

<h3>Duplicate headings</h3>

<p>Two sections named <code>## Examples</code> become <code>#examples</code> and <code>#examples-1</code> on GitHub, GitLab, and Jekyll, or <code>#markdown-examples</code> and <code>#markdown-examples_1</code> on Bitbucket.</p>

<h3>Skipping H1</h3>

<p>If your static site renders the H1 from frontmatter, turn off <strong>Include H1</strong> so the TOC starts at the first H2 and indentation flattens by one level.</p>
