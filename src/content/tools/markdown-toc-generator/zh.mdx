<h2>使用步骤</h2>

<ol>
  <li>把 Markdown 粘贴到左侧输入区。</li>
  <li>在 <strong>锚点风格</strong> 里选目标平台 — GitHub / Hugo、GitLab、Jekyll 或 Bitbucket。</li>
  <li>用 <strong>最小标题级别</strong> 与 <strong>最大标题级别</strong> 圈定要进 TOC 的层级范围。</li>
  <li>如果文档标题由模板单独渲染，关掉 <strong>包含 H1</strong>。</li>
  <li>直接复制生成的目录；或者打开 <strong>标记模式</strong>，得到带 <code>&lt;!-- toc --&gt;</code> 标记的完整 Markdown。</li>
</ol>

<h2>四种锚点风格</h2>

<ul>
  <li><strong>GitHub / Hugo</strong> — 转小写、去标点、空格转连字符、保留 Unicode。Hugo 自 v0.62 起的 goldmark 渲染器默认与 GitHub 一致。</li>
  <li><strong>GitLab</strong> — 同 GitHub，但把连续多个连字符合并为一个，匹配 GitLab Flavored Markdown。</li>
  <li><strong>Jekyll（kramdown 经典）</strong> — 先剥掉所有非 ASCII 字符再生成 slug。仅在 Jekyll 站没有切换到支持 Unicode 的 slugger 时使用。</li>
  <li><strong>Bitbucket</strong> — 每个锚点都带 <code>markdown-</code> 前缀，重复项用 <code>_N</code> 而非 <code>-N</code>。</li>
</ul>

<h2>标记模式</h2>

<p>
  标记模式让 TOC 永远新鲜，又不动原文其它部分。
  在你想放目录的位置写两条 HTML 注释：
</p>

<pre><code>&lt;!-- toc --&gt;
&lt;!-- /toc --&gt;</code></pre>

<p>
  之后随时跑一遍生成器，两条注释之间的内容会被替换为新 TOC。
  如果文档还没有标记，工具会在第一个标题之前自动插入一段标记块，
  复制粘贴回去就能 commit。
</p>

<h2>实例</h2>

<h3>简单目录，GitHub 风格</h3>

<p>输入：</p>

```markdown
# API 参考

## 鉴权

### OAuth 流程

## 接口

### GET /users
### POST /users
```

<p>输出（GitHub 锚点，缩进 2 空格）：</p>

```markdown
- [API 参考](#api-参考)
  - [鉴权](#鉴权)
    - [OAuth 流程](#oauth-流程)
  - [接口](#接口)
    - [GET /users](#get-users)
    - [POST /users](#post-users)
```

<h3>重复标题</h3>

<p>两个 <code>## 示例</code> 在 GitHub / GitLab / Jekyll 下分别变成 <code>#示例</code> 和 <code>#示例-1</code>；Bitbucket 下变成 <code>#markdown-示例</code> 和 <code>#markdown-示例_1</code>。</p>

<h3>跳过 H1</h3>

<p>如果你的静态站从 frontmatter 渲染 H1，关掉 <strong>包含 H1</strong>，TOC 会从第一个 H2 开始，整体缩进降一级。</p>
