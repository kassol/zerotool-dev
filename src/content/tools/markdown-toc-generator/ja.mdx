<h2>使い方</h2>

<ol>
  <li>左側の入力エリアに Markdown を貼り付けます。</li>
  <li><strong>アンカー形式</strong> でターゲット — GitHub / Hugo、GitLab、Jekyll、Bitbucket — を選択。</li>
  <li><strong>最小見出しレベル</strong> と <strong>最大見出しレベル</strong> で目次に含める階層を絞り込みます。</li>
  <li>ドキュメントタイトルがテンプレートで別途レンダリングされる場合は、<strong>H1 を含む</strong> をオフにしてください。</li>
  <li>生成された目次をコピーするか、<strong>マーカーモード</strong> をオンにして <code>&lt;!-- toc --&gt;</code> マーカー入りの完全な Markdown を取得します。</li>
</ol>

<h2>4 つのアンカー形式</h2>

<ul>
  <li><strong>GitHub / Hugo</strong> — 小文字化、句読点除去、空白をハイフンに、Unicode は保持。Hugo は v0.62 以降 goldmark で GitHub と同じ規則を採用。</li>
  <li><strong>GitLab</strong> — GitHub と同じ規則に加え、連続するハイフンを 1 つに圧縮。GitLab Flavored Markdown のレンダリングと一致。</li>
  <li><strong>Jekyll（kramdown classic）</strong> — 非 ASCII を先に除去してから slug 化。Unicode 対応の slugger に切り替えていない Jekyll サイト向け。</li>
  <li><strong>Bitbucket</strong> — 全てのアンカーが <code>markdown-</code> プレフィックス付き、重複は <code>-N</code> ではなく <code>_N</code>。</li>
</ul>

<h2>マーカーモード</h2>

<p>
  マーカーモードは目次を常に最新に保ちつつ、本文の他の部分には触れません。
  目次を置きたい場所に 2 つの HTML コメントを書いておきます：
</p>

<pre><code>&lt;!-- toc --&gt;
&lt;!-- /toc --&gt;</code></pre>

<p>
  あとはジェネレーターを実行するたび、2 つのコメント間の内容が新しい TOC に置換されます。
  マーカーがまだ無い場合は、最初の見出しの直前に新しいマーカー付きブロックが挿入されるので、
  コピーして commit するだけで運用に乗ります。
</p>

<h2>実例</h2>

<h3>シンプルな目次（GitHub 形式）</h3>

<p>入力：</p>

```markdown
# API リファレンス

## 認証

### OAuth フロー

## エンドポイント

### GET /users
### POST /users
```

<p>出力（GitHub アンカー、インデント半角スペース 2）：</p>

```markdown
- [API リファレンス](#api-リファレンス)
  - [認証](#認証)
    - [OAuth フロー](#oauth-フロー)
  - [エンドポイント](#エンドポイント)
    - [GET /users](#get-users)
    - [POST /users](#post-users)
```

<h3>重複した見出し</h3>

<p>2 つの <code>## 例</code> は GitHub / GitLab / Jekyll では <code>#例</code> と <code>#例-1</code>、Bitbucket では <code>#markdown-例</code> と <code>#markdown-例_1</code> になります。</p>

<h3>H1 を除外する</h3>

<p>静的サイトが H1 を frontmatter から描画している場合は <strong>H1 を含む</strong> をオフにします。TOC は最初の H2 から始まり、インデントが 1 段階浅くなります。</p>
