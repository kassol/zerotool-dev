<h2>사용 방법</h2>

<ol>
  <li>왼쪽 입력 영역에 Markdown을 붙여넣습니다.</li>
  <li><strong>앵커 스타일</strong>에서 대상 플랫폼 — GitHub / Hugo, GitLab, Jekyll, Bitbucket — 을 선택합니다.</li>
  <li><strong>최소 제목 레벨</strong>과 <strong>최대 제목 레벨</strong>로 목차에 포함될 제목 범위를 좁힙니다.</li>
  <li>문서 제목이 템플릿에서 따로 렌더링된다면 <strong>H1 포함</strong>을 끕니다.</li>
  <li>생성된 목차를 그대로 복사하거나, <strong>마커 모드</strong>를 켜서 <code>&lt;!-- toc --&gt;</code> 마커가 포함된 전체 Markdown을 가져옵니다.</li>
</ol>

<h2>네 가지 앵커 스타일</h2>

<ul>
  <li><strong>GitHub / Hugo</strong> — 소문자 변환, 구두점 제거, 공백을 하이픈으로, Unicode 유지. Hugo는 v0.62부터 goldmark 렌더러에서 GitHub과 동일한 규칙을 사용합니다.</li>
  <li><strong>GitLab</strong> — GitHub과 같은 규칙에 추가로 연속된 하이픈을 하나로 합칩니다. GitLab Flavored Markdown의 렌더링과 일치합니다.</li>
  <li><strong>Jekyll(kramdown classic)</strong> — 비 ASCII 문자를 먼저 제거한 뒤 slug를 만듭니다. Unicode 친화 slugger로 전환하지 않은 Jekyll 사이트 전용.</li>
  <li><strong>Bitbucket</strong> — 모든 앵커에 <code>markdown-</code> 접두사가 붙고, 중복은 <code>-N</code>이 아니라 <code>_N</code>으로 표시됩니다.</li>
</ul>

<h2>마커 모드</h2>

<p>
  마커 모드는 목차를 항상 최신으로 유지하면서, 문서의 나머지 부분은 건드리지 않습니다.
  목차가 위치할 자리에 두 개의 HTML 주석을 미리 적어 두세요:
</p>

<pre><code>&lt;!-- toc --&gt;
&lt;!-- /toc --&gt;</code></pre>

<p>
  이후 생성기를 실행할 때마다 두 주석 사이의 내용이 새로운 TOC로 교체됩니다.
  아직 마커가 없다면 첫 제목 바로 앞에 새 마커 블록이 삽입되므로,
  결과를 복사해 commit하기만 하면 됩니다.
</p>

<h2>예제</h2>

<h3>간단한 목차, GitHub 스타일</h3>

<p>입력:</p>

```markdown
# API 레퍼런스

## 인증

### OAuth 흐름

## 엔드포인트

### GET /users
### POST /users
```

<p>출력 (GitHub 앵커, 들여쓰기 공백 2칸):</p>

```markdown
- [API 레퍼런스](#api-레퍼런스)
  - [인증](#인증)
    - [OAuth 흐름](#oauth-흐름)
  - [엔드포인트](#엔드포인트)
    - [GET /users](#get-users)
    - [POST /users](#post-users)
```

<h3>중복된 제목</h3>

<p><code>## 예제</code>가 두 번 등장하면 GitHub / GitLab / Jekyll에서는 <code>#예제</code>와 <code>#예제-1</code>, Bitbucket에서는 <code>#markdown-예제</code>와 <code>#markdown-예제_1</code>이 됩니다.</p>

<h3>H1 제외</h3>

<p>정적 사이트가 frontmatter에서 H1을 렌더링한다면 <strong>H1 포함</strong>을 끄세요. TOC는 첫 H2부터 시작하고 들여쓰기가 한 단계씩 얕아집니다.</p>
