<h2>How to Use</h2>
<ol>
  <li>Click <strong>Pick from screen</strong>. Your cursor turns into a magnifier — click any pixel to sample.</li>
  <li>Or use the <strong>Native picker</strong> button for a built-in color dialog (works in every browser).</li>
  <li>The HEX, RGB, and HSL fields update in sync. Edit any field to recompute the other two.</li>
  <li>Click <strong>Copy</strong> next to any format to send it to your clipboard.</li>
  <li>Your last 8 sampled colors stay in the <strong>Recent colors</strong> strip — click a swatch to reload it.</li>
</ol>

<h2>EyeDropper API in 60 seconds</h2>
<p>
  <code>EyeDropper</code> is a browser-native Web API that lets a page sample pixels from anywhere on the user's screen. The page calls <code>new EyeDropper().open()</code>; the browser opens a system-level magnifier; on click, the promise resolves to an object whose <code>sRGBHex</code> property holds the picked color.
</p>
<p>
  The API is privacy-safe: the page never gets pixel data, only the single selected color. The user must initiate every pick with a click — there is no headless or scripted sampling.
</p>

<h2>When to use which mode</h2>
<ul>
  <li><strong>Pick from screen</strong> — when the color lives in another app, a screenshot, or a video frame.</li>
  <li><strong>Native picker</strong> — when you want to author a fresh color or use a familiar HSV wheel.</li>
  <li><strong>Type into a field</strong> — when you have an exact value (e.g. from a design spec) and need its sibling formats.</li>
</ul>
