<h2>How to Use</h2>
<ol>
  <li>Paste a JSON array of objects into the left panel. The tool converts automatically.</li>
  <li>Choose a <strong>delimiter</strong>: comma (default), semicolon, or tab.</li>
  <li>Toggle <strong>Flatten nested objects</strong> to expand nested keys into dot-path columns (e.g. <code>address.city</code>).</li>
  <li>Toggle <strong>Include header</strong> to add or suppress the header row.</li>
  <li>Click <strong>Copy</strong> to copy the CSV or <strong>Download .csv</strong> to save the file.</li>
</ol>

<h2>Worked Examples</h2>

<h3>Basic conversion</h3>
<p>Input JSON:</p>
<pre><code>{`[{"id":1,"name":"Alice"},{"id":2,"name":"Bob"}]`}</code></pre>
<p>Output CSV:</p>
<pre><code>{`id,name\n1,Alice\n2,Bob`}</code></pre>

<h3>Nested object flattening</h3>
<p>Input JSON:</p>
<pre><code>{`[{"user":{"name":"Alice","age":30}}]`}</code></pre>
<p>Output CSV (flatten on):</p>
<pre><code>{`user.name,user.age\nAlice,30`}</code></pre>

<h3>Fields with commas</h3>
<p>A field value containing a comma is automatically quoted:</p>
<pre><code>{`name,description\nWidget,"Small, round widget"`}</code></pre>

<h2>RFC 4180 Compliance</h2>
<p>
  This tool follows the RFC 4180 CSV standard. Fields containing the delimiter, double quotes, or line breaks
  are enclosed in double quotes. A literal double-quote is escaped as <code>""</code>.
</p>
