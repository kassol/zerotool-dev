<ol>
  <li>IBAN을 입력란에 붙여넣습니다. 공백, 하이픈, 대소문자는 무시되고 자동으로 정규화됩니다.</li>
  <li>입력 도중 실시간으로 검증이 실행됩니다. <strong>검증 결과</strong> 카드에서 유효성, 국가, 체크 디지트, 기대 길이가 표시됩니다.</li>
  <li><strong>BBAN 필드</strong> 카드는 SWIFT IBAN Registry에 따라 은행 코드, 지점 코드, 계좌 번호, 그리고 해당 국가의 국가 체크 디지트를 분해해 보여줍니다.</li>
  <li><strong>형식</strong> 카드에서는 인쇄용(4자리 단위 공백), 기계 판독용(공백 없음), 또는 BBAN만 한 번의 클릭으로 복사할 수 있습니다.</li>
  <li><strong>예시 불러오기</strong>를 사용하면 특정 국가의 공개 테스트 IBAN을 즉시 채워 비교할 수 있습니다.</li>
</ol>

<h2>IBAN이란</h2>
<p>
  IBAN(International Bank Account Number)은 ISO 13616으로 정의되고 SWIFT가 관리합니다.
  하나의 문자열에 네 가지를 담습니다 — 2자리 ISO 국가 코드, 2자리 체크 디지트,
  그리고 국가별로 정의된 BBAN(Basic Bank Account Number). BBAN은 고정 레이아웃으로
  은행·지점·계좌 식별자를 담습니다. 길이는 국가마다 고정 — 노르웨이 15자리, 영국 22자리,
  세인트루시아 34자리 — 이며 각 위치마다 숫자, 알파벳, 또는 영숫자 혼합 등의 문자 종류가 정의되어 있습니다.
</p>

<h2>mod-97 검사 방식</h2>
<p>의도적으로 단순하게 설계된 알고리즘으로, 브라우저에서 마이크로초 단위로 실행됩니다.</p>
<ol>
  <li>공백을 제거하고 IBAN을 대문자로 만듭니다.</li>
  <li>처음 네 글자(국가 코드 + 체크 디지트)를 끝으로 옮깁니다.</li>
  <li>각 알파벳을 두 자리 숫자로 치환합니다: <code>A → 10</code>, <code>B → 11</code>, …, <code>Z → 35</code>.</li>
  <li>결과 문자열을 하나의 큰 정수로 보고 <code>n mod 97</code>을 계산합니다.</li>
  <li>나머지가 <strong>1</strong>이면 체크섬이 정확합니다.</li>
</ol>
<p>
  많은 국가의 BBAN에 알파벳이 포함되므로(영국 은행 코드, 이탈리아의 첫자리 체크 등),
  알파벳 → 숫자 매핑은 부수적 처리가 아닌 핵심 과정입니다. ZeroTool은 가능한 경우 네이티브 <code>BigInt</code>를 사용하고,
  구형 브라우저에서는 청크 단위 모듈로 연산으로 대체합니다.
</p>

<h2>BBAN 분해에서 보이는 항목</h2>
<ul>
  <li><strong>은행 코드</strong> — 국가 내 발급 은행을 식별. 문자 종류는 국가별로 다름 — 알파벳(영국·네덜란드·아일랜드), 숫자(독일·프랑스·이탈리아), 영숫자 혼합(스위스·알바니아).</li>
  <li><strong>지점 코드</strong> — 많은 국가가 BBAN 안에 지점/sort-code를 별도 영역으로 보유 (영국 sort code, 프랑스 <em>code guichet</em>, 스페인 <em>oficina</em>).</li>
  <li><strong>계좌 번호</strong> — 은행 내부 계좌 식별자.</li>
  <li><strong>국가 체크 디지트</strong> — 프랑스, 이탈리아, 벨기에, 헝가리 등은 IBAN mod-97과 별도로 BBAN 안에 국가 체크 디지트를 내장합니다. 표시는 하지만 본 도구는 국가별 알고리즘을 별도로 검증하지 않습니다.</li>
  <li><strong>계좌 유형 / 보유자 표시 / 통화 코드</strong> — 일부 국가 고유 필드 (브라질은 계좌 유형 + 보유자, 모리셔스·세이셸은 통화 코드, 불가리아는 계좌 유형).</li>
</ul>

<h2>일반적인 사용 사례</h2>
<ul>
  <li>SEPA 송금이나 전신환 발행 전에 자리바꿈 오타 잡기.</li>
  <li>결제 폼에서 사용자가 입력한 IBAN을 정리 — 유료 IBAN API 이전 단계에서 클라이언트 측 mod-97 검사가 가장 저렴한 1차 필터.</li>
  <li>결제 게이트웨이(Stripe, Adyen, Mollie, Braintree)가 제공하는 테스트 IBAN의 국가 레이아웃 확인.</li>
  <li>OCR 송장 인식 결과 디버깅 — 가장 흔한 OCR 오류(0/O, 1/I, 5/S)는 모두 mod-97을 깨므로 즉시 감지됩니다.</li>
  <li>익숙하지 않은 송장을 결제하기 전 은행 코드가 예상과 일치하는지 확인.</li>
</ul>
