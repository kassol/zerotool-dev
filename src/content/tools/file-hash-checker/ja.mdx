<h2>使い方</h2>
<ol>
  <li>アップロード領域へファイルをドロップする、またはクリックして選択します。</li>
  <li>SHA-256 はデフォルトで有効。必要に応じて SHA-1 / SHA-384 / SHA-512 / MD5 を追加で選択します。</li>
  <li><strong>「ハッシュを計算」</strong>をクリックすると、結果が行ごとに表示されコピーボタンが付きます。</li>
  <li>ダウンロード検証のときは、配布元が公開している期待ハッシュを下部の欄に貼り付けます。大文字小文字を無視して照合し、どの結果と一致するかを表示します。</li>
</ol>

<h2>主なユースケース</h2>
<ul>
  <li><strong>ダウンロード検証</strong>：ISO・インストーラー・リリース成果物が公開された SHA-256 と一致するかを確認。</li>
  <li><strong>ビルド成果物の整合性</strong>：CI のアーティファクト、S3、企業プロキシ経由の転送で壊れていないかを確認。</li>
  <li><strong>端末間の同一性確認</strong>：2 つのコピーがバイト単位で同じかを素早く判定。</li>
  <li><strong>レガシー互換</strong>：多くのディストリビューションは MD5 / SHA-1 を併記しており、本ツールなら一度に確認できます。</li>
</ul>

<h2>計算のしくみ</h2>
<p>
  すべてローカルで完結します。<code>File.arrayBuffer()</code> でファイルを <code>ArrayBuffer</code>
  として一括読み込みし、SHA-1 / SHA-256 / SHA-384 / SHA-512 は
  <code>crypto.subtle.digest()</code> を通じてブラウザのネイティブ暗号実装（多くの場合
  ハードウェアアクセラレーション付き）で計算します。MD5 は Web Crypto には実装されていないため、
  ページに同梱したパブリックドメインの MD5 ルーチンを同じバイト列に対して走らせます。
  得られた 16 進ダイジェストはそのまま結果行に描画され、どこにも送信されません。
</p>

<h2>SHA-256 とレガシーアルゴリズムの使い分け</h2>
<p>
  SHA-256 をデフォルトにしているのには理由があります。SHA-2 ファミリーで実用的な衝突攻撃は
  発見されておらず、現代のツールチェーン（rustup、cargo、npm、Docker レジストリ、
  GitHub Releases、Linux の各種ミラー）はすべて SHA-256 を公開しています。SHA-1 は
  2017 年の SHAttered 攻撃で衝突可能性が示され、MD5 の衝突生成はとっくに実用化済みです。
  配布元が MD5 / SHA-1 だけを公開している場合は、それを「ダウンロードが壊れていないかの目印」
  と捉えてください。「真正性の保証」ではありません。
</p>

<h2>設計上の境界</h2>
<p>
  本ツールは単一ファイルを一括でハッシュ化することに特化しています。ストリーミングや署名検証、
  GPG / PGP 鍵操作、外部通信は一切行いません。巨大なアーカイブにはコマンドラインを使ってください：
  <code>sha256sum</code>（Linux）、<code>shasum -a 256</code>（macOS）、
  <code>certutil -hashfile</code>（Windows）。真正性の検証には配布元の署名鍵と
  <code>gpg --verify</code> や <code>cosign verify</code> を組み合わせるのが正攻法です。
</p>
