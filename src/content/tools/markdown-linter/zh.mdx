<h2>使用方法</h2>
<ol>
  <li>在左侧面板中粘贴或输入 Markdown 内容。</li>
  <li>右侧实时显示检查结果，无需点击按钮。</li>
  <li>点击任意结果可跳转到编辑器中对应的行。</li>
  <li>修复问题后，结果会立即更新。</li>
</ol>

<h2>检测内容</h2>
<p>
  检查器应用完整的 <a href="https://github.com/DavidAnson/markdownlint/blob/main/doc/Rules.md" target="_blank" rel="noopener">markdownlint 规则</a>，包括：
</p>
<ul>
  <li><strong>MD001</strong> — 标题层级每次只能增加一级</li>
  <li><strong>MD009</strong> — 行尾不应有空格</li>
  <li><strong>MD010</strong> — 不使用制表符（应使用空格）</li>
  <li><strong>MD012</strong> — 不连续出现多个空行</li>
  <li><strong>MD022</strong> — 标题前后应有空行</li>
  <li><strong>MD025</strong> — 文档中只能有一个顶级标题</li>
  <li><strong>MD031</strong> — 代码围栏前后应有空行</li>
  <li><strong>MD034</strong> — 不使用裸 URL，应使用链接语法</li>
  <li><strong>MD041</strong> — 第一行应为顶级标题</li>
</ul>

<h2>隐私</h2>
<p>所有处理均在客户端完成，Markdown 内容不会被上传或存储。</p>
