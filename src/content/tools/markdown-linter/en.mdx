<h2>How to Use</h2>
<ol>
  <li>Paste or type Markdown in the left pane.</li>
  <li>Lint results appear in real time on the right — no button press needed.</li>
  <li>Click any result to jump to that line in the editor.</li>
  <li>Fix the flagged issues and watch the results update instantly.</li>
</ol>

<h2>What Gets Checked</h2>
<p>
  The linter applies the full set of <a href="https://github.com/DavidAnson/markdownlint/blob/main/doc/Rules.md" target="_blank" rel="noopener">markdownlint rules</a>, including:
</p>
<ul>
  <li><strong>MD001</strong> — Heading levels should only increment by one level at a time</li>
  <li><strong>MD009</strong> — No trailing spaces at end of lines</li>
  <li><strong>MD010</strong> — No hard tabs (use spaces instead)</li>
  <li><strong>MD012</strong> — No multiple consecutive blank lines</li>
  <li><strong>MD022</strong> — Headings should be surrounded by blank lines</li>
  <li><strong>MD025</strong> — Only one top-level heading per document</li>
  <li><strong>MD031</strong> — Fenced code blocks surrounded by blank lines</li>
  <li><strong>MD034</strong> — No bare URLs — use link syntax instead</li>
  <li><strong>MD041</strong> — First line should be a top-level heading</li>
</ul>

<h2>Privacy</h2>
<p>All processing is client-side. Your Markdown content is never uploaded or stored.</p>
