<h2>使い方</h2>
<ol>
  <li>ページ読み込み時に 64 文字の <strong>code_verifier</strong> が自動生成される。</li>
  <li><strong>再生成</strong> ボタンで新しいランダム verifier を取得、または独自の値を貼り付け。</li>
  <li><strong>Challenge メソッド</strong> を選択 — 明確な理由がない限り S256 のまま。</li>
  <li><strong>code_challenge</strong> はリアルタイムで更新。/authorize リクエストにコピー。</li>
  <li><strong>認可 URL プレビュー</strong> を展開してエンドポイント・client_id・redirect_uri を入力。</li>
  <li><strong>トークン交換（cURL）</strong> を展開すれば、/token への即実行可能なコマンドが手に入る。</li>
</ol>

<h2>主な用途</h2>
<ul>
  <li><strong>ローカル OAuth デバッグ：</strong>verifier/challenge ペアを生成し、認可 URL をブラウザに貼って code を取得、curl プレビューで /token 交換を再現。</li>
  <li><strong>モバイル・デスクトップクライアント：</strong>バックエンドを立てずに iOS / Android / デスクトップアプリのテストフローを手動で実行。</li>
  <li><strong>新規プロバイダー対応：</strong>コードを書く前にプロバイダーが S256 を受け入れること、client_id 設定が正しいことを検証。</li>
  <li><strong>セキュリティレビュー：</strong>仕様書を開かずにライブツールで verifier の文字セットと長さが RFC 7636 に準拠しているか確認。</li>
</ul>

<h2>仕組み</h2>
<ol>
  <li><code>crypto.getRandomValues</code> で 32 バイトの乱数を取得し、base64url エンコード後に目的の長さに切り詰める。</li>
  <li>S256 では、verifier を UTF-8 エンコードしてから <code>crypto.subtle.digest('SHA-256', …)</code> でハッシュ計算。</li>
  <li>32 バイトのハッシュを base64url エンコード（+ を -、/ を _、= パディングを除去）して 43 文字の challenge を生成。</li>
  <li>plain モードでは challenge は verifier と同一。互換性のためだけに残されている。</li>
</ol>
