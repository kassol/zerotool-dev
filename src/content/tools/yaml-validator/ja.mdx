<h2>使い方</h2>
<ol>
  <li>YAMLの内容をテキストエリアに貼り付けます。</li>
  <li><strong>Validate</strong> ボタンをクリックします（または <kbd>Ctrl</kbd>+<kbd>Enter</kbd>）。</li>
  <li>YAMLが<strong>正しい</strong>場合、緑色の成功メッセージが表示され、解析済み構造がJSON形式で表示されます。</li>
  <li>YAMLが<strong>不正</strong>な場合、エラーボックスに理由・行番号・列番号と問題箇所のスニペットが表示されます。</li>
</ol>

<h2>エラー出力の見方</h2>

<p>バリデーション失敗時、エラーボックスには以下が含まれます：</p>
<ul>
  <li><strong>理由</strong> — 何が問題かの説明（例：<em>インデント不正</em>、<em>ストリームの予期しない終了</em>）。</li>
  <li><strong>行番号・列番号</strong> — 入力中のエラーの正確な位置。</li>
  <li><strong>スニペット</strong> — パーサーが示す前後のコンテキスト。多くの場合、問題の文字を指す <code>^</code> が含まれます。</li>
</ul>

<h2>よくあるYAML構文エラー</h2>
<ul>
  <li><strong>インデントの不統一</strong> — YAMLはネストにスペースのみを使用します（タブは禁止）。同じレベルで2スペースと4スペースを混在させると解析エラーになります。</li>
  <li><strong>キーの後のコロン欠如</strong> — <code>key value</code> は無効です。<code>key: value</code> と記述する必要があります。</li>
  <li><strong>特殊文字のクォートなし使用</strong> — 値に <code>:</code>・<code>#</code>・<code>[</code>・<code>]</code>・<code>{'{'}</code>・<code>{'}'}</code> などを含む場合は、クォートで囲むかエスケープする必要があります。</li>
  <li><strong>タブ文字</strong> — YAMLのインデントにタブは使用できません。すべてスペースに置き換えてください。</li>
  <li><strong>重複キー</strong> — 同じマッピングブロック内の重複キーは、YAML 1.2仕様では技術的に無効です。</li>
</ul>

<h2>例：正しいYAML</h2>

```yaml
server:
  host: localhost
  port: 8080
  tls: true

database:
  driver: postgres
  name: myapp
  credentials:
    user: admin
    password: "s3cr3t"

allowed_origins:
  - https://example.com
  - https://api.example.com
```

<h2>例：不正なYAML（インデントエラー）</h2>

```yaml
server:
  host: localhost
   port: 8080   # エラー：余分なスペースでインデントが不統一
```

<p>バリデーターは不整合が検出された行番号と列番号を報告するため、問題を素早く特定して修正できます。</p>
