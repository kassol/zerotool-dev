<h2>사용 방법</h2>
<ol>
  <li>YAML 내용을 텍스트 영역에 붙여넣습니다.</li>
  <li><strong>Validate</strong> 버튼을 클릭합니다 (<kbd>Ctrl</kbd>+<kbd>Enter</kbd> 단축키 사용 가능).</li>
  <li>YAML이 <strong>유효</strong>하면 녹색 성공 메시지와 함께 파싱된 구조가 JSON 형식으로 표시됩니다.</li>
  <li>YAML이 <strong>유효하지 않으면</strong> 오류 박스에 원인, 줄 번호, 열 번호 및 문제 코드 스니펫이 표시됩니다.</li>
</ol>

<h2>오류 출력 이해하기</h2>

<p>유효성 검사 실패 시, 오류 박스에는 다음이 포함됩니다:</p>
<ul>
  <li><strong>원인</strong> — 무엇이 잘못되었는지에 대한 설명 (예: <em>잘못된 들여쓰기</em>, <em>예상치 못한 스트림 종료</em>).</li>
  <li><strong>줄 번호 &amp; 열 번호</strong> — 입력에서 오류의 정확한 위치.</li>
  <li><strong>스니펫</strong> — 파서가 제공하는 주변 컨텍스트. 종종 문제 문자를 가리키는 <code>^</code>가 포함됩니다.</li>
</ul>

<h2>일반적인 YAML 구문 오류</h2>
<ul>
  <li><strong>일관성 없는 들여쓰기</strong> — YAML은 중첩에 공백만 사용해야 합니다 (탭 금지). 같은 레벨에서 2칸과 4칸 들여쓰기를 혼용하면 파싱 오류가 발생합니다.</li>
  <li><strong>키 뒤 콜론 누락</strong> — <code>key value</code>는 유효하지 않습니다. <code>key: value</code>로 작성해야 합니다.</li>
  <li><strong>따옴표 없는 특수 문자</strong> — 값에 <code>:</code>, <code>#</code>, <code>[</code>, <code>]</code>, <code>{'{'}</code>, <code>{'}'}</code> 등의 문자가 있으면 따옴표로 감싸거나 이스케이프해야 합니다.</li>
  <li><strong>탭 문자</strong> — YAML은 들여쓰기에 탭을 허용하지 않습니다. 모든 탭을 공백으로 교체하세요.</li>
  <li><strong>중복 키</strong> — 동일한 매핑 블록 내의 중복 키는 YAML 1.2에서 기술적으로 유효하지 않습니다.</li>
</ul>

<h2>예시: 유효한 YAML</h2>

```yaml
server:
  host: localhost
  port: 8080
  tls: true

database:
  driver: postgres
  name: myapp
  credentials:
    user: admin
    password: "s3cr3t"

allowed_origins:
  - https://example.com
  - https://api.example.com
```

<h2>예시: 유효하지 않은 YAML (들여쓰기 오류)</h2>

```yaml
server:
  host: localhost
   port: 8080   # 오류: 추가 공백으로 들여쓰기 불일치
```

<p>유효성 검사기는 불일치가 감지된 줄 번호와 열 번호를 보고하여 문제를 쉽게 찾아 수정할 수 있습니다.</p>
