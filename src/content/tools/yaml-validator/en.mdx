<h2>How to Use</h2>
<ol>
  <li>Paste your YAML content into the text area.</li>
  <li>Click <strong>Validate</strong> (or press <kbd>Ctrl</kbd>+<kbd>Enter</kbd>).</li>
  <li>If the YAML is <strong>valid</strong>, a green success message appears and the parsed structure is shown as JSON below.</li>
  <li>If the YAML is <strong>invalid</strong>, an error box shows the reason, line number, and column — plus a snippet of the offending area.</li>
</ol>

<h2>Understanding the Error Output</h2>

<p>When validation fails, the error box contains:</p>
<ul>
  <li><strong>Reason</strong> — a plain-English description of what went wrong (e.g., <em>bad indentation</em>, <em>unexpected end of stream</em>).</li>
  <li><strong>Line &amp; Column</strong> — the exact position of the error in your input.</li>
  <li><strong>Snippet</strong> — the surrounding context from the parser, often showing a caret (<code>^</code>) pointing at the problem character.</li>
</ul>

<h2>Common YAML Syntax Errors</h2>
<ul>
  <li><strong>Inconsistent indentation</strong> — YAML requires consistent use of spaces (never tabs) for nesting. Mixing 2-space and 4-space indentation on the same level causes parse errors.</li>
  <li><strong>Missing colon after a key</strong> — <code>key value</code> is invalid; it must be <code>key: value</code>.</li>
  <li><strong>Unquoted special characters</strong> — characters like <code>:</code>, <code>#</code>, <code>[</code>, <code>]</code>, <code>{'{'}</code>, <code>{'}'}</code> in values must be quoted or escaped.</li>
  <li><strong>Tab characters</strong> — YAML prohibits tabs for indentation. Replace all tabs with spaces.</li>
  <li><strong>Duplicate keys</strong> — while some parsers allow them, duplicate keys in the same mapping block are technically invalid YAML 1.2.</li>
</ul>

<h2>Example: Valid YAML</h2>

```yaml
server:
  host: localhost
  port: 8080
  tls: true

database:
  driver: postgres
  name: myapp
  credentials:
    user: admin
    password: "s3cr3t"

allowed_origins:
  - https://example.com
  - https://api.example.com
```

<h2>Example: Invalid YAML (indentation error)</h2>

```yaml
server:
  host: localhost
   port: 8080   # error: extra space causes inconsistent indentation
```

<p>The validator will report the line and column where the inconsistency is detected, making it easy to locate and fix.</p>
