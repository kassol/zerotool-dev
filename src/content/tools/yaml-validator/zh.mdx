<h2>使用方法</h2>
<ol>
  <li>将 YAML 内容粘贴到文本框中。</li>
  <li>点击 <strong>Validate</strong> 按钮（或按 <kbd>Ctrl</kbd>+<kbd>Enter</kbd>）。</li>
  <li>若 YAML <strong>合法</strong>，显示绿色成功提示，并在下方展示解析后的 JSON 结构。</li>
  <li>若 YAML <strong>不合法</strong>，错误框中显示错误原因、行号、列号及问题代码片段。</li>
</ol>

<h2>错误输出说明</h2>

<p>验证失败时，错误框包含以下信息：</p>
<ul>
  <li><strong>原因</strong> — 错误的通俗描述（如 <em>缩进错误</em>、<em>流结束异常</em>）。</li>
  <li><strong>行号与列号</strong> — 错误在输入中的精确位置。</li>
  <li><strong>代码片段</strong> — 解析器提供的上下文，通常包含指向问题字符的 <code>^</code> 标记。</li>
</ul>

<h2>常见 YAML 语法错误</h2>
<ul>
  <li><strong>缩进不一致</strong> — YAML 要求统一使用空格（禁止 Tab）缩进，同级混用 2 格和 4 格缩进会导致解析错误。</li>
  <li><strong>键后缺少冒号</strong> — <code>key value</code> 是非法的，正确写法为 <code>key: value</code>。</li>
  <li><strong>特殊字符未加引号</strong> — 值中含有 <code>:</code>、<code>#</code>、<code>[</code>、<code>]</code>、<code>{'{'}</code>、<code>{'}'}</code> 等字符时，需要用引号包裹或转义。</li>
  <li><strong>Tab 字符</strong> — YAML 禁止用 Tab 缩进，应将所有 Tab 替换为空格。</li>
  <li><strong>重复键</strong> — 同一映射块中出现重复键在 YAML 1.2 规范中是非法的。</li>
</ul>

<h2>示例：合法 YAML</h2>

```yaml
server:
  host: localhost
  port: 8080
  tls: true

database:
  driver: postgres
  name: myapp
  credentials:
    user: admin
    password: "s3cr3t"

allowed_origins:
  - https://example.com
  - https://api.example.com
```

<h2>示例：非法 YAML（缩进错误）</h2>

```yaml
server:
  host: localhost
   port: 8080   # 错误：多余的空格导致缩进不一致
```

<p>验证器会报告检测到缩进不一致的行号和列号，便于快速定位并修复问题。</p>
