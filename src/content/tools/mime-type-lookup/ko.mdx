<h2>사용 방법</h2>
<ol>
  <li>검색 상자에 확장자(점 포함/미포함 무관), MIME 타입, 또는 임의의 키워드를 입력하면 결과가 실시간으로 필터링됩니다.</li>
  <li>카테고리 칩으로 application / image / audio / video / text / font / multipart / message / model 중 하나로 범위를 좁힐 수 있습니다.</li>
  <li>각 결과의 <strong>복사</strong> 버튼으로 Content-Type 문자열을 클립보드에 넣고, 헤더나 API 요청에 바로 붙여 넣을 수 있습니다.</li>
  <li><strong>파일 감지</strong> 탭으로 전환해 로컬 파일을 끌어다 놓거나 클릭해 선택하면, 감지된 MIME, 추정 확장자, 앞 16바이트 hex, 브라우저가 보고하는 타입이 표시됩니다. 업로드되지 않습니다.</li>
</ol>

<h2>MIME 타입이란</h2>
<p>
  MIME 타입(공식 명칭 「미디어 타입」)은 바이트 시퀀스를 어떻게 해석할지 소프트웨어에 알려 주는 슬래시로 구분된 식별자입니다.
  최상위 타입은 굵은 가족 분류(image, audio, video, text, application, multipart, message, model, font)를, 하위 타입은 구체적인 포맷
  (<code>image/png</code>, <code>application/pdf</code>, <code>text/csv</code>)을 나타냅니다. 전체 문법은 RFC 6838에 정의되어 IANA에 등록됩니다.
  웹 서버는 <code>Content-Type</code> 헤더로 전송하고, 브라우저는 렌더러 선택에 사용하며, API는 요청 본문 협상에 활용합니다.
</p>

<h2>매직 바이트 감지의 원리</h2>
<p>
  모든 바이너리 포맷은 고유한 「지문」으로 시작합니다. 파서는 이를 통해 파일이 자칭과 일치하는지 확인합니다. PNG는
  <code>89 50 4E 47 0D 0A 1A 0A</code>, JPEG는 <code>FF D8 FF</code>, PDF는 <code>%PDF</code>, ZIP은 <code>50 4B 03 04</code>입니다.
  도구는 <code>File.slice()</code>로 앞 64바이트를 잘라 <code>FileReader</code>에 넘기고, 약 30개의 시그니처를 순회하며 첫 일치를 반환합니다.
  오프셋 0이 아닌 시그니처도 있습니다. MP4는 바이트 4의 <code>ftyp</code>, tar는 바이트 257의 <code>ustar</code> 매직을 확인합니다.
</p>

<h2>업로드 검증에 중요한 이유</h2>
<p>
  확장자나 브라우저 자체 신고 MIME만 믿는 것은 전형적인 업로드 취약점의 원인입니다. 사용자가 <code>shell.php</code>를 <code>image.png</code>로
  이름만 바꾸면 브라우저는 즐겁게 <code>image/png</code>라고 보고합니다. 서버 측 바이트 수준 검증이 유일하게 신뢰할 수 있는 방어선이며,
  본 도구는 서버 코드를 작성하기 전에 같은 검증을 로컬에서 미리 시뮬레이션할 수 있도록 만들었습니다. 「Detected MIME」, 「Likely extensions」,
  「Browser File.type」 세 열을 비교하면 어떤 입력이 허술한 검증 로직을 빠져나갈지 몇 초 안에 드러납니다.
</p>

<h2>컨테이너 포맷과 ZIP 문제</h2>
<p>
  현대의 오피스 포맷 대부분은 옛 포맷 위에 만들어진 컨테이너입니다. <code>.docx</code>는 XML의 ZIP 아카이브, <code>.xlsx</code>,
  <code>.pptx</code>, <code>.epub</code>, <code>.apk</code>, <code>.jar</code>는 모두 동일한 ZIP 매직 바이트를 공유합니다.
  도구가 <code>application/zip</code>을 보고할 때 이 상황을 명시합니다. 내부 포맷을 더 확인하려면 ZIP 중앙 디렉터리 항목 이름을 읽어야 합니다.
  「Detected MIME」은 바이트 레벨에서 항상 정확하고, 애플리케이션 레벨 타입은 한 단계 더 필요할 뿐입니다.
</p>

<h2>주요 활용</h2>
<ul>
  <li>S3 / Cloudflare R2 업로드에 올바른 <code>Content-Type</code>을 설정해 브라우저가 파일을 다운로드 대신 렌더링하도록.</li>
  <li>서드파티 API 호출 시 정확한 <code>Accept</code> 값 선택.</li>
  <li>CDN이 <code>.webp</code>를 잘못된 타입으로 서빙해 AVIF 협상을 무너뜨리는 문제 진단.</li>
  <li>파일이 페이지를 떠나기 전에 저렴한 사전 검증으로 활용하여 서버 검증 전에 거친 입력을 걸러내기.</li>
  <li>서버가 Content-Disposition을 잊었을 때 다운로드된 바이너리 blob의 실체 확인.</li>
</ul>

<h2>의도된 경계</h2>
<p>
  이 도구는 조회 테이블과 매직 바이트 감지기입니다. 바이러스 스캐너도, 권한 게이트도, 컨테이너 포맷의 내부 구조 파서도 아닙니다.
  고정 헤더가 없는 순수 텍스트(CSV, JSON, 일반 텍스트, 소스 코드)는 어떤 바이너리 시그니처에도 일치하지 않을 때 브라우저의
  <code>File.type</code>으로 대체됩니다. 그 이상으로 깊이 들어가는 텍스트 내용 감지가 필요하다면, 도메인에 맞춰 조율한 서버 라이브러리를 사용하세요.
</p>
