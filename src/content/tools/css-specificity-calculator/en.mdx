  - question: "What is CSS specificity?"
    answer: "CSS specificity determines which style rule wins when multiple rules target the same element. It is expressed as a three-part tuple (a, b, c) where a counts ID selectors, b counts class/attribute/pseudo-class selectors, and c counts element type and pseudo-element selectors."
  - question: "How is specificity calculated?"
    answer: "a = number of ID selectors (#id). b = number of class (.class), attribute ([attr]), and pseudo-class (:hover) selectors. c = number of element type (div, span) and pseudo-element (::before) selectors. The universal selector (*) and combinators (+, >, ~) contribute nothing."
  - question: "How do I compare two selectors?"
    answer: "Enter both selectors separated by a comma. The tool shows each specificity tuple side-by-side. Compare a column-by-column left to right — higher value in the first differing column wins."
  - question: "What about :not(), :is(), and :has()?"
    answer: ":not() and :is() take the specificity of their most specific argument. :where() always contributes zero specificity. :has() uses the specificity of its argument. This tool applies a simplified single-argument parse for these pseudo-classes."
  - question: "Does inline style or !important count?"
    answer: "Inline styles (style attribute) sit above (a, b, c) and !important overrides everything. This tool only calculates selector specificity — inline styles and !important are out of scope."

<h2>How to Use</h2>
<ol>
  <li>Type a CSS selector into the input field, for example <code>#nav .item:hover</code>.</li>
  <li>The specificity tuple <code>(a, b, c)</code> and a color-coded breakdown appear immediately.</li>
  <li>To compare selectors, separate them with commas: <code>#nav .item, .item:hover</code>.</li>
  <li>Click <strong>Copy</strong> next to any result to copy the tuple to the clipboard.</li>
</ol>

<h2>Reading the (a, b, c) Tuple</h2>
<ul>
  <li><strong>a — ID</strong>: Each <code>#id</code> selector adds 1 here.</li>
  <li><strong>b — Class / Attribute / Pseudo-class</strong>: Each <code>.class</code>, <code>[attr]</code>, or <code>:pseudo-class</code> adds 1.</li>
  <li><strong>c — Element / Pseudo-element</strong>: Each tag name or <code>::pseudo-element</code> adds 1.</li>
</ul>
<p>Compare left to right. The first column where two selectors differ decides which one wins. <code>(1,0,0)</code> always beats <code>(0,99,99)</code>.</p>

<h2>Common Examples</h2>
<ul>
  <li><code>*</code> → <code>(0, 0, 0)</code></li>
  <li><code>p</code> → <code>(0, 0, 1)</code></li>
  <li><code>.class</code> → <code>(0, 1, 0)</code></li>
  <li><code>#id</code> → <code>(1, 0, 0)</code></li>
  <li><code>#id .class:hover</code> → <code>(1, 2, 0)</code></li>
  <li><code>div > p + span::before</code> → <code>(0, 0, 4)</code></li>
</ul>
